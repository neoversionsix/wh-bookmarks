MD REF Config File Glossary



  • CONFIG_PRODUCT: The group of configuration values used in the Oracle Health Unified Analytics and Reporting content for specific analytics content areas. Do not change this value in the configuration file.
  • CONFIG_GROUP: ���The group of configuration code used in the Oracle Health Unified Analytics and Reporting content for a set of particular values. Do not change this value in the configuration file.
  • CONFIG_VALUE: The particular value used in the Oracle Health Unified Analytics and Reporting content to identify a client's configuration. For example, Triage is the triage event in the emergency department workflow. Do not change this value in the configuration file unless the Configuration File Values table below specifies that it needs to be updated. 
  • CONFIG_CLASS: The way that the CONFIG_VALUE value qualifies for the Oracle Health Unified Analytics and Reporting content. The value is the code set field used to qualify the value (for example, Common Data Foundation (CDF)), and then the code set number (for example, 220) that the field is pulled from. Code set fields include Concept CKI, CDF meaning, display, and code value. Clients can adjust this value based on their Millennium Platform configuration.
  • CONFIG_KEY: The specific value that qualifies based on what is set in the CONFIG_CLASS value. Clients can adjust this value based on their Millennium Platform configuration.
  • CONFIG_IND: This specific value used to be used to identify if a configuration value matched a client's Millennium Platform domain. This value is no longer used. The configuration status checking has moved to the CONFIG_F_VERFICATION dataset and this value is always displayed as N and can be ignored.
  • Concept CKI: This CONFIG_CLASS value represents the Concept CKI field on the CODE_VALUE table for a particular code set.
  • DISPLAY: This CONFIG_CLASS value represents the Display field on the CODE_VALUE table for a particular code set.
  • CDF: This CONFIG_CLASS value represents the CDF meaning field on the CODE_VALUE table for a particular code set.
  • CODE_VALUE: This CONFIG_CLASS value represents the Code Value field on the CODE_VALUE table for a particular code set.
  • DEFAULT: This CONFIG_CLASS value represents orders, results for clinical events, and so on.
  • Like: This configuration value references a general text value that applies to all values. For example, in Computerized Physician Order Entry (CPOE), the Order Personnel Position Like value references a general text value that applies to all physician positions.

  • Match: This configuration value references specific physician names. For example, in CPOE, the Order Personnel Position Match value references the specific physician position name.
