Overview

Cerner Millennium provides standardized, consistent display values for end-users. Examples include:

  • Encounter types: "Inpatient" / "Outpatient"
  • Languages: "Portuguese" / "French" / "Sign Language"
  • State/Province: "Missouri" / "Quebec"

Millennium can receive foreign data from many different systems. This data must be translated into Millennium display values.

Code Values & the CODE_VALUE Table

A Code Value is a unique, system-generated numeric value representing reference data. The Cerner Millennium database stores all reference data on the CODE_VALUE table.

  • Thousands of rows, continues to grow
  • Each row contains the display value for its code value

How Millennium Language is Stored

Example row on CODE_VALUE table:
code_value: 2553237765
code_set: 36
display: Portuguese
display_key: "PORTUGUESE"

CDF (Common Data Foundation) Meanings

  • Created by Cerner engineers to provide internal processing
  • The system is hard coded to look for certain values within a code set to proceed appropriately
  • Cerner solutions use CDF Meanings to function properly (trigger actions)
Examples: Displaying patient care hierarchy, displaying items in a drop-down box.

The Locations code set (220) differentiates between locations/units, rooms, and beds based on CDF meanings. The CDF meaning causes values to display in the appropriate hierarchy in Bedrock, Location.exe, and LocAlias.exe.

Code Sets

Code Sets are groupings of like reference data elements that organize code values. They are defined by Cerner and the client.

Note: Approximately the first 100,000 code sets are Cerner defined.

Examples:

  • Patient Visit Type (71)
  • Language (36)
  • States/Provinces (62)
  • Marital Status (38)
  • Counties (74)

Data Sent to Cerner Millennium

  • Cerner Millennium receives data from a variety of non-Millennium sources called interfaces
  • Each interface speaks its own language
  • The database translates data from outside interfaces
  • This allows end-users to understand data from any translated interface

Code Value Alias

A Code Value Alias is a value used to represent a piece of reference data. It is used to translate information from interfaces into standardized Millennium display values.

  • Stored on the CODE_VALUE_ALIAS table
  • Each alias is "tied" or "mapped" to a standardized display value through the code_value attribute (primary key – foreign key relationship)
Examples:
Portuguese aliases: PG7, PT
Sign Language alias: 65318
French alias: FR

Contributor Sources

  • A Contributor Source is the set of reference data a foreign/external system could send to Millennium
  • All potential code value aliases/translations a singular contributor system might send
  • Think of it as "the language a contributor system speaks" — a "translation dictionary"

Contributor System & Source Relationships

Two Types of Relationships

  • 1:1 — Each contributor system uses only one contributor source (like each country speaks one language)
  • Many:1 — Many contributor systems use the same contributor source (like some countries share the same language)
Examples: IDX has its own source. Cerner PathNet and Cerner RadNet share the same source — similar to how Italy, Spain, and Argentina might share a language versus speaking a unique one.

HL7 — Interface Information Format

  • Cerner Millennium receives data from a variety of non-Millennium sources
  • Each system can use different formats and variables
  • HL7 — International standards
  • Health Level Seven provides a standard for exchange, integration, sharing, and retrieval of electronic health information
  • Interfaces send HL7 messages with data in standard "placeholders"
  • Data gets processed by Cerner Millennium

Structure of an HL7 Message

Three major logical structures:

StructureDescriptionDelimiter
SegmentsFirst logical breakdown of an HL7 message (e.g. MSH, PID)Line break
FieldsBreaks down Segments, contained within two vertical linesPipe |Field|
ComponentsBreaks down Fields, separated within a field by a caret characterCaret ^Component^

Empty Fields and Components

Many HL7 messages do not contain information in every field or component.

  • Multiple pipe characters || indicate blank fields
  • Multiple caret characters ^^ indicate blank components

Counting Fields and Components

Within a PID Segment, PID Sequence indicates data placeholders:

  • Whole numbers = field number
  • Decimal numbers = component number
  • Important The segment header (MSH, PID) is NOT counted as a field

The MSH Segment

The Message Header (MSH) begins every HL7 message and may contain event triggers.

Example: An external ADT system admits a patient, sending an MSH segment with an event trigger.

MSH Event Trigger Examples
