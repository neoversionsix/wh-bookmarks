1. Results and Events Overview

What is an Event?

In Cerner Millennium, an event represents any recorded patient interaction or measurement. Examples include:

  • Lab results (e.g., blood chemistry, CBC)
  • Weight measurements
  • Temperature readings
  • Patient progress notes
Key Concept: A result is the outcome of a patient event. Results are stored on the CLINICAL_EVENT table in the database for many types of patient events.

2. Event Codes

An Event Code is a special type of code value reserved exclusively for results. Millennium uses event codes to reference specific patient events.

CODE_VALUE Table
code_value display
4228747 Weight Measured
Remember: The event code (e.g., 4228747) is not the actual result — it is the identifier that tells the system what type of result is being stored.

4. Interfaces & Results Definitions

Event Codes
  • Placeholder for results
  • A number against which discrete pieces of clinical data are stored
  • Must be tied to a Primitive Event Set
  • NOT the actual result itself
Code Value Aliases
  • Foreign/external system identifier used to represent a piece of information
  • Must be tied to a Millennium event code for translation
  • Allows external systems to map to internal event codes
Primitive Event Set
  • Display name of the result as seen by end users
  • Must be tied to an Event Code
  • Lowest viewable value on a Cerner Millennium flowsheet
  • Required for front-end display of results
Event Sets
  • Logical grouping of Primitive Event Sets
  • Creates organizational structure
  • Can be nested to form bigger, more generalized groups of information

7. Event Set Hierarchy (ESH) Overview

Event Sets can be grouped into an organized and configurable view for front-end users called the Event Set Hierarchy (ESH).

  • The ESH includes Primitive Event Sets (display names) and Event Sets (groupings of display names)
  • Event Codes are NOT part of the Event Set Hierarchy — they sit below it
  • The ESH defines what is seen on PowerChart flowsheets
ESH Structure

10. Other Results / Other Documents

Temporary Storage

The "Other Results" and "Other Documents" event sets serve as temporary holding locations:

  • Add-on-the-fly results/documents
  • Non-aliased results — the alias received becomes the display value
Examples of non-aliased results: 574986, NURASA89, Potassium
These are placed automatically under Other Results or Other Documents.

11. All Specialty Sections

The All Specialty Sections branch provides customized groupings of Primitive Event Sets for specific PowerChart flowsheet views.

  • Contains customized groupings tailored for specific clinical workflows
  • Results in increased (dual) maintenance — changes must be made in both the main hierarchy and specialty sections
Note: Increased (dual) maintenance is required when using Specialty Sections — changes may need to be made in both the main hierarchy and the specialty sections.
