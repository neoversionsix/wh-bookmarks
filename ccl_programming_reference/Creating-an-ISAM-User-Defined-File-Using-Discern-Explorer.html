Note

This topic discusses creating custom ISAM (non-RDMBS) tables. Custom ISAM tables were widely used by clients to create custom tables in Cerner Classic. This functionality was carried forward into Millennium Platform. Discern Explorer versions 8.3.9 and 8.3.10 introduced the ability to create temporary and permanent custom RDBMS tables. In most cases using a custom RDBMS table is a better option than using a custom ISAM table. See SELECT INTO TABLE Table_Name for more information on creating custom RDBMS tables.

The Select Into Table table_name command should only be used to create custom tables. It should not be used by Oracle development teams to create Millennium Platform production tables.   

ISAM are files created in Discern Explorer when using SELECT INTO TABLE or Discern Explorer Database Definition Language (DDL). Using Windows NT or AIX creates a C_ISAM file. Using OpenVMS creates an RMS file. Globally, these files are known as ISAM.

Two methods can be used to create an ISAM file that you can maintain and report against using Discern Explorer:

  • Use the SELECT INTO TABLE command to create both the file and Discern Explorer definition for the file. Creating a table using the SELECT INTO TABLE command takes the least amount of effort for creating a table. This method can be used to create subsets of Millennium Platform tables, your own tables that are independent of the Millennium Platform system, or a hybrid of the two types of data. These tables then can be maintained and reported against using Discern Explorer commands. The negative to using this method is that you are limited to having only one column per key and you can only create tables with one level.
     
  • Create your own definition using a Creating Complex Files Using DDL and then create the file using a combination of Discern Explorer programs and operating system commands. You can create keys with multiple fields, tables with multiple levels, and can modify the definition of the file to add, remove, or change fields that take effect immediately after you re-include the file definition.
