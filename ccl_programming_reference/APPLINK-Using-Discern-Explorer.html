APPLINK is the routine used to open an application or a patient's chart.

Note

For this routine to work, the following META tag must be included in the HEAD of the html file: <META content='APPLINK' name='discern'>

Syntax

APPLINK (linkmode,launchobject,commandlineargs)

Metasymbols

linkmode
A numeric value representing the mode to start the application link.

ModeDescription

0

Used for starting a solution by executable name.

1

Used for starting a solution by the application object, such as Discern Analytics 2.0 (DA2.exe).

100

Used for launching a file, link, or executable through a shell execute. Use this to open a URL or link if you have Citrix configured for server-to-client redirection.

launchobject

A textual string representing the executable name, application object, file, or link to start based on the linkmode value. Context variables, when supported by the solution, can be substituted into the APPLINK launchobject. Using the context variable $APP_AppName$ in place of the executable name is useful if you want to open a patient chart into the current solution.

commandlineargs

A textual string representing the command line arguments that are to be passed to the solution. The arguments provided are dependent on the specifications of the parameters expected by the executable. A special command line argument /FIRSTTAB="TABNAME" can be added to the list of arguments if the solution provided matches the solution that is opening the link and a /PERSONID and /ENCNTRID argument is provided with a valid person id and encounter id. This opens the patient chart of the provided person id within the current solution and default to the tab provided in the FIRSTTAB argument. The TABNAME corresponds to the first-matching REPORT_PARAM field of the Discern Report preferences configured in Preference Maintenance (PrefMaint.exe). Additionally, when the /FIRSTTAB parameter is used, a plus sign (+) can be added to the end of the TABNAME to launch into the Quick Add dialog if supported by the tab. This is only supported when Enhanced Navigation is enabled in the solution. Context variables, when supported by the solution, can be substituted into the APPLINK commandlineargs.

Examples

The following example illustrates the launching of a patient chart. If the link is clicked when the user is already in PowerChart, the patient's chart specified by the PERSONID with the encounter specified by ENCNTRID opens in the current solution and defaults to the DISCERN REPORT tab. If the link is clicked from a solution other than PowerChart, the PowerChart solution starts a new PowerChart executable with the provided PERSONID and ENCNTRID but does not default to the tab provided.

<a href='javascript:APPLINK(0,"Powerchart.exe","/PERSONID=589848 /ENCNTRID=5960967 /FIRSTTAB=^DISCERN REPORT^")'>Launch Patient Chart in Discern Report tab</a>

This example illustrates opening a web page through a shell execute. When _Citrix_is configured for server-to-client redirection and the link is clicked, your local default web browser opens and is directed to the link provided.

<a href='javascript:APPLINK(100,"http://www.cerner.com","")'>Launch "http://www.cerner.com" in local web browser</a>

The following example illustrates the launching of a patient chart. If the link is clicked when the user is in PowerChart or any of the PowerChart clones, the patient chart specified by the PERSONID with the encounter specified by ENCNTRID opens in the current solution and defaults to the PowerOrders tab. Since the plus sign + is added to the end of the TABNAME, the Add Order dialog box associated with PowerOrders also opens.

<a href='javascript:APPLINK(0,"$APP_APPNAME$","/PERSONID=589848 /ENCNTRID=5960967 /FIRSTTAB=^PowerOrders+^")'>Open a patient's chart directed to the PowerOrders tab with the Add Order dialog open</a>
