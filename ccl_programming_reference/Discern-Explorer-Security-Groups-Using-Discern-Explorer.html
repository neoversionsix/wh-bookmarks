Before you include objects in the object library, ensure that you understand how Discern Explorer uses groups to provide security. When a user includes or runs an executable program in the object library, the Discern Explorer group to which the user belongs is important to the process.

Millennium Platform security allows access to Discern Explorer. Once you access Discern Explorer, another level of security is in place. Each user is assigned to one of 100 groups named GROUP0 (DBA) through GROUP99. By default, all users with access to Discern Explorer are assigned to GROUP1. Users who are assigned to GROUP1 can run queries and execute programs created by GROUP1 and GROUP0 users. Users who are assigned to GROUP1 can only drop and create programs created by GROUP1 users.

GROUP0 is a database administrator (DBA) and has a higher security than all the other groups. A DBA has all of the privileges of a GROUP1 user and also can create a user as a DBA, create and drop databases, and assign users to groups.

A user must be specifically assigned by a GROUP0 (DBA) user to groups 2-99 using the CREATE CCLUSER command. Groups 2-99 have limited access to Discern Explorer queries and must be granted privileges (using the GRANT EXECUTE or GRANT SELECT commands) to run programs.

Discern Explorer security does not control security to RDBMS tables. Your RDBMS sign-on controls access to the RDBMS tables.

When you include an executable program, the group to which you belong is associated with the object name in the object library. For example, if a DBA includes the source code TESTPROG.PRG, which contains the command, CREATE PROGRAM TESTPROG, the object is stored in the object library as TESTPROG:DBA. If a GROUP1 user includes the same source code, the object is stored in the object library as TESTPROG:GROUP1 This is not a problem if either user did not modify the source code. But if the GROUP1 user changes the source code before including it in the object library, the object code for TESTPROG is not the same for each group.

When you run the executable program, Discern Explorer looks for the object associated with the group assigned to you unless you specify the group associated with the program. For example, the DBA can enter TESTPROG:GROUP1 GO to run the version created for GROUP1 and the GROUP1 user can enter TESTPROG:DBA GO to run the version created for the DBA. Groups 2-99 must be given privileges to execute programs that do not reside in their groups.

If only a DBA version of the program exists, both DBA and GROUP1 users can execute the DBA version by entering TESTPROG GO. If only a GROUP1 version of the program exists, the GROUP1 users can execute the program by entering TESTPROG GO. The DBA must enter "TESTPROG:GROUP1 GO" to execute the program.
