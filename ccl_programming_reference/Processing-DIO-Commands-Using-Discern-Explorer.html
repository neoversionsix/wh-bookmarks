When writing Reportwriter clauses you need to understand how Discern ExplorerÂ interprets and compiles DIO commands. When DIO commands are issued, they are stored in a buffer as part of the output code. This is unlike the column and row statements that determine the position of the data when it is printed. Therefore, if you are using ROW and COL commands, you must allow for room in the buffer to store the DIO commands. For example, if you are using row and column references and you issue the DIO command, COL 0 "{COLOR/20}{B}", the reference for the next print item would be COL 13 to leave room for the DIO commands to be stored in the buffer. The quotation marks around the DIO commands are required to identify how the command is stored in the buffer. You can include multiple DIO commands within the quotation marks.

You can determine whether any of the DIO commands are being overwritten in the buffer by displaying the output using the Displayer (use SELECT INTO MINE in your program). The data that is extracted along with the DIO commands that are being stored are displayed. If any of the DIO commands are overlaid by data, that command does not execute.

A maximum of 300 columns can be stored in the buffer when using DIO commands. Therefore, you should issue the command ROW+1 to increment the lines in the buffer. If you are using row and column references in your program, this command affects the position of the data in the report. If you are using the POSITION command, however, this command increments the buffer only and does not affect the position of the data in the report. This is another reason that using the POSITION command is more efficient.
