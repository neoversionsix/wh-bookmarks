When the source code is created using the editor, it must be included in the object library before it can be run as an executable program. If you use the %EDIT directive to create the source code, use the %INCLUDE directive to include the program in the object library (such as %I source_name, where source_name is the name of the text file to be translated into an executable program). For example, if you enter %E TESTPROGRAM.CCL on the command line to enter your Discern Explorer program through the editor, you would then include the program in the object library by entering %I TESTPROGRAM.CCL.

The %I directive tells Discern Explorer to perform the following steps:

  1. Check the file for errors in Discern Explorer grammar and syntax, and flag any it finds.
  2. Translate (or compile) the file into executable form.
  3. Add the executable program (the new object) to the object library if the CREATE PROGRAM command is used in the program.

If you use the %RUN directive to edit the source code, it is included automatically in the object library when you exit the editor. Using this directive, it is not necessary to include the file with the %INCLUDE directive after saving the file.

The operating system (Linux, AIX, HP-UX) ulimit quotas should be considered for user accounts that are granted access to command-line Discern Explorer to compile programs and perform other tasks. This includes executing ORAGEN3, SELECT INTO TABLE Table_Name, CREATE CCLUSER, and other commands which update the Discern Explorer dictionary files. The ulimit file quota is required to have a higher limit than the size of the dic.dat file, which may exceed 1 GB (1,073,741,824 bytes) in size. See the man ulimit operating system help command, and AIX and Linux resources for additional ulimit documentation.
