Discern Explorer displays a message when you exit the Displayer or after a program has completed if the output did not go to the Displayer. This message is used for ISAM files.

*0)*PERSON:P(N#0)(Rng:0)KEY1<0>Rec<606>
   *1)*ENCOUNTER:E(N#5)(Rng:1)Row<606>
     *2)*ORDERS:O(N#5)(Rng:2)Row<606>
        *3)*ORDER_CATALOG:OC(N#5)(Rng:3)Row<606>

Note

This message identifies the tables that were read and the order in which you wrote the joins in your program. This is not necessarily the order in which Oracle processed them. The KEY column can be ignored. The Rec number will display the same number of rows all the way down. Most of the time it can also be ignored.

For programs that include a PLAN, JOIN, or ORJOIN clause, the first table read is identified in the first line of the message by 0) and the second table read is identified in the second line of the message by 1). If the program reads multiple files using a direct linear path (using a JOIN), the third table read is identified in the third line of the message by 2).

If the program reads multiple files using a nonlinear (disjoint) path (using an ORJOIN), the third table read is identified in the third line of the message by 1) because ORJOIN redirects the read from a linear path to a nonlinear path. See Creating a Nonlinear JOIN Path Using Discern Explorer to learn more about the ORJOIN message.

*Oracle Indexes*

Oracle tries to optimize the query. In doing so, it may decide to use a full scan instead of an indexed read. Use the DM2_CBO_PLAN_ACUTAL_REPORT to determine which indexes Oracle is actually using. See DM2_CBO_PLAN_ACUTAL_REPORT for more information.

To search for the description of the Oracle index, see CCLORAINDEX.
