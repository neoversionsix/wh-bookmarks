A screen application is an interactive program that uses commands to control the placement of data on the screen. It is important to understand that these screen applications can only be ran from command-line Discern Explorer using a terminal emulator. Screen applications allow you to validate the user input, provide help on valid responses, or provide documentation for a given prompt. You also can display a screen that has lines and boxes, or text that is bold, double-wide, underlined, blinking, or displayed in reverse-video. Also use programming constructs, such as GO TO statements.

Complete the following steps to create screen applications:

  1. Define the SQL commands to generate the report. See Define SQL Commands Using Discern Explorer for more information.
  2. Define the program as a PAINT clause, which identifies it as a screen application. See Define the PAINT Clause Using Discern Explorer for more information.
  3. Draw the initial screen using lines, boxes, and text. See Draw the Initial Screen Using Discern Explorer for more information.
  4. Declare variables; create record structures; process data internally to set variables or load record structures. See Declare Variables Using Discern Explorer for more informaiton.
  5. Accept data from the user. See Accept Input Using Discern Explorer for more information.
  6. Set help. See Using SET HELP Using Discern Explorer for more information.
  7. Set validation. See Validating Data Using Discern Explorer for more information.
  8. Perform other programming constructs. See Performing Additional Programming Constructs Using Discern Explorer for more informaiton.
