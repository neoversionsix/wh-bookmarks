You can develop a customized menu using the Explorer Menu application. The menu allows you to run Discern Explorer executable programs from within an application rather than requiring the user to have access to the Discern Explorer building tools. The menu provides the ability to execute programs. It also controls security to those programs.

The Explorer Menu application provides access to two levels. One level provides access to build and maintain the menu items. The second level provides access to run programs that have already been added to the menu. Most often, there is a centralized responsibility to build and maintain the Explorer Menu. This allows the client to build their customized menu and control the programs being added along with the security to each.

If the Explorer Menu is built with security, the menu is displayed based on the current users application group. Users only see the menus and programs to which they have security.

The menu can be set up to display two types of items:

  • Discern Explorer executable programs
  • Submenus

The use of sub-menus allows the capability to group programs for easy access and adds the ability to secure a group of related programs.

The Explorer Menu Help Pages documents the application and explains the process for customizing the menu and adding Discern Explorer executable programs.

For a Discern Explorer program to be used in Explorer Menu, it must conform to the following:

  • The program cannot be a screen application that uses the PAINT clause.
  • The program must have a SELECT INTO statement with a prompt for an output device.
  • If the program does not use Reportwriter, the WITH FORMAT, SEPARATOR = " " commands must be used.
  • The line length cannot exceed 2000 columns.
  • The length of the prompts cannot exceed 59 characters.
