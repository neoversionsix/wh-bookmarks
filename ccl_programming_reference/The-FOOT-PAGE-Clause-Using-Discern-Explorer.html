The FOOT PAGE clause enables you to print a page footer and perform report items at the end of each page. The report items in the FOOT PAGE clause are executed when the maximum number of rows (MAXROW) has been printed or a BREAK command is encountered. The syntax of the FOOT PAGE clause is shown below:

Syntax

[FOOT PAGE report_item {,report_item }]

Keep these points in mind when using the FOOT PAGE clause:

  • The FOOT PAGE clause is optional.
  • A SELECT command can include only one FOOT PAGE clause.
  • If you use the FOOT PAGE clause, you must use at least one report item.
  • If you use the FOOT PAGE clause, you must also use a HEAD PAGE clause.
  • Use absolute row referencing in the FOOT PAGE clause

The following example uses the FOOT PAGE clause to print the program name at the bottom of each page.

Example

FOOT PAGE
	ROW 57
	COL 0 LINE_S
	ROW 58
	COL 0 "This report created by the Discern Explorer Program:"
        COL +1 CURPROG
	ROW 59
	COL 0 LINE_S

In the FOOT PAGE clause, we use absolute row referencing instead of relative row referencing. Absolute row referencing allows us to always place the page footer at the bottom of the page. In our example, we always want the footer to be on lines 57, 58, and 59 Using absolute row referencing also helps us avoid the following error:

%CCL-W-246-INPATS(0,0){MREL.}Row movement in FOOT PAGE caused a page break and will be ignored.

This error is caused when a relative row reference would advance the current row past the maximum number of rows allowed on the page. When using absolute row referencing in the FOOT PAGE clause, you must force a page break before you reach the end of the page. The BREAK command is used to force page breaks. You need to execute the BREAK command while enough room exists to print the report items in the FOOT PAGE clause. For example, if the FOOT PAGE prints information on lines 57, 58, and 59, you want to issue a BREAK command before any report item gets printed on line 57, 58, or 59. The following example forces the page break:

DETAIL
	IF(ROW + 1 >= 57) BREAK ENDIF
	COL 10 NAME
        ROW +1
