The Discern Explorer Data Dictionary is your guide to Oracle tables. The dictionary provides the following information:

  • The tables on which specific columns are located.
  • The names of those columns.
  • The data type and length of those columns.
  • The names of indexes.
  • A description of how the tables and columns are used.

Relational Databases (RDB) provide an easy way to store data from many different disciplines as well as lots of disparate data. The advantage of an RDB is that information that is alike, unique, and specific to some defined process or concept is stored in one table (like an ENCOUNTER table). Information related to that table that may be repeated (like orders for an encounter) further qualifies the original table (admitting physician information or diagnoses) and is stored on another related table. One or more unique keys or indexes connect the tables. For example, a person's unique ID is stored on the PERSON table and the ORDERS table. When you use Discern Explorer to select all persons and orders with the same person ID, the rows from each table relate to the other because they share the same person ID.

Each table has columns and those columns in turn have data types. Data types indicate the type of data (numeric, character, date and time) as well as its length. We take some liberties by using some nonrelational terms like column and row. In true relational form, a database is made up of multiple tables that store specific information. A table contains multiple rows of data that is made up of multiple columns. The intersection of a row and column (such as a piece of data like "Bob Jones") is called an attribute.

Discern Explorer provides two programs, CCLGLOS and CCLORATABLE, to enable you to access the Data Dictionary and generate reports of the columns, tables, and indexes. To run the program, type the program name followed by GO on the Discern Explorer command line and press ENTER.
