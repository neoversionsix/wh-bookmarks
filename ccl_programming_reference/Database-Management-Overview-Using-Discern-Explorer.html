Note

This topic discusses creating custom ISAM (non-RDMBS) tables. Custom ISAM tables were widely used by clients to create custom tables in Cerner Classic. This functionality was carried forward intoMillennium Platform. Discern Explorer versions 8.3.9 and 8.3.10 introduced the ability to create temporary and permanent custom RDBMS tables. In most cases using a custom RDBMS table is a better option than using a custom ISAM table. See SELECT INTO TABLE Table_Name for more information on creating custom RDBMS tables.

The Select Into Table table_name command should only be used to create custom tables. It should not be used by Oracle development teams to create Millennium Platform production tables. 

Occasionally clients need to create files that are either subsets of Millennium Platform system files or are separate from data in Millennium Platform. Some examples could be all patients who had STAT orders in the past month or a program that tracks job titles, anniversaries, and job descriptions. You can create your own database, insert records into that database, update, and even delete records from that database using Discern Explorer. This topic illustrates several options to help you create and maintain your own databases.

You can create and maintain your own databases and then create reports against these databases using Discern Explorer commands. Three commands (INSERT, UPDATE, and DELETE) enable you to perform database maintenance while still using the familiar SELECT command to create reports against your files.

Your operating system sign-on controls access to database management commands. Only those users with the appropriate operating system privileges can use the INSERT, UPDATE, and DELETE commands to modify tables. To modify tables, you must have write privileges granted to you or be the owner of the user-defined table you want to modify.

Several steps should be followed when creating and maintaining your own databases.

  1. Create the file and Discern Explorer definition.
  2. Define the file with modify privilege.
  3. Create programs to perform the following:
    1. Insert rows on the table.
    2. Update rows on the table.
    3. Delete rows on the table.
    4. Create reports from the rows on the table.
