Keep the following points in mind when creating a simple query:

  • The names of the columns from which data is to be extracted must be included in the SELECT command.
  • Ensure that column names include the name of the table in which they are included, separated by a period (such as PERSON.NAME_FULL_FORMATTED).
  • Separate columns with a comma if more than one column is selected.
  • All tables from which columns are selected must be listed in the FROM clause.
  • Columns are displayed or printed in the order in which they are included in the SELECT command.
  • Use optional display options to override default Discern Explorer column formatting.
  • A single display option can be used, but missing options must be replaced with semicolons if following options are used.
  • The WHERE clause must have at least one expression, a relational operator, and an expression. It also can have multiple expressions connected by the qualifiers AND and OR.
  • A column can be used to qualify the output without first selecting it to be included in the output.
  • Rows are sorted according to the order in which the columns are listed in the ORDER clause.
  • You can use a column to order the output without including it in the output.
  • Sorts can be nested up to ten deep.
  • The WITH clause can include multiple options, separated by commas.
