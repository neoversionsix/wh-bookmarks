Before you create an executable program, check the object library to determine whether the program name you plan to use, has been used previously. This will eliminate the possibility of overriding someone else's object code in your group. To list the programs by name in a group, use CCLPROG or CCLPROT. If a program exists with the indicated name, it will be displayed.

Complete the following steps to create and run an executable program:

  1. Access the editor.
  2. Write the source code.
  3. Save the source code and exit the editor.
  4. Include the source code to create the object in the object library.
  5. Execute the program.

Complete the following steps to create and run an executable program:

  1. Run CCLPROG to check that the object name you plan to use does not already exist.
  2. Write the source code, including DROP and CREATE PROGRAM commands in the editor. Save and include the file.
  3. Execute the program.

It is a good habit to check the object library before creating a new executable program using the DROP PROGRAM and the CREATE PROGRAM commands. This eliminates overriding someone else's object code in your group. To view program names by group, use the program CCLPROG and enter the object name you want to define. Discern Explorer® displays the object name if one exists. Complete the following steps to view program names by group:

  1. Sign on to Discern Explorer. At the command line, type in CCLPROG GO.
  2. Press ENTER.
  3. Press ENTER to accept MINE.
  4. Press P for program.
  5. Press ENTER.
  6. Type in the object name TABLEDEF2.
  7. Press ENTER.
    1) CCLPROG GO
    MINE
    P
    TABLEDEF2
    The object name, TABLEDEF2, is displayed under the DBA group.
  8. Run CCLPROG before creating executable programs to ensure that the object name does not already exist.
  9. Click Exit to exit the Displayer.
  10. Run the CCLPROG program again using <your_init>_EXEC as the object name.
  11. Click Exit to exit the Displayer.

If no records qualify, the object name does not exist and is available for you to use. If an object has already been created with the name <your_init>_exec, run CCLPROG again using a different set of initials until you find an object name that is not in use.

For the second part of the process, use an editor to create a text file containing the Discern Explorer commands. Two new commands are introduced in the following topics:
