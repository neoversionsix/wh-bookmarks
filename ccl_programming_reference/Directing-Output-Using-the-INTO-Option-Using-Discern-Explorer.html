When you use the SELECT command to create a query, the system automatically displays the output of the query on the terminal screen. To override this default, specify the output device using the INTO option. The syntax of the SELECT command with the INTO option is shown below:

SELECT [INTO output_device]

output_device

Identifies where the results of the query should be sent. See the following list of valid values: |

Value

Description

CRT

Prints to a dedicated printer.

CURDIO

Temporarily stores the output of a select that uses the DIO = dio_device control option in memory. After the SELECT INTO CURDIO completes, CURDIO can be used to access the memory. For example, Set rec->text = CURDIO could be used to move the results of the SELECT INTO CURDIO into the text field of a record structure named rec.

file_name

Stores the results to an extract file with the name CCLUSERDIR:file_name.DAT, where file_name is the name of the file created when you send the output to a file instead of a printer or terminal.

FORMS

Displays output using the Displayer.

MINE

Displays output using the Displayer.

NOFORMS

Scrolls the results to the terminal.

NL:

Selects output into null. This option may be used to internally process data or load an array or record structure. (SELECT INTO "NL:")

printer_name

Spools the results to a printer using the logical in the format LPnn, where nn represents the number assigned to the printer.

TABLE table_name

Builds an extract file and automatically stores the definition for the file in the data dictionary. The new file is written to your default directory (usually CCLUSERDIR) and has the same name as the table_name. Any previous definition with that table name is automatically dropped and replaced with the new definition. Control options WITH ORGANIZATION, WITH SIZE, WITH FORMAT, and WITH APPEND, are useful with the SELECT INTO TABLE option. The ORDER clause determines keys for INDEXED files. This clause cannot be used with reports.

TRIM(expression)

Use this form to trim any trailing spaces if the expression is a global variable set to a file, printer, CRT, CSS, FORMS, MINE, or NOFORMS.
