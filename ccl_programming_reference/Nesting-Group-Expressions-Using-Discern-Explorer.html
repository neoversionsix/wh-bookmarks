You can nest HEAD group_expression clauses up to ten deep. For example, you might have an ORDER clause like the one below, which groups the patients in a nursing unit by gender.

ORDER	NURSE_UNIT,
	GENDER_DISP,
	P.NAME_LAST_KEY

Instead of printing the gender data in each detail row you can add a group_expression to print the gender data one time for each logical group.

HEAD NURSE_UNIT
	COL 0 "Nursing Unit:"
	COL +2 NURSE_UNIT
	ROW +1
HEAD GENDER_DISP
	COL 0 GENDER_DISP
	COL +1 " Patients In This Nursing Unit"

HEAD group_expression clauses must be in the same order as the group_expression items are listed in the ORDER clause. You do not have to create a HEAD group_expression clause for every item in the ORDER clause. But the HEAD group_expression clauses that are used must follow the order of the items in the ORDER clause. For example, if you have the following ORDER clause:

ORDER	NURSE_UNIT,
	GENDER_DISP,
	P.NAME_LAST_KEY

You could use the following HEAD group_expression clauses:

HEAD NURSE_UNIT
	COL 0 "Nursing Unit:"
	COL +2 NURSE_UNIT
	ROW +1
HEAD P.NAME_LAST_KEY
	COL 0 P.NAME_LAST_KEY
	ROW +1

But the following is incorrect because the HEAD group_expression clauses do not follow the order of the ORDER clauses:

HEAD P.NAME_LAST_KEY
	COL 0 P.NAME_LAST_KEY
	ROW +1
/*This is incorrect because the Nurse_Unit is listed before P.Name_Last_Key in the ORDER clause.*/
HEAD NURSE_UNIT  
	COL 0 "Nursing Unit:"
	COL +2 NURSE_UNIT
	ROW +1

You can use Row + 0 as the only report item in a HEAD group_expression if you do not want to perform any specific actions on the group.
