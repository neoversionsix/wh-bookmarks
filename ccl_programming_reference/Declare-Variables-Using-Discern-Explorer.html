This page discusses declaring variables in the context of a backend screen application. For information and examples of creating variables in non-screen application programs, see the Declare command.

The next step is to declare and initialize user variables. Only two variables are required in the program example. They are used to store the data that is accepted when the user runs the program (printer and username). The program below defines the printer as a character string using the FILLSTRING( ) function and then sets the default to "MINE". The username is initialized and the default set to "ABC123" in one command.

DROP PROGRAM CCL_SCREEN4 GO
CREATE PROGRAM CCL_SCREEN4

PAINT

#INITS
SET PRINTER = FILLSTRING(20," ")
SET PRINTER = "MINE"
SET USER_NAME = "ABC123"

#DRAW_BOX
CALL BOX(2,2,20,78)
CALL LINE(6,2,77,XHOR)
CALL TEXT(4,25,"EXAMPLE SCREEN APPLICATION!")
CALL TEXT(8,5,"01 PRINTER:")
CALL TEXT(10,5,"02 ENTER USERNAME:")

CALL VIDEO(R)
CALL TEXT(20,20,"PRESS <HOME> TO END, <HELP> FOR INFO")
CALL VIDEO(N)

CALL PAUSE(5)

SELECT	ACT = EVALUATE(P.ACTIVE_IND,1,"ACTIVE",2,"INACTIVE"),
	P.PERSON_ID,
	DEPT = UAR_GET_CODE_DISPLAY(P.DEPARTMENT_CD),
	POSITION = UAR_GET_CODE_DISPLAY(P.POSITION_CD),
	P.NAME_FULL_FORMATTED,
	P.USERNAME,
	P.EMAIL,
	P.PHYSICIAN_IND,
	P.UPDT_DT_TM
FROM	PRSNL P
WHERE	P.USERNAME = "ABC123"

WITH NOCOUNTER


END
GO

SET Variable Command

The variables are declared and initialized using the SET variable command. The syntax of this command is shown below:

SET variable = expression [GO]

The variable name can be any combination of letters or numbers. Ensure that you do not use names that are already in use by Discern Explorer (any command like SELECT or SET, or any reserved variables like CURDATE or CURACCEPT). The expression to which you set a variable can be any alphabetic, numeric, or mathematics equation, an array, or a function (like the FILLSTRING( ) function).

The word "GO" is optional based on whether the command is contained inside an executable program or not. The word "GO" is not used in the CREATE PROGRAM command; it is used only as a stand-alone command outside the constructs of CREATE PROGRAM and END.
