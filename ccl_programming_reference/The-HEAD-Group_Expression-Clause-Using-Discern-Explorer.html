The HEAD group_expression clause enables you to execute report items on logical groups of data returned by the SELECT command.

Syntax

The syntax of the HEAD group_expression clause is shown below.

{HEAD group_expression
	report_item {,
	report_item }}

Keep these points in mind when using the HEAD group_expression clause:

  • The HEAD group_expression clause is optional.
  • A query can include a maximum of ten HEAD group_expression clauses.
  • Group_expressions used in the HEAD group_expression clause must be listed in the ORDER clause and be listed in the same sequence as the expressions in the ORDER clause.
  • If the HEAD group_expression clause is used, at least one report item must be used.

The HEAD group_expression clause is useful for two purposes:

  • Set up an expression that has logical groups of information associated with it.
  • Perform report items whenever a new expression or logical group is encountered.
