Logical groups of information are created when you use an ORDER clause in your SELECT command. Once the data is extracted, the ORDER clause sorts the data. The columns, or elements, you define in the ORDER clause coincide with the HEAD group_expression clause. By sorting the query, you are creating logical groups of data. The examples below have sorted the selected information using the ORDER clause.

ORDER	NURSE_UNIT,
	P.NAME_LAST_KEY

Using the logical groups, you can perform report items in the HEAD or FOOT clause. For example, instead of printing the nursing station in the DETAIL clause for every patient, you can use a HEAD group_expression clause to print the nursing station as the heading for that group of patients, as shown in the example below.

HEAD NURSE_UNIT
	ROW +1
	COL  0 "Nursing Unit:"
	COL +2 NURSE_UNIT
	ROW +1

The HEAD group_expression clauses are located above the DETAIL clause and should be in the same order as expressions listed in the ORDER clause.

In the example, two expressions are listed in the ORDER clause; the select expression Nurse_Unit and the column P.NAME_LAST_KEY. A HEAD group_expression clause is defined only for the nursing station. The person's name is printed in the DETAIL clause. It is necessary to create a HEAD group_expression clause only when the data is used to create a heading for a logical group.
