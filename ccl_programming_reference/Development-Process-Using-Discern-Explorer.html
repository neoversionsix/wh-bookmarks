Oracle recommends that all new custom programs, queries, or reports be created and tested in a nonproduction environment. Test the system to ensure that the correct data is returned and that programs, queries, and reports are as efficient as possible. Use the program DM2_CBO_PLAN_ACTUAL_REPORT to review the efficiency of the program.  Steps for validating that the returned data is accurate are unique to each program, query, or report.  After a program, query, or report has been developed and tested it can then be moved into the production environment. Establish change control procedures that require review of the code and the testing procedures before moving a program, query, or report into a production environment. Ensure that the change control procedures outline a requirement for saving source code files in a standard location.

Make modifications to existing programs, reports, and queries in a nonproduction environment, tested for efficiency and accuracy of the data, and then moved to the production environment. Establish change control procedures for copying source code files from a standard location to the nonproduction environment before modification and for copying the modified source code files back to the standard location after testing and validation of the modifications have been completed.
