Discern Explorer is described as a nonprocedural language because the rules of syntax that you must follow are few and Discern Explorer is tolerant of variations in how programs are formatted. The nonprocedural aspect of Discern Explorer gives you great flexibility in how you format or visually arrange your source code. Because spaces and blank lines generally have no effect on Discern Explorer processing, they can be used to make source code understandable for reading, editing, and debugging.

Although Discern Explorer is a nonprocedural language, certain syntax rules exist that must be followed for a program to run correctly. These rules are listed in the following table.

Element

Interpretation

/* */

The /* indicates the start of a block comment in a source code file and the */ indicates the end of the block comment. This enables Discern Explorer to distinguish between comments and Discern Explorer commands. Comments can be included in a Discern Explorer source code file, and in no way affect the execution of the program. Block comments cannot be nested in block comments.

;

The semicolon and everything to the end of the line is treated as a comment. If the semicolon is not the first character on the line, a backslash \ at the end of the line is treated as a continuation character. If the semicolon is used as the first character of a line, and the backslash \ is placed at the end of the line, the backslash is part of the comment.

!

When used as the first character of a line, the exclamation point and everything to the end of a line is treated as a comment. If the backslash \ is placed at the end of the line it is treated as part of the comment. When an exclamation mark ! is immediately followed by an equal sign =, that combination of characters is treated as a not equal operator instead of the beginning of a comment.

,

Commas separate column names, table names, and options. Certain reserved words also serve as separators (for example, FROM, WHERE, and ORDER BY).

%

The percent sign indicates a directive command in the Discern Explorer program (such as %EDIT or%HELP).

( )

Parentheses alter the precedence that the system follows when executing Discern Explorer commands. Standard algebraic precedence is followed unless parentheses are used to change it. Parentheses also denote that an argument must be used with a function, such as CNVTDATE().

" "

Quotation marks denote character data, as opposed to numeric data. Quotation marks are not required for numeric entries; they are required for character strings. The following delimiters can be used to denote character strings.
' single quote
" double quote
^ flex
~ wave
| bar
@len:string@ where len is the number of characters to treat as a string and string is the actual character string.

\

Discern Explorer limits the length of a line in a source code file to a maximum of 132 characters. The backslash \ is used as a continuation character at the end of a line in a source code file in the Create Program command. When used this way the backslash indicates that the line continues onto the next line of the file. This is helpful when you try to assign a long character string to a variable or record structure item.

If a semicolon ; or an exclamation mark ! is used as the first character of a line and the backslash \ is placed at the end of the line, the backslash \ is part of the comment. The next line is not a continuation of the comment.

If the backslash \ is placed at the end of the line in a comment that begins with the ; or ! indented, not as the first character of a line, the backslash \ is a continuation character. The next line is a continuation of the comment.

Discern Explorer version 8.12.3 increases the maximum size of a literal string symbol from 32,767 characters to 65,535 characters. Before Discern Explorer version 8.12.3, the maximum size of a string that could be created using the continuation character in a compile was 32,767 characters. In Discern Explorer versions 8.12.3 and later, this size is 65,535 characters.

Beginning in Discern Explorer versions 8.17.0 and 9.05.0 compiling with SET TRACE WARNING3 enabled, places CONTINUATION: at the beginning of lines in the listing that are continued from using the backslash \ at the end of the previous line.

When qualifying on character strings, Discern Explorer treats the * and ? as wildcards. The backslash can be used to qualify on a literal * or ? character. For example, Where O.Order_Mnemonic = "*TCOUNT" would qualify any Order_Mnemonic that ended with " TCOUNT". Using Where O.Order_Mnemonic = "\ * TCOUNT" would qualify any Order_Mnemonic that began with "* TCOUNT". That is the literal * followed by TCOUNT. Two backslashes are used to qualify on a literal backslash. For example Where O.Order_Mnemonic = "\ \ R*" would qualify any Order_Mnemonic that began with the literal backslash character followed by the letter R followed by any number of characters.

GO

Use GO after a Discern Explorer command to execute the command. GO is not used after Directive commands (such as %EDIT or%HELP). GO is not used after commands that are in a CREATE PROGRAM command. Instead the GO is used at the end of the CREATE PROGRAM command.

CAPS

Discern Explorer is not case-sensitive with respect to commands. Discern Explorer is case-sensitive only when processing character strings.

integer.W

Beginning in Discern Explorer version 8.13.0, using .w at the end of an integer causes the integer to be treated as a W8 data type. For example:

declare var1 = W8

set var1 = 123456789012345678.w

Before Discern Explorer version 8.13.0, CNVTLONG() is needed to assign a value to a W8 data type item. For example:

declare var1 = W8

set var1 = cnvtlong("123456789012345678")

To make your source code more readable, you can use blank lines between each major section. Using spaces and tabs effectively also can make source code more readable. An example of such formatting is used in the source code presented below.

SELECT	P.NAME_FULL_FORMATTED,
	P.BIRTH_DT_TM,
	P.PERSON_ID,
	P.LANGUAGE_CD,
	P.SEX_CD

FROM	PERSON P

WHERE	P.BIRTH_DT_TM > CNVTDATETIME("01-JAN-1997")

GO

This source code is easier to read than it would be if all the tabs were deleted, as is the case in the following example.

SELECT  P.NAME_FULL_FORMATTED, P.BIRTH_DT_TM, P.PERSON_ID, P.LANGUAGE_CD, P.SEX_CD
FROM  PERSON P WHERE P.BIRTH_DT_TM > CNVTDATETIME("01-JAN-1997") GO

You may see variations in how spaces are used in the sample programs throughout the Help Page topics, but you need not use the spacing re-created here. Develop your own standards that make the source code understandable to you and to other users.

Discern Explorer commands are not case-sensitive. Commands written in uppercase or lowercase are treated the same. In Discern Explorer, text embedded in quotations, stored in character fields, variables, and record structures are case-sensitive.

Discern Explorer commands can be continued across multiple lines. This provides further flexibility in how programs are formatted and increases readability. The following examples illustrate three possible ways to enter the same Discern Explorer command. Note, also, the variation in spacing and capitalization in these examples.
