You can make two types of row or column references, absolute and relative. An absolute reference names the position specifically as in, ROW 3 or COL 50. An absolute row movement does not affect the column number. A relative reference describes the position as it relates to some other point as in ROW +1 or COL +5. The reference is relative to the current row or column position. A relative row movement resets the column number to 0.

A relative reference can move forward using the plus (+) sign, or backward using the minus (-) sign. If the last character was printed at column 30, and you specify the next report item is to be printed at COL +2 the next report item is printed at column 32. Likewise, if the last line is printed on row 20 and you specify the next report item is to be printed at ROW +1 the next report item is be printed on row 21.

If you do not specify a row number, Discern Explorer defaults to the current row and prints the next item on the same row as the previous item. To avoid printing report items on top of each other in the DETAIL clause, the row must be incremented after each row is read.

When you print a select_expression, the length of the expression is fixed. For example, the NAME_LAST_KEY column on the PERSON table is defined as a 100-byte character column. The data stored in the column, may be less than 100 characters, such as JONES. When the NAME_LAST_KEY column is printed on the report starting in column one, the printing stops in column 100, regardless of how many characters are actually stored in the NAME_LAST_KEY column. Therefore, if the next report_item issued is COL +2, the next item starts printing at column 102.

The following example shows absolute and relative referencing in the DETAIL clause.

DETAIL	COL P.NAME_FULL_FORMATTED
	COL 50 SEX_DISP
	COL +2 P.BIRTH_DT_TM
	ROW +1

Keep these points in mind when using ROW and COL commands.

  • A relative row movement resets the column number to 0. Therefore, if the current position is row 20, column 30, when you issue a ROW +1 command, the report_item begins printing at row 21, column 0.
  • An absolute row movement does not affect the column number. Therefore, if the current position is row 20, column 30, when you issue a ROW 25 command, the report_item begins printing at row 25, column 30.
