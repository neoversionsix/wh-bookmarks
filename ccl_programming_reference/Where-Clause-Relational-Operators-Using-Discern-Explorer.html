Relational operators separate the expressions in a WHERE clause.

The following are valid relational operators:

OperatorDescriptionExample

=

Equal to

P.NAME_LAST_KEY = "SMITH"

!=

Not equal to

P.NAME_LAST_KEY != "SMITH"

>

Greater than

E.REG_DT_TM > CNVTDATETIME("01-FEB-1999")

<

Less than

E.REG_DT_TM < CNVTDATETIME("01-FEB-1999")

>=

Greater than or equal to

E.REG_DT_TM >= CNVTDATETIME("01-FEB-1999")

<=

Less than or equal to

E.REG_DT_TM <= CNVTDATETIME("01-FEB-1999")

BETWEEN

Inclusive range

Enables you to specify a value between two expressions.
P.NAME_LAST_KEY BETWEEN "A*" AND "C*"

IN

Multiple OR qualification

Enables you to specify a value in one or more expressions. P.NAME_LAST_KEY IN ("SMITH", "ROBERTS", "TAYLOR")

Compare the same data type when qualifying on expressions. When qualifying on character fields, embed the value in quotes. Also, the character data stored in your Oracle system is case-sensitive. It is helpful to perform some data validation before qualifying on columns. Simply select the columns from the tables and add the control option of MAXREC = 100. This retrieves 100 records and allows you to view the actual data stored in those columns.
