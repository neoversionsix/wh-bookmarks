You can sort data first by a primary column, and then in the primary column by a secondary column, and then in the secondary column by a tertiary column, and so on, up to ten deep. Discern Explorer determines which column is primary and which is secondary, by the order in which the sort columns are listed in the ORDER clause. The first listed column is the primary column; the second listed column is secondary, and so forth. Each column listed in the ORDER clause is a subsort of the column listed before it.

The following sample query sorts first by birth date, sex code in each birth date, and then by patient name in birth date. In this example, the BIRTH_DT_TM column is the primary sort column, the SEX_CD column is the secondary sort column, and the NAME_LAST_KEY column is the tertiary sort column.

SELECT	P.NAME_FULL_FORMATTED,
	P.BIRTH_DT_TM "MM/DD/YYYY",
	P.SEX_CD
FROM	PERSON P
WHERE	P.NAME_LAST_KEY > "A" 
	AND P.BIRTH_DT_TM > CNVTDATETIME("01-JAN-1940")
ORDER	P.BIRTH_DT_TM,
	P.SEX_CD,
	P.NAME_LAST_KEY
GO
