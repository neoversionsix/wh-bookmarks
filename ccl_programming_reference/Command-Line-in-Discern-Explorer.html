A command-line query means that you are creating an ad hoc query that extracts data from the database when the command is entered. When you sign on to Discern Explorer, you are at the single command-line editor; this is where command-line queries are entered. A buffer stores the syntax entered on the command line until you execute the command using GO.

Sign in to Discern Explorer.

Basic SELECT Command

The SELECT command is briefly covered here for you to view some of the components of the command line. A more detailed description of SELECT is provided under the SELECT command topic.

The SELECT command has three required elements; SELECT, FROM, and GO.

SELECT
  select_expression
FROM
  table_name
GO

A simple Discern Explorer query uses only the SELECT, FROM, and GO clauses. The select_expressions define the columns or user-defined variables that you want to extract. The FROM clause defines the table containing those columns. GO executes the query. The simple example below extracts the patient's name from the Person table. The multiple example extracts the patient's name and patient birth date.

SIMPLE:

SELECT 	PERSON.NAME_FULL_FORMATTED 
FROM 		PERSON 
GO

MULTIPLE:

SELECT 	PERSON.NAME_FULL_FORMATTED,

PERSON.BIRTH_DT_TM 

FROM		PERSON 
GO

When selecting from a table, separate each select_expression by a comma as shown in the above "multiple" example. The FROM clause follows the select_expressions. Separate each table_name by a comma when accessing more than one table.

SELECT 
	PERSON.PERSON_ID,
	ORDERS.ORDER_MNEMONIC
FROM 
	PERSON,
	ORDERS 
WHERE 
	PERSON.PERSON_ID = ORDERS.PERSON_ID 
GO
  1. Type SELECT PERSON.NAME_FULL_FORMATTED FROM PERSON GO at the command line.
  2. Press ENTER.

Display Counter

The display counter increments as the query is processing. The first number in the record shows the number of physical records read in the first table. The second number indicates the total number of physical records being read between all tables. The row shows the number of logical records being created--these are the number of rows processed during the select to find the data you want to use to qualify. The qual increments when a record matches the criteria you entered for the report (for example, show only inpatients). Sort only increments when you have an ORDER clause. Len displays the total number of columns that are occupied in the query.

Rec:	14676	 14677	Row:	29352		Qual:	14676		Sort:	0		(Len:	116)

Remember that you can press F6, or CTRL+C to interrupt and see the accumulated results without running to completion.

The display counter redisplays when you exit the displayer.

Displayer

The Displayer is the default output device for your Discern Explorer programs. Ensure that you now see a line for each person record in the PERSON table. The column name displays at the top of the displayer. The total number of rows and columns are displayed in the highlighted portion at the bottom of the view. 

Use keyboard shortcuts, the first letter of the item, to access the items at the bottom of your screen.

  1. Press W, (Width) to change between 132 and 80 columns.
  2. Press M, (Margin) and then the <down_arrow>, to jump to the bottom of the output. Use the arrow keys to jump to the top, bottom, left, or right margins.
  3. Press F, (Find) and type part of a persons name and press ENTER.
  4. Press Return key twice to exit the Displayer.

Display Tree

The display tree defines the tables being accessed. The query uses the PERSON table without a join option. We cover aliases in this tutorial and expand on keys and join options in training.
An example of the display tree that is seen when you exit the displayer is illustrated below:

0)PERSON:P(N#0)(Rng:0)KEY1<0>Rec<4512>

Element

Description

0)

Indicates first table read.

PERSON

Table name read.

P

Alias for the table.

(N#0)

No special joining options in effect / #0 tells how null tuples are processed.

(Rng:0)

Identifies number of tables read, 0 is the last table read.

KEY<0>

The key or index used to drive the query, RMS queries only.

Rec<4512>

Total number of records read for this table.

Use the command-line query when doing troubleshooting, data validation, or initial investigation to determine the columns needed. The example below gives you all of the columns in the PERSON table and aligns the columns under the headings using the WITH CHECK option. This command allows you to see the data stored in each column, rather than just the column name from the Data Dictionary. Use the wildcard as a select_expression for data validation.

  1. Type, SELECT * FROM PERSON WITH CHECK GO.
  2. Press ENTER.

Note

The birth datetime columns are Date Quad 8 byte columns, DQ8. This represents both a date column and a time column but only the date displays here. We learn how to format these dates and times in the "Display Options" section.

While at the command line, a buffer holds up to 10 previous lines. To access the buffer, press the UP and DOWN arrow keys. Discern Explorer stores each line in memory until you issue the GO command. If you make a mistake, you can reset the memory buffer by typing RESET or pressing the F19 key.

Note

The F19 key varies from keyboard to keyboard depending upon your emulation.
