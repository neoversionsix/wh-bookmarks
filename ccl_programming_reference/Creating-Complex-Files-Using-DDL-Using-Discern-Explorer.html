Note

This topic discusses creating custom ISAM (non-RDMBS) tables. Custom ISAM tables were widely used by clients to create custom tables in Cerner Classic. This functionality was carried forward intoMillennium Platform. Discern Explorer versions 8.3.9 and 8.3.10 introduced the ability to create temporary and permanent custom RDBMS tables. In most cases, using a custom RDBMS table is a better option than using a custom ISAM table. See SELECT INTO TABLE Table_Name for more information on creating custom RDBMS tables.

The Select Into Table table_name command should only be used to create custom tables. It should not be used by Oracle development teams to create Millennium Platform production tables.

Creating a table using the SELECT INTO TABLE command is fast and easy. If you want multiple columns in a table, multiple table levels, or expect to maintain the definition of the file by adding, changing, or removing fields, you may need to use the Discern Explorer Database Definition Language (DDL). DDL commands provide you with greater flexibility in customizing a file by giving you control over the exact length and format of keys, fields, and tables.

Before you use DDL to create files, you must have a working knowledge of it. The CREATE DATABASECREATE_DATABASE and CREATE DDLRECORDCREATE_DDLRECORD topics will get you started. These two commands are used together to create table definitions.

The syntax of CREATE DATABASE is shown in the following example:

CREATE DATABASE database_name
	ORGANIZATION (organization_type)
	FORMAT (record_format)
	SIZE (integer)
	[ TYPE (directory_type) ]
	{ [UNIQUE] KEY key_number (offset, key_length) }
	[ SCHEMA (schema_name) ]
GO

The syntax of CREATE DDLRECORD is shown in the following example:

CREATE DDLRECORD record_name
FROM DATABASE database_name
	{begin_table_item
	{level_number field_name
	{ (occur_number) ]
	[BY by_name]
	[ = [ field_modifier]  field_type]
	[ REDEFINES field_name ]
	[ CCL (field_name {, field_name } ) ]
	[ GLOSSARY (field_name) ]
	[macro]	}
	end_table_item	}
[WITH ACCESS_CODE = integer]
GO

The CREATE DATABASE command and the CREATE DDLRECORD command usually go in the same source file as one program.
