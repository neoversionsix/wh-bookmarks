Note

This topic discusses creating custom ISAM (non-RDMBS) tables. Custom ISAM tables were widely used by clients to create custom tables in Cerner Classic. This functionality was carried forward into Millennium Platform. Discern Explorer versions 8.3.9 and 8.3.10 introduced the ability to create temporary and permanent custom

Relational Database Management System (RDBMS) tables. In most cases, using a custom RDBMS table is a better option than using a custom ISAM table. See the SELECT INTO TABLE Table_Name for more information about creating custom RDBMS tables.

The Select Into Table table_name command should only be used to create custom tables. It should not be used by Oracle development teams to create Millennium Platform production tables. 

To create the Windows NT or AIX C-ISAM data file, use the SELECT INTO TABLE method. The following example creates an empty C-ISAM file named CATALOG_1 with three keys:

SELECT INTO TABLE "TABLE_NAME"
 KY1 = FILLSTRING(21," "), 
 KY2 = FILLSTRING(60," "),
 KY3 = FILLSTRING(40," "),
 DATA = FILLSTRING(879," ")
{FROMDUMMYT
   WHERE1=0
   ORDERBY KY1, KY2, KY3
   WITHORGANIZATION = I}
GO
