The syntax of the Reportwriter clauses in the SELECT command is shown below. The section pertaining to the Reportwriter has been delineated with asterisks.

SELECT
[DISTINCT]
[INTO	output_device]
	select_expression ["template;display_qualifier;format"]{,
	select_expression ["template;display_qualifier;format"]}
[FROM	table_name [alias] {,
	table_name [alias] }]
PLAN	range_name [WHERE qualification]
	(join_operator range_name [WHERE qualification]
[WHERE	qualification]
[GROUP BY expression]
[HAVING	condition]
[ORDER	select_expression [DESC] {,
	select_expression [DESC] }]

/*********** REPORTWRITER SECTION ***************/
[HEAD REPORT
	report_item {,
	report_item }]
[HEAD PAGE
	report_item {,
	report_item }]
{HEAD	group_expression
	report_item   {,
	report_item }}
[DETAIL
	report_item {,
	report_item  }]
{FOOT	group_expression
	report_item {,
	report_item }}
[FOOT PAGE
	report_item {,
	report_item  }]
[FOOT REPORT
	report_item {,
	report_item }]
\********** END REPORTWRITER SECTION ************\
[WITH	control_option {,
	control_option }]
GO

The following concepts help you master the Reportwriter:

  • Any column or group_expression to be displayed using the Reportwriter must first be selected (as a select_expression) in the syntax of the SELECT command. If a column is used only in an expression (such as in the IF( ) statement), it does not need to be selected in the syntax.
  • Use quotation marks to mark text that is printed literally. Use both sets of quotation marks to print quotation marks; for example, ` "PATIENT PROCEDURE REPORT"' is printed like this: "PATIENT PROCEDURE REPORT".
  • Any variables you use in the Reportwriter section should be initialized first in the HEAD REPORT clause.
  • Write and troubleshoot your Discern Explorer query before you add the Reportwriter section.
  • Include line comments (designated by semicolons) before and after the Reportwriter section. The entire Reportwriter section then can be treated as a comment by replacing the semicolons with the slash and asterisk comment markers for troubleshooting the program without the Reportwriter section.
