Note

This topic discusses creating custom ISAM (non-RDMBS) tables. Custom ISAM tables were widely used by clients to create custom tables in Cerner Classic.  This functionality was carried forward intoMillennium Platform.  Discern Explorer versions 8.3.9 and 8.3.10 introduced the ability to create temporary and permanent custom RDBMS tables. In most cases, using a custom RDBMS table is a better option than using a custom ISAM table. See SELECT INTO TABLE Table_Name topic for more information on creating custom RDBMS tables

The Select Into Table table_name command should only be used to create custom tables. It should not be used by Oracle development teams to create Millennium Platform production tables. 

After you create a table, you can insert, update, or delete rows on it. Only indexed and relative files can be modified this way. Before any of the database update commands are used, however, you must issue a DEFINE command with a MODIFY option to allow modification of the table. For example, to open the CATALOG table with the ability to add, change, or delete a row in that table, the following DEFINE command would be used:

DEFINE CATALOG WITH MODIFY GO

This command typically is embedded in the CREATE PROGRAM command. If you use it this way, the word GO can be omitted. The DEFINE command assumes that the data file resides in the CCLUSERDIR directory. If the data file does not exist in that directory, you must also define the location, as shown in the following command:

DEFINE CATALOG IS "PATH1:[USER.XYZ]CATALOG" WITH MODIFY

Always make sure that you have a recent backup of the data file before you perform any inserts, updates, or deletes on the file.

See DEFINE Using Discern Explorer for more information about the DEFINE command.
