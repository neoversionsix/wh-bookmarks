Script files that are retained and may be modified later can be created using an editor. Using an editor is similar to using a word processor. Mistakes can be corrected easily and the program can be saved, eliminating the need to retype the commands each time you want to run them.

To access an editor, enter %EDIT or %E on the command line, followed by a space, then the name of the file you want to create (or open, if you created it earlier). For example, to create or access the source code file named PATNAMES.CCL, enter %E PATNAMES.CCL. Discern Explorer searches for the file in the CCLUSERDIR directory.

By default, Discern Explorer uses the EDT or EMACS editor. If you prefer, you can use an alternate editor (such as the VI, TPU, or the LSE editor) by using the %L directive. For example, to use the VI editor, enter %L VI at the command line. The next time you use the %E directive, Discern Explorer uses the VI editor. You also can access an editor by entering its name (such as EDT) and the filename at the operating system $ prompt. The file is created in the directory from which you accessed the editor (your default directory).

If the system cannot find the file, it creates the file in the CCLUSERDIR directory (or your default directory) and assigns the filename to it. If Discern Explorer finds the file, it displays the contents of the file. You can now edit the file using the editor's various editing commands.

To execute the commands you have saved in a file using an editor, use the %INCLUDE (%I) directive. For example, %I PATNAMES.CCL executes the Discern Explorer commands saved in the PATNAMES.CCL editor file.

The %RUN (%R) directive combines the %E and %I directives. Entering %R PATNAMES.CCL opens the PATNAMES.CCL file in the editor. When you exit the editor,Â Discern Explorer includes or execute the commands saved in the file.
