At a minimum, use the following naming convention for custom programs:

1_Mnemonic_Description

Where Mnemonic is the client mnemonic or short string that identifies the client and Description is a short indication of what the program does. The most important element of this convention is that all client custom program names begin with a number. The standard for production programs written by Oracle is to name the program beginning with a letter. If all client custom program names begin with a number and all standard Oracle program names begin with a letter, a custom program will never over write a standard program or a standard program will never over write a custom program. Using the number 1 followed by your client mnemonic makes it easy to identify all of your custom programs.

Consider adding an element that identifies the author and the solution to your specific program naming convention, for example, 1_Mnemonic_Description_Solution_Initials.

Where Solution could be PHARM for pharmacy reports and RAD for radiology reports and Initials are the initials of the person that wrote or owns the program.

Program names are limited to a total of 30 characters.
