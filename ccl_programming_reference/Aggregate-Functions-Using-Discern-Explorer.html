Aggregate functions are special functions that you can use to calculate values based on groups of elements. They can be used in the FOOT group_expression and FOOT REPORT clauses of the Reportwriter, and in select expressions with some restrictions.

The syntax for using an aggregate function in the FOOT group_expression and FOOT REPORT clauses is displayed below:

{aggregate_function (expression [WHERE qualification])
	["template; display_qualifier, display_format"]}

Valid aggregate functions in the FOOT group_expression and FOOT REPORT clauses are:

Function

Description

AVG

Returns the average of two or more values.

COUNT

Returns the count of an expression.

MAX

Returns the largest value of an expression.

MEDIAN

Returns the middle value of an aggregate list.

MIN

Returns the smallest value of an expression.

PERCENT

Returns the specified percentage value of an aggregate list. The default value is 50%.

STDDEV

Returns the standard deviation of an expression.

SUM

Returns the sum of two or more values.

VARIANCE

Returns the variance of an expression.

The most common aggregate function is the COUNT( ) function. This function enables you to count the number of rows extracted for a particular FOOT clause. You can use a WHERE clause to identify which rows to count. The following example counts the total number of patients and the number of patients with lengths of stay over five days for each nursing unit.

FOOT NURSE_UNIT
	ROW +1
	COL 45 "Total number of patients for this nursing unit: "
	COL 95 COUNT(NAME)
	ROW +1
	COL 45 "Patients with LOS > 5 for this nursing unit: "
	COL 95 COUNT(NAME WHERE LOS >5)

The following example uses the Sum( ) and Avg( ) functions to calculate the total number of days and average number of days that patients have been in the nursing unit.

FOOT NURSE_UNIT
	ROW +1
	COL 45 "Total number of days for this nursing unit: "
	COL 95 SUM(LOS)
	ROW +1
	COL 45 "Average length of stay for this nursing unit: "
	COL 95 AVG(LOS)
