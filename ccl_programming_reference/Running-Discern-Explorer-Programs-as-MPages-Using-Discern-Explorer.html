Discern Explorer (CCL) programs can be set up in Preferences Maintenance Tool (PrefMaint.exe) to run from a customizable organizer or chart tab in PowerChart or any of the PowerChart clones (FirstNet, INet, SurgiNet) through MPages. This is similar to running a PowerChart report; however, it has the advantage of using the display capabilities of DiscernOutputViewer control, which is integrated into MPages. This provides prompt support, more output file-type options, and additional viewing options.

Complete the steps below to add MPages, which displays a Discern Explorer program or a URL as a tab in PowerChart or a PowerChart clone.

  1. Open Preferences Maintenance Tool (PrefMaint.exe). From the Application list, select the solution to use with PowerChart, (FirstNet, INet, SurgiNet). Filter by the position you want and select the Search for Preferences option. From the View menu, select View PVC_Name to filter preferences by the PVC Name.
  2. Expand the solution name tree and the level (Organizer or Chart) from the Level tree control. Click the level you want (Organizer or Chart), and click Add Tab.
  3. From the Available Tabs list, select Discern Report (the component reference name for MPages). Click the down arrow to move Discern Report to the Existing Tabs list. Discern Report is moved to the end of the Existing Tabs list. The placement of Discern Report in the Existing Tabs box defines where the tab is displayed in the solution. To reposition the Discern Report tab in the Existing Tabs list, select Discern Report. Click the up and down buttons located to the right of the Existing Tabs list to position the tab elsewhere in the list.
  4. Click Apply. Click OK to save the new tab assignment. Discern Report is displayed as an option under the level you want (Organizer or Chart).
  5. Select the Discern Report option. A list of preferences is displayed in the Existing Preference box.
  6. To define the title of the tab, enter the title in VIEW_CAPTION in the Preference Name column. Discern Report is the tab's default title.
  7. To define a Filtered TOC View in PowerChart for which this tab should be hidden from, select the code value in the HIDE_IN_TOC_FILTER in the Preference Name column. By default this preference is empty and the tab is not hidden from any of the TOC filters. Multiple TOC filters can be added by adding additional instances of this preference. Note: This preference is only honored if the TOOLBAR_STYLE preference is set to PowerChart Enhanced.
  8. To hide the tab from a display in the Filtered TOC View in PowerChart, specify a code value for the view in the HIDE_IN_TOC_FILTER preference. The component can be hidden from multiple views by adding multiple instances of this preference. This preference only applies when the PowerChart TOOLBAR_STYLE is running with Enhanced Navigation.
  9. In the Level tree, select the Discern Report component (underneath the Discern Report tab item). To define the program that runs and is displayed when you click the Discern Report tab, locate REPORT_NAME in the Preference Name column. In the corresponding Value cell, enter the Discern Explorer program name. The program must have an output device prompt associated with it, and the CCL object for the program must exist in the domain before executing the tab. CCL group names are allowed (for example, eks_monitor:group1).
    If a web page, local file, or other browser compatible content is to be loaded instead of a Discern Explorer program, prefix the link with the special tag <url>. In this case, the URL of the content should be specified in the Value field of the REPORT_NAME. For example, <url>http://www.cerner.com or <url>file://c:/temp/index.html) and the REPORT_PARAM preference is not necessary. The context variables are supported in the REPORT_NAME when using this preference in this manner (for example: <url>http://www.myportal.com?user_id=$usr_personid$ to pass in the user ID as a parameter of a website. See Context Variables Available Using Discern Explorer for more information.
  10. When using a Discern Explorer program in the REPORT NAME, the REPORT_PARAM preference column is optional. Use it to enter the parameter list for the prompts that are passed to the program specified in the REPORT_NAME preference. The list of parameters provided in this preference is comma-separated with text strings in quotations. Patient context data can be passed as a parameter using special variable names encapsulated in the '$' character and is case insensitive; for example, "MINE", CURDATE, CURTIME - 30, CURDATE, CURTIME, "EKS*", "Y", "*", value($PAT_PersonId$). See Context Variables Available Using Discern Explorer for more information. If the program has prompts, you might want the prompt parameters to be specified at run-time, which can be done by leaving this preference blank.
  11. Select the option for the HTML_POPUP_WINDOW_MODE. This is a global-setting to control how the MPages tab handle HTML pop-up windows. To control pop-up windows in an individual HTML page, use the CCLNEWSESSIONWINDOW option instead.
     
    1. 0: This option indicates to use the browser default behavior for pop-up windows. This option does not forward the HTTP session context, such as security login information, to the window. If you need to access the HTTP session in the pop-up window, use option 1 or CCLNEWSESSIONWINDOW. This is the default setting.
       
    2. 1: This option indicates to use the internal pop-up window, so HTTP session context can be accessed in the pop-up.
       
  12. Click Apply, and then click OK. Restart the solution for the following changes to be recognized:
    1. A new tab is displayed in the configured solution as specified by the Preference Maintenance preferences at the given level. When you click the tab, the Discern Explorer program specified for the tab in the REPORT_NAME preference runs with the parameter list specified in the REPORT_PARAM preference and be displayed in the solution.
    2. The output of the Discern Explorer (CCL) program or URL is displayed in the DiscernOutputViewer control and standard toolbar options for report-viewing and navigation is available.
    3. A new menu called Navigation with the same report-viewing and navigation options also is available from the solution's menu.

At the Chart level, a Discern Explorer (CCL) program or web page can also be added as a component of a Chart Summary or Patient Information tab. Complete Steps 7 through 10 to configure the Discern Explorer (CCL) program or web page to be run as a component in the Chart Summary or Patient Information tab.

Discern Explorer (CCL) controls the output displayed on a tab, and the output is called by the tab when it is clicked. This output can be any of the currently supported output file types: ASCII, RTF, PDF, PS, CSV, or HTML. You can use existing reports or create new ones; the only requirement is that the data be selected into MINE. For help in building Discern Explorer (CCL) Programs see Building Discern Explorer (CCL) Programs to Display HTML Output.
