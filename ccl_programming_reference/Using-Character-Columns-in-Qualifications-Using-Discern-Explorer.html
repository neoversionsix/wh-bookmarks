If you qualify against a column that contains character data (instead of numeric data), you must embed in quotation marks the value to be compared to the column value. Character data stored in the system is case-sensitive; "ROB" is not the same as "rob" or "Rob". You also can use a wildcard to identify part of the text. The wildcard indicates that as long as the specified text is found, it does not matter what is in the rest of the column. For example, to include information for people whose last names begin with "ROB" no matter what characters follow the "B", enter the following command:

WHERE P.NAME_LAST_KEY = "ROB*"

Discern Explorer uses three wildcards.

  • The asterisk (*) wildcard can represent multiple characters. If you use the ROB* qualification shown above, Discern Explorer extracts names such as Robb, Robbins, Robertson, Robinson, and so on, as long as the first three letters are "ROB".


  • The question mark (?) wildcard represents a single character. For example, in the command below, Discern Explorer searches for all names that begin with "R", end with "B", and have a single letter between the "R" and "B".

    "WHERE P.NAME_LAST_KEY = "R?B"

    In this case, Discern Explorer extracts names such as RAB, RIB, or ROB. Use the asterisk wildcard after the "B" ("R?B*") to include names such as RABIN, RIBBET, or ROBERTSON.

  • The backslash ( \ ) wildcard begins an exact search. For example, if you want to search for an asterisk or question mark, precede the asterisk or question mark with a backslash. For example, in the command below, Discern Explorer searches for a procedure called *T COUNT.

    WHERE O.ORDER_MNEMONIC = "\*T COUNT"
