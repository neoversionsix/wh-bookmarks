Parentheses can be used override the default order of precedence. For example, 1 + 2 * 3 would evaluate to 7. Because the precedence of * multiply is before the precedence of + add, 1 + 2 * 3 is evaluated by multiplying 2 and 3 then adding 1. So the result is 7. Adding parentheses causes, (1 +2) * 3, to evaluate to 9. In this example 1 is added to 2 and then multiplied by 3, so the final result evaluates to 9.

You might use arithmetic operations to determine how far from the normal high a result value is. To calculate the difference, subtract the result value from the normal high value. The following example performs this arithmetic operation using two columns on the PERFORM_RESULT table to create an expression named DIFFERENCE.

DIFFERENCE = P.NORMAL_HIGH - P.RESULT_VALUE_NUMERIC

This expression could be placed in a SELECT command as shown below:

SELECT	DIFFERENCE = P.RESULT_VALUE_NUMERIC - P.NORMAL_HIGH,
	P.NORMAL_HIGH,
	P.RESULT_VALUE_NUMERIC
FROM	PERFORM_RESULT P
GO
