Note

This topic discusses creating custom ISAM (non-RDMBS) tables. Custom ISAM tables were widely used by clients to create custom tables in Cerner Classic. This functionality was carried forward intoMillennium Platform.  Discern Explorer versions 8.3.9 and 8.3.10 introduced the ability to create temporary and permanent custom RDBMS tables. In most cases, using a custom RDBMS table is a better option than using a custom ISAM table. See SELECT INTO TABLE Table_Name for more information on creating custom RDBMS.

The Select Into Table table_name command should only be used to create custom tables. It should not be used by Oracle development teams to create Millennium Platform production tables.


The DELETE command is used to delete rows from a user-defined file. Only authorized Oracle personnel should use the DELETE command on Oracle production tables.  The syntax of the command is shown in the following example:

DELETE
[FROM table_name [alias] {,table_name [alias]} ]
SET delete_expression = expression
	{,delete_expression = expression}
[PLAN range [WHERE qualification]
	[join_operator range [WHERE qualification]}]
WHERE qualification
WITH delete_option
GO

The following example deletes all rows from the CATALOG file that have a program type of D:

DROP PROGRAM CAT_DELETE GO
CREATE PROGRAM CAT_DELETE

DEFINE CATALOG WITH MODIFY
DELETE FROM CATALOG C
WHERE	C.PGMTYPE = "D"
END
GO

The WHERE clause is required in a delete program to prevent all of the rows in the table from being deleted. To delete all the rows in a table, use the WHERE clause, WHERE C.SEQ = 1.

See DELETE Using Discern Explorer for more information about the DELETE command.
