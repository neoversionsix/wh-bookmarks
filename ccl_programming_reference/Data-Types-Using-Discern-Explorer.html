Data used by Discern Explorer can be stored in several ways. The data type defines the data's characteristics. For instance, datetime columns can be formatted in many ways, code values can display information about the code or its meaning, and character data can be shortened or lengthened using the SUBSTRING( ) function.

The data type of a column can be determined by running TABLEDEF2 or CCLORATABLE on the column's table.

Care must be taken not to confuse data types with display formats.

Note

Numbers are infinite but the storage structure used to store F8 and F4 floating point values is finite. In some cases a value that is as close to the actual value as possible is stored in the finite structure. By default,Discern Explorer displays F8 and F4 values rounded to 2 or 6 digits of precision. The recommended guideline is to use raw F8 or F4 values for all calculations but then round the final number to a reasonable precision before displaying, using the value in a qualification, or converting the F8 or F4 to another data type. For example:

declare f8_var = f8 with protect, noconstant( 1.15 / 0.05) go

call echo(f8_var) go

Echoes 23.000000

But:

declare i4_var = i4 with protect go

set i4_var = cnvtint(f8_var) go

call echo(i4_var) go

Echoes 22

When converting an F8 or F4 value to an integer, CNVTINT() drops the precision and returns the number that is to the left of the decimal point. Using the FORMAT() function to override the default rounding to 6 digits of precision shows that f8_var is actually storing a value that is very close to 23.0 but not exactly 23.0.

call echo(format(f8_var ,"###.################" )) go

Echoes 22.9999999999999964

Using the ROUND() function to round f8_var to a more reasonable precision before using CNVTINT() to convert the F8 to an integer returns 23.

call echo(cnvtint(round(f8_var,10))) go

Returns 23.

Note

The data type may also include an L field modifier to indicate that the Character column in the database has a linked NLS column. This is an internal method for identifying global environments. It does not change the data type of this column. If you run TABLEDEF2 on the PERSON table, LVC100 is displayed on the NAME_LAST_KEY column to indicate that it is a global VC100 column. Use NAME_LAST_KEY to qualify and sort queries. In the International environment, the L modifier indicates to the system to automatically use the NLS column to avoid language issues on character strings.
