Note

This topic discusses creating custom ISAM (non-RDMBS) tables. Custom ISAM tables were widely used by clients to create custom tables in Cerner Classic. This functionality was carried forward intoMillennium Platform.  Discern Explorer versions 8.3.9 and 8.3.10 introduced the ability to create temporary and permanent custom RDBMS tables.  In most cases, using a custom RDBMS table is a better option than using a custom ISAM table. See SELECT INTO TABLE Table_Name for more information on creating custom RDBMS tables.

The Select Into Table table_name command should only be used to create custom tables. It should not be used by Oracle development teams to create Millennium Platform production tables.

The UPDATE command is used to change the data in the records in a file.

The syntax of this command is shown in the following example:

UPDATE
	[FROM table_name [alias] {,table_name [alias]} ]
SET		update_expression = expression
		{,update_expression = expression}
[PLAN range [WHERE qualification]
	{join_operator range [WHERE qualification]}]
WHERE	qualification
GROUP BY	expression
HAVING	condition
WITH	update_option
[GO]

Note

The words FROM and INTO can be used interchangeably in the UPDATE command. The examples in this section use INTO.

The following program illustrates how to update rows that already exist in a table.

DROP PROGRAM CAT_UPDATE GO
CREATE PROGRAM CAT_UPDATE

DEFINE CATALOG WITH MODIFY
UPDATE INTO CATALOG C
SET	C.DESC2 = "Program to test update command"
WHERE	C.OBJECT_NAME1 = "TEST_INS"
END
GO

The WHERE clause qualifies on the rows to be updated. If you do not qualify on the rows, all the rows in the table are updated.

See UPDATE Using Discern Explorer for more information.
