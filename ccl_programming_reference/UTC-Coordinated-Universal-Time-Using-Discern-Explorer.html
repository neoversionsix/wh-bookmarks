Coordinated Universal Time (UTC) is the time at zero degrees longitude. The acronym UTC is a compromise between the English CUT and French TUC acronyms. This longitude is by definition that of a line engraved in a brass plate in the floor of the Old Royal Observatory at Greenwich, England. Purists will correctly state that a difference between UTC and Greenwich Mean Time (GMT) exists. This difference is the result of how time is synchronized with the rotation and tilt of the earth. But for most intents and purposes UTC time can be considered to be equivalent to GMT.

In contrast to local time, UTC does not implement time zones. If I am in London and you are in New York, and we speak on the telephone, we could both record the start time of the call at 6:00 UTC. Whereas if we used local time to record the start time, the call would have started at six o'clock standard time in London compared to one o'clock standard time in New York.

When healthcare organizations have facilities in multiple local time zones, UTC is used to store dates and times as a standard time. The UTC can then be formatted to be displayed in local time.

Note

Generally you do not need to adjust the time to account for different time zones when writing a datetime to the data base. Instead the UTC datetime is written to the data base. When the datetime is selected from the data base it is formatted to display in the local time zone. If needed the DATETIMEZONEFORMAT function is used to format the datetime to a time zone other than the current local time zone.


Below is a summary of Discern Explorer and UTC.

Behavior of Select Statements

By default Discern Explorer select statements format datetime fields to local datetime.

RDB select statements format datetime fields as UTC. The RDB command does not perform any implicit datetime conversions.

Reserved Variables Related to UTC

Reserved variable CURUTC returns 0 (zero) if UTC is disabled and 1 if UTC is enabled.

Reserved variable CURUTCDIFF returns the difference in seconds between UTC and local time if UTC is enabled, 0 (zero) if UTC is disabled.

Reserved variable CURTIMEZONE returns the current local time zone name.

Reserved variable CURTIMEZONEAPP returns the timezone index for the calling application regardless of UTC mode.

Reserved variable CURTIMEZONESYS returns the timezone index for the back-end system regardless of UTC mode.

SYSDATE variable returns the UTC datetime.

SYSTIMESTAMP variable returns the UTC datetime with fractional seconds.

CURDATE variable returns the local date as an Internal System Date.

CURTIME variable returns the local time in hhmm.

CURTIME2 variable returns the local time in hhmmss.

CURTIME3 variable returns the local time in 10-milliseconds.

Behavior of Datetime Functions

CNVTDATETIME, CNVTDATETIME2, CNVTAGEDATETIME, CNVTTIMESTAMP, functions:

  1. Assume a literal string is in local and convert to UTC.
    cnvtdatetime("01-JAN-2003 13:27:50") or cnvtdatetime("01-JAN-2003") converts from local to UTC datetime
  2. Assume a datetime is already in UTC and returns asis.
    cnvtdatetime(p.updt_dt_tm) does not convert because p.updt_dt_tm is already in UTC
  3. Assume a separate date and time is in local and converts to UTC.
    cnvtdatetime(curdate,curtime3) converts from local to UTC datetime
  4. If less than 02-Jan-1800, then this is a special begin date and the datetime is returned unadjusted.
  5. If greater than 02-Jan-2100, then this is a special end date and the datetime is returned unadjusted.

Behavior of Formatting Functions

FORMAT function formats a UTC datetime to the local time zone because the assumption is that all formatted datetimes are to be displayed as local datetimes.

DATETIMEZONEFORMAT function formats a UTC datetime to an indicated time zone.

DATEBIRTHFORMATfunction formats the birth datetime to the birth time zone.

DATEDECEASEDFORMAT function formats the deceased datetime to the deceased time zone.

When a Datetime is Passed as an Argument to the Following Functions

DAY, HOUR, JULIAN, MINUTE, MONTH, WEEKDAY, YEAR, CNVTDATE, CNVTTIME, CNVTMIN, CNVTLOOKAHEAD, CNVTLOOKBEHIND

Discern Explorer first converts the datetime to a local datetime before processing the function.

UTC-Related Functions

CNVTDATETIMEUTC function can be used to change datetime expressions to and from UTC.

CNVTDATETIMEUTC2 function can be used to convert a string that represents a date and a string that represents a time to and from a UTC datetime value.

CNVTDATETIMEUTC3 function can be used to convert a string that represents a date and a time to and from a UTC datetime value.

CNVTDATETIMERDB function can be used to convert datetime expressions to and from UTC. Valid in the Where clause or Group By queries on RDBMS tables.

DATETIMEZONEFORMAT returns the adjusted datetime based on the time zone index and formats it into a meaningful display.

DATETIMECMPUTC function can be used to adjust rdb_date_time columns from local to UTC or UTC to local, and return the number of days between two rdb_date_time columns.

DATETIMEZONEBYINDEX function returns the timezone name for the timezone index.

DATETIMEZONEBYNAME function returns timezone index for the timezone name.

DATETIMEZONE function returns the adjusted datetime based on the timezone index.

DATETIMEZONEUTC function returns the adjusted datetime based on the timezone index.

UTC-Related Format Masks

The FORMAT function displays the timezone with the ZZZ or ZZZZZZZZZZZZZZZZZZZZZZZZZZZZZZ mask.

The ZZZ format mask displays the three character abbreviated time zone.

call echo(format(sysdate,"MMM-dd-yyyy HH:MM:SS ZZZ;3;q"))
MAY-19-2003 12:42:03 Ame

The ZZZZZZZZZZZZZZZZZZZZZZZZZZZZZZ format mask displays the 30-character time zone name.

call echo(format(sysdate,"MMM-dd-yyyy HH:MM:SS ZZZZZZZZZZZZZZZZZZZZZZZZZZZZZZ;3;q"))
MAY-19-2003 12:42:03 America/Chicago
