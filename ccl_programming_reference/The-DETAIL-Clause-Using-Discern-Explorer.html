The DETAIL clause lets Discern Explorer work with report items at their lowest level, which may be one column for a row, or one column from an array of columns for a row. The DETAIL clause enables Discern ExplorerÂ to work through the data one row at a time. The syntax of the DETAIL clause is shown below:

[DETAIL	report_item {,
	report_item }]

Keep these points in mind when using the DETAIL clause:

  • The DETAIL clause is optional. If you are creating a summary report, the summary information can be included as totals in the foot sections of the report.
  • A SELECT command can include only one DETAIL clause.

The following example prints the name, sex display value, and birth date for each row selected by the query.

DETAIL	COL 0 P.NAME_FULL_FORMATTED
	COL 50 SEX_DISP
	COL +2 P.BIRTH_DT_TM
	ROW +1
