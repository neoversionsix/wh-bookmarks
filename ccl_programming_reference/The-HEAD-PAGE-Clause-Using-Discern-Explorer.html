The HEAD PAGE clause enables you to define a page header or heading, as well as perform report items at the beginning of each page. The syntax of the HEAD PAGE clause is shown below.

Syntax

[HEAD PAGE
	report_item {,
	report_item }]

Keep these points in mind when using the HEAD PAGE clause:

  • The HEAD PAGE clause is optional.
  • The SELECT command can include only one HEAD PAGE clause.
  • If the HEAD PAGE clause is used, at least one report item must be used.

Header and Footer refer to information repeated at the top or bottom of the pages of a report. A header or footer does not have to stay the same throughout the report; it can be changed at any point so following pages are printed with the new header or footer definition. Alternating header and footer definitions can be defined for duplex pages so that page numbers on a report are always on the outside edge.

Headers provide a sense of consistency throughout the report and should include certain standard elements such as report name, page number, and report date.

A Column Heading is one or more lines of text that describe the contents of the report by column. The column heading can change in one page, from page to page, or only as report sections change. The example below shows a page header and the associated data under column headings.

Example

Page: 2
Name:	        Sex:	Birth Date:	Room-Bed:
Brown, Joseph	M	09/02/1997	101-A
Jones, Sue	F	07/23/1956	102-A
Smith, William	M	10/10/1973	108-B

Header and column heading are not synonymous. Headers are printed at the top of one or more pages; column headings label columns in the report and can change in a page, or be repeated for several pages.
