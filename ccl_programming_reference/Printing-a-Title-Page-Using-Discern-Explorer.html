Use the BREAK command at the end of the HEAD REPORT clause to place the output of the HEAD REPORT clause on a separate title page. When you force a page break in the HEAD REPORT clause, you must create a flag to prevent Discern Explorer from processing the HEAD PAGE clause twice on the page after the HEAD REPORT. If you eliminate the flag, the second page of the report has a HEAD PAGE for the first page (created by the HEAD REPORT clause), and the HEAD PAGE for the second page. The following example shows how to create and use the flag to prevent the HEAD PAGE clause from executing twice.

HEAD REPORT
	PG_BREAK = "N"	;initialize the flag
	;additional HEAD REPORT commands go here
	BREAK
HEAD PAGE
       IF (PG_BREAK = "Y")
           ;additional HEAD PAGE commands go here
       ENDIF
	PG_BREAK = "Y"

In this example, the flag PG_BREAK is initialized to "N" in the HEAD REPORT clause. The IF( ) statement in the HEAD PAGE clause does not qualify on the first pass through the HEAD PAGE. At the end of the HEAD PAGE, outside of the IF( ) statement, the flag PG_BREAK is set to "Y". PG_BREAK equals "Y" for each subsequent pass through the HEAD PAGE clause and process it once for every page after the HEAD REPORT page.
