To simplify the entry of table names, you can define an alias, or abbreviation, for the table name in the FROM clause. To define an alias, enter the alias after the table name in the FROM clause, as shown below.

FROM	PERSON P,
	ORDERS O,
	ORDER_CATALOG OC

Each time the system encounters the alias where a table name should be, it substitutes the table name. The sample query below selects the encounter identification number, the person identification number, the location code, and the arrival date and time from the ENCOUNTER table using the alias E for the ENCOUNTER table. The system substitutes the table name ENCOUNTER each time it sees E where a table name should be.

SELECT	E.ENCNTR_ID,
	E.PERSON_ID,
	E.LOCATION_CD,
	E.ARRIVE_DT_TM
FROM	ENCOUNTER E
GO

When using aliases, follow these guidelines:

  • Aliases must be unique in each query or program.
  • If an alias is defined, it must be used consistently in the query. For example, do not use PERSON.NAME_FULL_FORMATTED on one line and PR.NAME_FULL_FORMATTED on another.
  • The alias needs to be short. It is the OracleÂ standard that the alias be a logical abbreviation of the table name.
