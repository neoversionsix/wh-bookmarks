The PROMPT clause enables you to define text to be displayed when the program is executed prompting the user for input. The syntax of the PROMPT clause is shown below:

PROMPT	"prompt_string" [= default]{,
	"prompt_string" [= default]}
	[with prompt_name{,prompt_name}]

Note

The previous example is a syntax example. To use prompts, you must create an executable program. See the following example.

Keep these points in mind when using the PROMPT clause:

  • Prompts can be used only in Discern Explorer programs using the CREATE PROGRAM command.
  • The PROMPT clause follows the CREATE PROGRAM command and is placed before the SELECT command.
  • Prompts are separated by commas.
  • The word PROMPT is used only once.
  • The prompts must be defined in your query in the same order as the parameters to which they relate. For example, the first prompt relates to parameter $1 and the second prompt relates to parameter $2.
  • The WITH clause is used to associate the prompt name with the position of the prompt. Currently the $<name> can only be used for prompt or paint programs and not full screen application programs.

You can set up detailed prompts that instruct the user what data needs to be entered to execute the command, as well as what type of data is expected. For example, the following program has two parameters that are two different types of data. The NAME_LAST_KEY column is a character column, which requires the parameter be entered in quotation marks; the ACTIVE_IND column is a numeric column, which requires the parameter be entered as a number. The PROMPT clause illustrates how to set up prompts for these parameters:

PROMPT	"ENTER THE LAST NAME (EMBED IN QUOTES)> ",
	"ENTER ACTIVE INDICATOR (1 FOR ACTIVE, 0 FOR INACTIVE)> "

Notice that the prompt strings are embedded in quotation marks. Discern Explorer prints the prompt exactly as you have typed it. Also notice that the prompts ask for a particular piece of information and then instruct the user on how to enter the information.

The prompts must be defined in the same order in which their corresponding parameters are set up in the program so that Discern Explorer can correctly match the user's response to the prompt with its corresponding parameter. In other words, the prompt for the $1 parameter must be defined first, the prompt for the $2 parameter must be defined second, and so on. When the query is executed, Discern Explorer assigns the input from the first prompt to $1, the input from the second prompt to $2, and so on. If you are using the WITH prompt_name, then the input from the first prompt is assigned to the first prompt_name in the with clause.

If you use a parameter more than once in a Discern Explorer program, it is not necessary to prompt for the value each time it is used. Discern Explorer automatically fills the parameter when a response to the prompt is entered.

When you execute the program, Discern Explorer displays each prompt as you have entered it, each time waiting for a valid value to be entered.

Example

DROP PROGRAM PROMPTS GO
CREATE PROGRAM PROMPTS
PROMPT
          "ENTER PRINTER NAME: " = "MINE",
          "ENTER FILE NAME        : " = "D*"
           WITH PRINTER, FILENAME
SELECT INTO $PRINTER
D.FILE_NAME
FROM DFILE D
WHERE D.FILE_NAME = $FILENAME
END
GO
