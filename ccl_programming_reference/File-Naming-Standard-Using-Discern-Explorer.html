Files that are created and saved using VE (VisualExplorer.exe) should be named using the 1_Mnemonic_Description.VCL convention. Since VE uses the file name as the program name in the DROP PROGRAM and CREATE PROGRAM commands that it generates, naming the file 1_ Mnemonic_Description.VCL ensures that the program naming convention is followed. The .VCL extension is created automatically by VE.

Source code files created using DiscernVisualDeveloper.exe (DVDev) or a text editor that create Discern Explorer programs using the CREATE PROGRAM Program_name command should have the same name as the program they create and have a .PRG extension. For example a source code file that contained the commands:

Drop program 1_mne_custprog go
Create program 1_mne_custprog
â€¦
End Go

should be named 1_mne_custprog.PRG.

Include files that are referenced in source code files using the %I file_name directive should be named with an .inc extension. Any legal file name for the operating system can be used to name the file, but the recommended convention is to use:

1_mnemonic_description.inc
