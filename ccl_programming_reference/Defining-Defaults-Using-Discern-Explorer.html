You can set up default responses for prompts so the user only has to press ENTER to respond to the prompt. This is useful if you find you run the program using certain responses most of the time. This enables you to press ENTER through the prompts and Discern Explorer runs the program using the default responses you have set up. One last advantage is that defining default prompts also defines the data type for the parameter. This allows you to avoid using quotation marks for strings when responding to a prompt for a program because you have used quotation marks to define your default.

Keep these points in mind when using defaults:

  • The default is not displayed on the screen.
  • Enter the default value as part of your PROMPT clause.
  • By defining a default, you are defining the data type of that parameter.

The program below uses a default of MINE for the output device. Unless you are selecting into a file and defining the location of a directory (such as "CER_TEMP:PAT_PO"), the output device does not need to be embedded in quotation marks. The default for the order mnemonic is BUN, which is a character data type. Because the default defines the data type, the user's response to the prompt is embedded in quotes automatically, allowing the user to enter a response without quotation marks.

To execute this program, enter PAT_ORDERS GO at the Discern Explorer command line. Discern Explorer prompts for the Output Device and then waits for you to type a response and press ENTER. If you press ENTER without entering a response, Discern Explorer sends the output to the displayer. Next, Discern Explorer prompts for the order mnemonic. You can enter in a Mnemonic and press ENTER. If you press ENTER without entering in a Mnemonic, Discern Explorer searches for BUNs. You can terminate the program while the prompts are displayed by pressing PF3.
