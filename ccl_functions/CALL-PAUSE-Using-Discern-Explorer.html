The CALL PAUSE command causes a program to wait for the defined number of seconds. This slows the display so users can read messages or watch information placed on the screen before other CALL commands refreshes or clears it. CALL PAUSE should only be used in interactive back end Discern Explorer  Screen Application programs. Using CALL PAUSE in programs that run through servers creates inefficient delays in the program. For interactive Discern Explorer, no limit exists on the number of seconds to pause. For programs that run through servers, the maximum delay is two seconds. 

Syntax

CALL PAUSE (seconds)

Metasymbol

Seconds

The number of seconds to pause the screen.

Examples

The following examples illustrate different ways to use CALL PAUSE:

To pause the display for 10 seconds, enter the following command:

CALL PAUSE (10)

To pause the display for 1.5 minutes, enter the following command:

CALL PAUSE (90)

The following example pauses the display for 333 seconds between the call echo commands:

call echo(build("starting testing   :", curtime))go
call pause(333)go
call echo(build("working again      :",curtime))go
