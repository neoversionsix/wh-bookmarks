The SET ERROR command determines what action a program takes if an error occurs. (An error occurs when a command is not executed. Normally in this situation you are returned to the command line.) The default is OFF, which indicates that no action is taken. The SET ERROR command is only valid outside the context of a SELECT|UPDATE|INSERT|DELETE|MERGE command. The SET ERROR command is similar to the SET WARNING command.

Use GO at the end of the command to make it valid throughout a Discern Explorer session. If you do not use GO, the command is valid only during the execution of the program that called it. It must be embedded within an executable program (between the CREATE PROGRAM and END commands).

Syntax

SET ERROR = error_option [GO]

Metasymbol

error_option

Identifies the action that is taken when an error occurs. The following options are valid:

NameDescription

label_name

Branch to the set of Discern Explorer commands following the Label_Name.

OFF

Do not take any action when an error occurs.

Examples

The following example uses SET ERROR to execute the CALL ECHO command when an error occurs.

DROP PROGRAM CCL_ERROR_BRANCH GO
CREATE PROGRAM CCL_ERROR_BRANCH

SET ERROR = ERROR_LABEL

DECLARE ACTIVE_IND = I4

SET ACTIVE_IND = "A" ;will cause and error

GO TO EXIT_SCRIPT

#ERROR_LABEL
CALL ECHO("ACTIVE_IND IS A NUMERIC DATA TYPE NOT A CHARACTER DATA TYPE")

#EXIT_SCRIPT

END
GO
