The CASE command compares a CASE expression to the OF expressions and performs actions when a match is found. 

At least one case_item is required. If multiple OF expressions are true. Only the case_items for the first true OF expression are executed. If a case_item is not specified for an OF expression, the case_item for the next OF expression is executed. Even if that OF expression does not match the Case (expression).

The CASE command is valid in the Report Writer section of a query or in the contents of a Discern Explorer program.

Discern Explorer versions 8.14.2 and 9.02.2 introduced support for CLASS methods in the CASE command.

Syntax

CASE (expression)
	OF expression: [case_item {,case_item}]
	{OF expression: [case_item {,case_item}}]
	[ELSE case_item {,case_item}]
ENDCASE

Metasymbols

case_item
A command or instruction performed by a CASE statement. Valid CASE items are command and report_item.

Command
A Discern Explorer instruction that can be executed by itself (for example, BROWSE, CREATE TABLE, DROP PROGRAM, and SELECT) or included as part of another instruction (for example, CASE and WHILE).

CASE(Expression)

An expression to compare to the OF expressions.

OF Expression

An expression compared to the CASE(Expression). 

Discern Explorer version 8.10.3 introduced the ability to use an item list in the OF expression clauses. An itemlist is a list of values enclosed in the Value() function. In the OF expression, the itemlist can be a list of values in the Value() function or an itemlist passed in as a $parameter.


An expression is the highest-level metasymbol among constants, field names, variables, and so on. The following expressions are valid:

alias.field_name
array_name [dimension]
array_name [dimension1, dimension2]
array_name [dimension1, dimension2, dimension3]
constant
expression arithmetic_operator expression
-expression
(expression)
function(expression{,expression})
parameter
record_name{->field_name}
table_name.field_name
variable

Note

Square brackets are part of the array_name syntax; they do not indicate an optional part of the syntax.

report_item
A printable data element, calculation, or conditional used in a Report Writer clause. The following items are valid report items:

  • aggregate_expression
  • BREAK command
  • CALL <uar>
  • CALL <sqlfunction>
  • CALL <subroutine>
  • CALL ECHO
  • CALL TEXT
  • CALL TRACE
  • CALL CENTER
  • CALL PRINT
  • CALL DEBUG
  • DECLARE
  • CASE command
  • WHILE command
  • FOR command
  • IF command
  • MACRO command
  • ROW and COL commands
  • select_expression assignment
  • Text
  • variable

Examples

If you are searching for person aliases and want to print your own description of the alias type for each person's record, you might construct the following CASE command:

COL 1 NAME
COL 25
CASE (PA.PERSON_ALIAS_TYPE_CD)
	OF "13766": "Medical Record Number:"
	OF "2093597": "Account Number:"
	OF "13767": "Social Security Number:"
	ELSE ALIAS_DISPLAY
ENDCASE
ROW + 1
COL 50 ALIAS

The example below uses items lists in the OF expression. Discern Explorer version 8.10.3 or higher is required for this example.

drop program ccl_test_itemlist_case go
create program ccl_test_itemlist_case

prompt "type = " = 0,
          "value = " = 0 
with param, val
   declare code_type = i4 with protect,noconstant($val)
   case (code_type)
     of $param:         call echo("case)Found it in $param")
     of value(10,20,30): call echo("case)Found it in (10,20,30)")
     else                call echo("case)Not Found")
   endcase
end
go
execute ccl_test_itemlist_case value(1,2,3), 2 go
execute ccl_test_itemlist_case value(1,2,3), 20 go
execute ccl_test_itemlist_case value(1,2,3), 40 go

The example below echoes AAA first time. Since multiple OF expression "AAA" are used. Only the first OF expression "AAA" is executed.

declare var1 = C3 with noconstant("AAA")
case(var1)
	of "AAA" : call echo(^AAA first time^)
	of "AAA" : call echo(^AAA second time^)
	of "BBB" : call echo(^BBB^)
endcase

The example below echoes Var1 = AAA - OF expression BBB echoed. Since the OF expression "AAA" does not specify a case_item, the next case_item (call echo(concat(^Var1 = ^,var1,^ - OF expression BBB echoed^))) is executed.

declare var1 = C3 with noconstant("AAA")
case(var1)
	of "AAA" :
	of "BBB" : call echo(concat(^Var1 = ^,var1,^ - OF expression BBB echoed^))
	of "CCC" : call echo(concat(^Var1 = ^,var1,^ - Of expression CCC echoed^))
endcase
