The debugger expects that the program that you are debugging was created using a file with the same name as the program and a .PRG or .SUB extension. That file is assumed to be in CCLUSERDIR. For example, when debugging a program named my_program, the source is assumed to be CCLUSERDIR:my_program.prg or CCLUSERDIR:my_program.sub. The SET SOURCE command is used to specify the source code file if the program was created using a file that is not in CCLUSERDIR, has a file name that does not match the program name, or does not use .PRG or .SUB as the extension. When the debugger searches for the source code file, it looks in the default source under the .PRG extension. If it cannot find the file with a .PRG extension, it checks for the .SUB extension. If the program is not in the default source under either extension, use SET SOURCE to specify the source code file.

Syntax

S[ET] S[OURCE] "directory_name:[file_name]"

Metasymbols

directory_name
The name of a directory.

file_name
The name of a file. When combined with the file's location (the node, disk drive, and directory), file_name forms the full file name, or "file specification".

Example

The following example lists how to compile and run a program in the debugger, and identify a source that is not in the default CCLUSERDIR.

  1. At the Discern Explorer command line, set compile for DEBUG mode by entering SET COMPILE DEBUG GO.
  2. Compile and run a program in DEBUG mode. For example, you can issue the following command at the Discern Explorer command line: %I CCLSOURCE:PROGRAM_NAME.PRG.
  3. Turn on the debugger by entering SET TRACE DEBUG GO.
  4. Run the program you compiled in DEBUG mode. For example, PROGRAM_NAME GO.
  5. The Debugger window is displayed, but not the source text because the program is not in the default CCLUSERDIR. To point to the right directory, issue the following command at the debugger command line:
SET SOURCE "CCLSOURCE:PROGRAM_NAME.PRG"

The debugger displays the source text, and you can continue the debugging process.
