Before a command is executed, the VALUE( ) function converts a variable or expression to a literal value before it is substituted for a parameter. By default, variables are passed by the actual name, not the value represented by the variable name. The VALUE( ) function overrides the default and passes the value of the variable or expression.

When VALUE() is used in the EXECUTE command to pass a variable to a program, the variable replaces the $ symbol in the program.

The VALUE( ) function can be used to pass a list of literal values as a single parameter in the EXECUTE command. When the $ symbol for that parameter is used in an equality qualification, it is treated as an IN statement. For example, in the qualification:

Where p.sex_cd = $gender

The VALUE( ) function can be used in any portion of a Discern Explorer program that does not recognize variables (for example, in a WITH clause or a SELECT INTO clause). In addition, it can be used in DELETE, INSERT, SELECT, and UPDATE commands that are called multiple times in the same Discern Explorer program. Ensure that you do not use the VALUE() function in the Plan, Join, Where, or Reportwriter clauses of a SELECT command. When used in a Reportwriter clause of a SELECT command, the variable or expression enclosed in the VALUE() function is converted to a literal value before the SELECT is executed. That value is not changed by subsequent executions of the Reportwriter clauses. If $gender is equal to value(123.0, 124.0, 125.0), the qualification is treated as Where p.sex_cd in (123.0, 124.0, 125.0).

Discern Explorer version 8.12.3 introduced an enhancement that generates a warning if the length of the expression exceeds the maximum declared length, and the SET TRACE WARNING is enabled.  

Syntax

VALUE(expression[,expression])

Metasymbol

expression

A value of any data type.

Examples

The SET SPOOL command expects a literal file name and printer queue name to be passed as parameters. For example: SET SPOOL = "TURN_RPT.DAT" LP01 GO. If a program prompts for the file name and printer queue name, the VALUE() function can be used to convert the $ symbols for the prompts to literal values.

SET SPOOL = VALUE($Filename) VALUE($Outdev) GO

In this example, $req_nums is set equal to "VALUE(1029,1030,1037,1038,1051,1052)" by the EXECUTE command. When $req_nums is used in the where clause is treated as WHERE R.REQUEST_NUMBER IN (1029,1030,1037,1038,1051,1052).

drop program test:dba go
create program test:dba
   prompt "enter the name of an output device" = MINE
	,"enter a request number : " = 0
with outdev, req_nums
select into $outdev
 r.request_number,
r.request_name
from request r
 where r.request_number = $req_nums
 end go
execute test:dba noforms, value(1029,1030,1037,1038,1051,1052) go
