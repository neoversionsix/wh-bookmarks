The SET ACCEPT command is used in Discern Explorer screen application programs to determine how data should be accepted and redisplayed in input fields (CALL ACCEPT boxes). When a CALL ACCEPT input field is redisplayed, the data displayed could be the default option defined for the CALL ACCEPT or the data entered by the user. The SET ACCEPT command is important when you give the user the option to correct data on the screen and need to call the input field again for correction or when you are displaying a screen again for data entry.

If the SET ACCEPT command is not issued, Discern Explorer uses the NOCHANGE option as the default. The first time a CALL ACCEPT input field is displayed it displays the default options from the CALL ACCEPT command.

Use GO at the end of the SET ACCEPT command to make it valid throughout a Discern Explorer session. When GO is not used with the SET ACCEPT command, it must be embedded within an executable program (between the CREATE PROGRAM and END commands). Issuing the SET ACCEPT command from within an executable program makes the command valid within that program and any programs executed from within that program.

Syntax

SET ACCEPT = redisplay_option [GO]

Metasymbol

redisplay_optionDetermines whether the last data should be redisplayed or whether the default option should be used. The following values are valid:

CHANGE

Display the last data typed into this CALL ACCEPT input field.

NOCHANGE

Display the default from the CALL ACCEPT command.

PATCHECK

Turn on the check for pattern-matching characters (for example, * or ?) in CALL ACCEPT input fields. This is the default.

NOPATCHECK

Turn off the check for pattern-matching characters in CALL ACCEPT input fields.

SCROLL

Capture the value of the key pressed. This is equivalent to the S modifier in the CALL ACCEPT command.

NOSCROLL

Do not capture the value of the scroll or function keys pressed.

TIME

Indicate the timeout period in seconds for the following CALL ACCEPT commands. If the user does not respond to the CALL ACCEPT within the timeout interval specified, the default is taken as the response and the next command is executed. The syntax follows: SET ACCEPT = TIME(timeout_interval).

NOTIME

No timeout period is in effect for the CALL ACCEPT commands.

UPPER

Force entries in CALL ACCEPT input fields to uppercase.

NOUPPER

Accept whatever case is used when an entry is made in a CALL ACCEPT field.

VIDEO

Set the default video mode for future accepts. The syntax is: SET ACCEPT = VIDEO (video_option{video_option}). This is equivalent to the CALL VIDEO command.

Examples

The following examples illustrate different ways to use SET ACCEPT:

To redisplay data entered by the user when the CALL ACCEPT input field is redisplayed, enter:

SET ACCEPT = CHANGE

To display (always) the defaults from the CALL ACCEPT command when a CALL ACCEPT input field is redisplayed, enter:

SET ACCEPT = NOCHANGE

To turn off checking for pattern-matching characters entered in a CALL ACCEPT input field, enter:

SET ACCEPT NOPATCHECK ;DON'T DO PATSTRING CHECK FOR ACCEPTS
