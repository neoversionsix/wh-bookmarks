The SET DOC command is similar to the SET HELP command. It identifies the type of documentation that is called when the user presses the DO key at a CALL ACCEPT input field. When possible, the documentation window is displayed to the left or right of the CALL ACCEPT input field so that the user can view both the CALL ACCEPT input field and the documentation. The documentation window overlays the CALL ACCEPT input field if it is too large to be displayed to one side of the field. Use the SET DOC = POS option to override the default location of the window.

The documentation scrolls automatically when you press PF1, PF2, NEXT SCREEN, PREV SCREEN, and the ARROW keys. Press ENTER, HOME, or PF3 to close the documentation window and return the cursor to the CALL ACCEPT input field.

Use GO at the end of the command to make it valid throughout a Discern Explorer session. If you do not use GO, the SET DOC command is valid only during the execution of the program that called it, and it must be embedded within an executable program (between the CREATE PROGRAM and END commands).

Syntax

SET DOC = doc_option [GO]

Metasymbol

doc_option
Determines whether you can press the DO key to display documentation on an input field in a Discern Explorer program. The following values are valid:

Value

Description

OFF

Documentation cannot be displayed by pressing the DO key.

POS

Determines the position of the documentation window on the screen according to the following syntax: POS(begin_row,begin_col,num_row,num_col). The default position is column 1, row 3.

SELECT

When the DO key is pressed, displays the results of a valid Discern Explorer query reading tables from the data dictionary.

Examples

The following examples illustrate different ways to use SET DOC:

The following example uses the SELECT option with the SET DOC command to create a window that displays documentation for the PERSON table.

SET DOC =
SELECT INTO "NL:"
	DOCUMENTATION = T.DEFINITION
FROM	DM_TABLES_DOC T
WHERE	T.TABLE_NAME = "PERSON"
WITH	NOCOUNTER, CHECK

The next example uses the SELECT option with the table DUMMYT to create a window of information that does not come from a table in the data dictionary.

SET DOC = SELECT INTO "NL:"
	DOCUMENTATION = FILLSTRING(80," ")
FROM	DUMMYT T

DETAIL
	ROW 1, COL 1, "NEED TO KNOW WHAT DATE YOU",
	ROW 2, COL 1, "WANT TO START LOOKING FOR.",
	ROW 3, COL 1, "IN THE FOLLOWING FORMAT: 020193",
	ROW 4, COL 1, "WHICH MEANS:  FEB 1, 1993"
WITH	NOCOUNTER,NOFORMFEED,CHECK,NOHEADING,NOFORM
