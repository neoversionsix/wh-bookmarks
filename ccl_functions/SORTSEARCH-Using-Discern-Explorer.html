SORTSEARCH( ) determines if a sort key value exists in the memsort tree. If a memory sort is performed, SORTSEARCH( ) returns one of the following values:

  • 0 if the sort arguments are not found.
  • 1 if the sort arguments are found.
  • -1 if the sort arguments do not match the actual Order clause list.

SORTSEARCH( ) returns 0 if a file or RDBMS sort is performed.

Syntax

SORTSEARCH(sort_argument[{,sort_argument}])

Metasymbols

sort_argument

Value to match each of the items in the Order By clause.

Examples

In the following example, stat is set equal to 1 if a row in the result set exists where the e.module_name is equal to " EKS_ABDOMINAL_XRAY" and the e.active_flag is equal to "A".

select into $outdev
	e.module_name
	, e.active_flag
	, e.updt_dt_tm
from
	eks_module   e
where e.module_name = "EKS_*"  and e.active_flag = "A"

order by
	e.module_name
	, e.active_flag

head report
	stat = sortsearch("EKS_ABDOMINAL_XRAY","A")
	col 10 ^Return of SortSearch("EKS_ABDOMINAL_XRAY","A") = ^
	col +1 stat  ";L"
	row +2
	col 10 "Module Name:"  
	col 40 "Active Flag"
	col 45 "Update Date Time:"
	row +1
detail
	col 10 e.module_name
	col 40 e.active_flag 
	col 45 e.updt_dt_tm
	row +1
with nocounter, memsort
