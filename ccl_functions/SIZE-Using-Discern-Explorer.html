SIZE( ) returns the size of the specified expression. The type of size information returned is determined by the size_option selected. Options 2 and 3 only apply to an attribute on a table. Options 6 to 9 traverse the record structure, which is more expensive to derive than Options 1 to 5, which are quick to derive.

Syntax

SIZE(expression(,size_option))

Metasymbols

expression
An expression.

size_option
A number that determines the type of size information returned for expression. The following options are valid:

Option

Description

1

Converted expression internal storage size (length for a character expression, four-byte binary for an integer, eight-byte binary for a floating-point number).

2

The default display size for attribute on the table.

3

The default internal storage size for an attribute on the table.

4

Display the total size of an array or record structure member (length * dimension).

5

Display the number of elements (positions) in an array or record member.

6

Display the record length of all fields.

7

Display the record length of all fields, plus the internal storage of 8 bytes per varchar and varlist items.

8

Display the number of varchar items in the record.

9

Display the number of varlist items in the record.

10

The number of active and inactive varchar items in record. This is zero for Discern Explorer-owned records, and only is used to track the linked list for server-owned records.

11

Number of active and inactive varlist items in record. This is zero for Discern Explorer-owned records, and only is used to track the linked list for server-owned records.

12

Display the stack size of a process.

default

If no option is specified or an invalid option is specified, option 1 is used by default.

Examples

In the below example, if X = "ABC" then VAL = 3.

VAL = SIZE(X,1)

If the Record Structure list, REC1->CHANGES, holds 10 elements, NUMBER_TO_CHG is set equal to 10.

SET NUMBER_TO_CHG = SIZE(REC1->CHANGES,5)
