The TRACE function returns 0 if the trace is off, 1 if the trace is on, or -1 for an unknown trace.

Syntax

TRACE(trace_option [,opt_pos])

Metasymbols

trace_option

A text expression representing a Discern Explorer trace option. It must be specified in uppercase letters.

opt_pos

The optional position to return the value for multiple value traces. This was introduced in Discern Explorer versions 8.14.0 and 9.02.0.

Opt_PosDescription
0Return 1 if trace is on, and 0 if trace is off (default).
1

Return the value for Position 1 for the following traces:

CLASSVER

DEPRECATED

FLUSH

QUALPREDCHK

QUERYTIMEOUT

RDBARRAYFETCH

RDBARRAYINSERT

RDBSTMTCACHE

RDBTHREADSAFE

REFLOG

TIMEDRIFT

2

Return the value for Position 2 for the RDBEXPAND and DTTMCALC traces. 

The RDBEXPANDGTTDMIN trace sets the second position for RDBEXPAND, and the DTTMCALCFLEX trace sets the second position for DTTMCALC.

Examples

The following example echoes 1 if the Warning trace is active, and 0 if the Warning trace is not active:

CALL ECHO(TRACE("WARNING")) GO

Since "NOTATRACE" is not a valid trace option, the following example echoes -1:

CALL ECHO(TRACE("NOTATRACE")) GO

The following example uses the opt_pos option:

call echo(build("WARNING=",trace("WARNING"))) go
call echo(build("RDBARRAYFETCH=",trace("RDBARRAYFETCH"))) go
call echo(build("RDBARRAYFETCH pos1=",trace("RDBARRAYFETCH",1))) go
call echo(build("DTTMCALC=",trace("DTTMCALC"))) go
call echo(build("DTTMCALC pos1=",trace("DTTMCALC",1))) go
call echo(build("DTTMCALC pos2=",trace("DTTMCALC",2))) go    
call echo(build("RDBEXPAND=",trace("RDBEXPAND"))) go
call echo(build("RDBEXPAND pos1=",trace("RDBEXPAND",1))) go
call echo(build("RDBEXPAND pos2=",trace("RDBEXPAND",2))) go
