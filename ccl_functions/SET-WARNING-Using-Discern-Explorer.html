The SET WARNING command identifies what action a program takes if a warning occurs. (A warning is generated when a command is executed, but something goes wrong, such as when data is truncated or an incompatible option is encountered.) The default is OFF, indicating that no action is taken. The SET WARNING command is similar to the SET ERROR command. The SET WARNING is only valid outside the context of a SELECT|UPDATE|INSERT|DELETE|MERGE command.

Use GO at the end of the command to make it valid throughout a Discern Explorer session. If you do not use GO, the command is valid only during the execution of the program that called it. It must be embedded within an executable program (between the CREATE PROGRAM and END commands).

Syntax

SET WARNING warning_option \[GO\]

Metasymbol

warning_option

Identifies the action to take when a warning occurs. The following options are valid:

Option

Description

label

Branch to the set of Discern Explorer instructions represented by label.

OFF

Do not take any action when a warning occurs.

Examples

In the following example, the SET WARNING command determines that Discern Explorer should execute the instructions represented by the label HOME_LABEL if a warning occurs:

SET WARNING HOME_LABEL
