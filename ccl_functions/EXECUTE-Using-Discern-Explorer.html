The EXECUTE command has two forms: EXECUTE and EXECUTE FROM.

When executing a program, EXECUTE enables you to pass parameters to and run a program. This first form of the command is used to run a Discern Explorer program that is stored in the object library. When EXECUTE is used to execute a stored program, from the interactive Discern Explorer command line, the word EXECUTE is optional. When executing a stored program that contains a Prompt clause, parameters can be used to assign values to the prompts. The parameter values are assigned sequentially to the prompts. In other words, the first parameter is assigned the first prompt, the second parameter is assigned to the second prompt, and so on. If any parameters are omitted, Discern Explorer uses the default value from the Prompt clause. Stored programs can use $n symbols to accept parameters without using the Prompt clause. The $n symbols can be in the range of $1 - $99. When executing a stored program that uses $n symbols, parameters must be used to assign a value to each $n symbol. The number of parameters used must correspond to and match the $n symbols defined in the program. The value of the first parameter is assigned to $1, the value of the second parameter is assigned to $2, and so on. Parameters can be literal strings (for example, LP01 for a printer parameter) or values (for example, V_PRINTER). If you pass the variables as parameters, the VALUE( ) function must be used. The VALUE( ) function instructs Discern Explorer to use the value assigned to the variable, not the literal string of the variable.

When executing a label range within a program, EXECUTE FROM branches to a label and from there executes all of the instructions. Once those instructions have been executed, control passes to the next label in the range specified. When all labels have been executed, control returns to the command following the EXECUTE FROM command, unless the command was called from a WHILE, IF, FOR, or CASE command.

The EXECUTE and EXECUTE FROM commands can include other EXECUTE or EXECUTE FROM commands, up to ten levels deep. You can execute any range or program, and they can execute another range or program up to ten times. You can embed an EXECUTE command in an EXECUTE FROM command.

If a branch is created in a Discern Explorer program by an EXECUTE FROM command, you can embed a GO TO command in the label range to which the program branches. Discern Explorer checks for the GO TO command in an EXECUTE FROM command called from a WHILE, IF, FOR, or CASE command if the SET TRACE ERROR option is turned on.

If Discern Explorer captures a SET HOME, SET ERROR, or SET WARNING value, it unconditionally branches to the indicated instructions and does not return to the EXECUTE FROM instructions. If SET HOME, SET ERROR, or SET WARNING is included in a subprogram, it only applies to the subprogram.

Syntax

Executing a program:

[EXECUTE] object_name [:group_name] [parameter1 {,parameter-n}] [WITH execute_option] GO

Executing a range of labels:

EXECUTE FROM begin_label TO end_label

Metasymbols

begin_label

The name of the label beginning a range of labels to be executed.

end_label

The name of the label ending a range of labels to be executed.

group_name

The identifier of a class of users, usually associated with a specific level of Discern Explorer privileges. Each user is assigned to a category using the CREATE CCLUSER command. Valid groups are DBA and GROUP0 through GROUP99.

object_name

The name of a program stored in the Discern Explorer Object Library using the CREATE PROGRAM command.

parameter

A value passed to a Discern Explorer program either by the user in response to a prompt or at the time of execution, or by using a run-time library routine. Within a stored program, parameters are identified by a dollar sign ($) followed by an integer (for example, $1, $2, and so on) or by a dollar sign ($) followed by a name that is defined in the With statement of the Prompt clause in the Create Program command. Parameters can be created for the following:

  • control_option
  • expression
  • output_device
  • select_expression

execute_option

  • REPLACE(Name_From, Name_To [,Replace_Option])
    Use this option to execute a program and replace a name. The WITH REPLACE option temporarily replaces the name in the Name_From and uses the Name_To instead.  Discern Explorer versions 8.13.2 and 9.01.2 introduced the ability to use a namespace in Name_From and Name_To.

    Name_From: Generally in uppercase and embedded in quotes. It can be a name, string, or namespace.

    Name_To: Generally in uppercase and not embedded in quotes. It can be a name, string, or namespace. Embedding Name_To in quotes can prevent the name from being replaced in an Execute With Replace() command in the called program.

    Before Discern Explorer versions 8.14.2 and 9.02.2 an invalid Name_From and or Name_To cause a warning message. Valid Replace(Name_From,Name_To) options are executed. In Discern Explorer versions 8.14.2 through 8.15.1 or 9.02.2 through 9.03.1, an invalid Name_From and or Name_To causes an error message. Valid Replace(Name_From,Name_To) options are not executed. In Discern Explorer versions 8.15.2, 9.03.2 and later, an invalid Name_From and or Name_To causes a warning message. Valid Replace(Name_From,Name_To) options are executed.


    Replace_Option: Introduced in Discern Explorer version 8.7.1.

    Option

    Description

    0

    Replace name|namespace in the child program (default).

    1

    Replace subroutine name|namespace in the child program.

    2

    Replace record name|namespace, but not the table name in the calling program. This option is introduced in Discern Explorer versions 8.14.2 and 9.02.2.

    3

    Same as Option 0, but propagate replace to grandchild programs. This option is introduced in Discern Explorer versions 8.14.3 and 9.02.3.

    4

    Same as Option 1, but propagate replace to grandchild programs. This option is introduced in Discern Explorer versions 8.14.3 and 9.02.3.

    5

    Same as Option 2, but propagate replace to grandchild programs. This option is introduced in Discern Explorer versions 8.14.3 and 9.02.3.

    6

    Only replace table names. This option is introduced in Discern Explorer versions 8.15.0 and 9.03.0.

    7

    Same as Option 6, but propagate replace to grandchild programs. This option is introduced in Discern Explorer versions 8.15.0 and 9.03.0.

    8

    Replace all substrings in a string. Used for CALL PARSER. This option is introduced in Discern Explorer versions 8.16.0 and 9.04.0.

    9

    Same as Option 8, but propagate replace to grandchild programs. This option is introduced in Discern Explorer versions 8.16.0 and 9.04.0.

  • TIME = <timeout_seconds>
    Use this option to set the maximum execute time in seconds for each query in the executed program.
  • CURNAMESPACE = <namespace_string>
    Use this option to override the current namespace for the execution of the program.

Examples

The following examples illustrate different ways to use EXECUTE:
When EXECUTE is used to execute a stored program from the Discern Explorer command line, the word "EXECUTE" is optional. For example, to run the program TABLEDEF2 from the Discern Explorer command line, both of the commands shown below work:

EXECUTE TABLEDEF2 GO
TABLEDEF2 GO

In the below example (run from the Discern Explorer command line), the parameters passed to the program are literal strings. "FORMS" corresponds to the first parameter (identified as $1 in the program) and "TRAIN*" corresponds to the second parameter ($2).

GETUSERS FORMS, "TRAIN*" GO

The below example (run from the Discern Explorer command line) has no run-time parameters to substitute and is owned by GROUP1. If the user is not a member of GROUP1 that person must either be a DBA or must receive permission (through the GRANT command) for their user group. 

RETALLPAT:GROUP1 GO

In the below, the EXECUTE command is issued from within another program. The parameters are passed as values using the VALUE( ) function. (In the program PAT_AUDIT2, the $1 accepts the printer ID and $2 accepts the selected sort.

EXECUTE PAT_AUDIT2 VALUE(V_PRINTER), VALUE(V_SORT)


The below example is part of a larger program; it uses the EXECUTE command within a program to execute the commands in the range from label LABEL_1 to label LABEL_10. Within a program you must precede a label by a pound sign (for example, #LABEL_1 represents LABEL_1), but you can omit the pound sign from the EXECUTE command. 

EXECUTE FROM LABEL_1 TO LABEL_10
#WRAP_UP
CALL TEXT(24,1,"END PROGRAM?")
CALL ACCEPT(24, 15, "P;CU","Y" WHERE CURACCEPT IN ("Y", "N"))
IF (CURACCEPT = "Y") GO TO END_IT
	ELSE GO TO START_IT
END IF

#LABEL_1
EXECUTE PAT_AUDIT
EXECUTE PAT_AUDIT2
CALL CLEAR(1,1)

#LABEL_10

#END_IT
CALL CLEAR(1,1)

END
GO


The below command executes the program XYZ_GET_LIST while replacing the REQUEST record with ORDREQUEST and the REPLY record with ORDREPLY. The parameters to the REPLACE() option need to be specified in uppercase. 

EXECUTE XYZ_GET_LIST
WITH	REPLACE("REQUEST",ORDREQUEST),
	REPLACE("REPLY",ORDREPLY)
GO


The following command executes the program CCL_EXECUTE_REPLACE_PROG, replacing the strings "REC1" and "SUB1" with REC2 and SUB2.

execute ccl_execute_replace_prog with replace("REC1",REC2), replace("SUB1",SUB2)


The following command executes the program CCL_EXECUTE_REPLACE_PROG, replacing the subroutines named SUB_REC1 but not the record structures named SUB_REC1 or other "SUB_REC1" strings:

execute ccl_execute_replace_prog with replace("SUB_REC1",SUB_REC2,1)

The following example executes the CCL_TEST program, replacing the names SOMERECORD and SUB1 with myRecord and sub2:

execute ccl_test with replace("SOMERECORD", myRecord),replace("SUB1",sub2,1)

The following example executes the CCL_TEST program, replacing the names SOMERECORD and SUB1 with the namespaces n1::myRecord and n1::sub2:

execute ccl_test with replace("SOMERECORD", n1::myRecord),replace("SUB1",n1::sub2,1)

The following example executes the CCL_TEST program, replacing the namespaces n1::SOMERECORD and n1::sub1 with the names myRecord and sub2:

execute ccl_test with replace(n1::SOMERECORD, myRecord),replace(n1::sub1,sub2,1)

The following example executes the CCL_TEST program, replacing the namespaces n1::SOMERECORD and n1::SUB1 with the namespaces n1::myRecord and n1::sub2:

execute ccl_test with replace(n1::SOMERECORD, n1::myRecord),replace(n1::SUB1,n1::sub2,1)
