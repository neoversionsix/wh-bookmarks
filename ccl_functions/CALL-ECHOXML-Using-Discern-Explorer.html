The CALL ECHOXML command echoes the contents of a record structure to the displayer, default log or listing file, or a user-defined filename as an XML stream. Before Discern Explorer versions 8.14.1 and 9.02.1, CALL ECHORECORD accepted a record structure name and echoed the entire contents of the record structure. Discern Explorer versions 8.14.1 and 9.02.1 introduced the ability to pass a record structure member to echo the contents of that member and its children only.

See the CNVTRECTOXML() function for information on returning the contents of a Discern Explorer record structure as a VC data type in XML format. For cases in which you are returning the XML string to MPages or a program to display in the Discern Output Viewer (DiscernOutputViewer.ocx), you most likely want to use the CnvtRecToXML() function to set the _Memory_Reply_String public memory variable over the Call EchoXML command to write the XML string to a file and then reading the file.

Syntax

CALL ECHOXML(record_name[,output_name[,append_mode]])

Metasymbols

record_name
The name of the record structure.

record_member
A reference to a record structure list item, group item, or field item.

output_name
The name of the file where the contents of the record structure are written.

Append_mode
0        Create a new version of the output_name file (default).
1        Append new information to the existing output_name file.

Examples

In the following example the record structure, ORDERS_REC, is loaded and CALL ECHOXML is used to echo out the contents to the listing file or default log file:

DROP PROGRAM RECORDECHO_TEST GO
CREATE PROGRAM RECORDECHO_TEST

RECORD ORDERS_REC(

1 LIST[*]

	2 ORDER_ID = F8
	2 CATALOG_CD = F8

)

SELECT INTO "NL:"
O.ORDER_ID
FROM ORDERS O

HEAD REPORT
	CNT = 0

DETAIL

	CNT = CNT + 1
	IF(MOD(CNT,10) = 1)
		STAT = ALTERLIST(ORDERS_REC->LIST,CNT + 9)
	ENDIF

	ORDERS_REC->LIST[CNT]->ORDER_ID = O.ORDER_ID
	ORDERS_REC->LIST[CNT]->CATALOG_CD = O.CATALOG_CD

FOOT REPORT
	STAT = ALTERLIST(ORDERS_REC->LIST,CNT)

WITH NOCOUNTER,MAXREC = 5

CALL ECHOXML(ORDERS_REC)

END
GO

The below example sends the contents of the record structure, ORDERS_REC, to a file named RECORDRESULTS.DAT in CCLUSERDIR. If the file already exists, a new version of the file is created.

CALL ECHOXML(ORDERS_REC,"RECORDRESULTS")

The below example sends the contents of the record structure, ORDERS_REC, to a file named RECORDRESULTS.DAT in CCLUSERDIR. If the file does not already exist, it is created. If the file already exists, the new information is appended to the existing file.

CALL ECHOXML(ORDERS_REC , "RECORDRESULTS" , 1)

The below examples assume that the program has created and populated the following record structure:

record rec ( 
	1 today = vc 
	1 user_id = f8 
	1 array_item[3] = i4 
	1 plist [*] 
		2 person_id = f8 
		2 name_full_formatted = c30 
		2 olist [*] 
			3 order_id = f8 
			3 order_display = c30 
) with protect

The following example echoes the content of the rec->today field item:

call echoxml(rec->today)

The following example echoes the content of the rec->plist and all of its child items, including olist:

call echoxml(rec->plist)

The following example echoes the content of the the olist that is under the second position of plist:

call echoxml(rec->plist[2].olist)
{"serverDuration": 539, "requestCorrelationId": "ef166f848c13aa5a"}
