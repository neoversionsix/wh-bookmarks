The CALL COMPILE command includes the input file and echoes out errors and output to the trace file.

Lines in the input file exceeding 132 characters are truncated to 132 characters by the compile. The input file is not modified but only the first 132 characters of the line are compiled.

Call Compile sets the Reserved Variable CURQUAL to 1 if the compile is successful or 0 if unsuccessful. If Call Compile generates a CCL-E-7 Unexpected end of file error message, CURQUAL is set to 1.

The operating system (Linux, AIX, HP-UX) ulimit quotas should be considered for user accounts that are granted access to command-line Discern Explorer to compile programs and perform other tasks. This includes executing ORAGEN3, SELECT INTO TABLE Table_Name, CREATE CCLUSER, and other commands which update the Discern Explorer dictionary files. The ulimit file quota is required to have a higher limit than the size of the dic.dat file, which may exceed 1 GB (1,073,741,824 bytes) in size. Review the man ulimit operating system help command, and AIX and Linux resources for additional ulimit documentation.

Syntax

CALL COMPILE(<input_file_name>[,<trace_file_name>][,<echo_linenum_option>])

Metasymbols

Input_file_name
The name of the file to be compiled. The default location is CCLUSERDIR.

Trace_file_name
The name of the trace file. The default location is CCLUSERDIR.

Echo_linenum_option

Option

Description

1

Echoes out line numbers to output_file (default).

2

Does not echo out line numbers.

3

Does not echo out line numbers, and removes blank lines and one-line comments.

Examples

The following example compiles CCLUSERDIR:TAT_REPORT.PRG and stores any errors in CCLUSERDIR:TAT_TRACE.DAT:

CALL COMPILE ("TAT_REPORT.PRG","TAT_TRACE.DAT")
