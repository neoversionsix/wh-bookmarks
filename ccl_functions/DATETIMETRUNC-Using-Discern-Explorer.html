The DATETIMETRUNC( ) function allows you to truncate a specified datetime to the beginning datetime for the following periods:

  • Century
  • Quarter
  • Year
  • Month
  • Week
  • Day
  • Hour
  • Minute.

When Coordinated Universal Time (UTC) is turned on and DATETIMETRUNC() is used either outside of a SELECT command, in the selection list of an outermost SELECT, or in a Reportwriter section of a SELECT, the DATETIMETRUNC() function assumes that a datetime value is in UTC. The system then converts the datetime value to a local datetime before truncating. If UTC is turned on and DATETIMETRUNC() is used in a query in a way that causes it to be processed at the Relational Database Management System (RDBMS) level, the truncation is performed on the UTC datetime value before it is adjusted to the local time zone. Using DATETIMETRUNC() in the Where clause, Order By clause, Group By clause, or in an OLAP expression are some of the cases where the truncation is performed on the UTC datetime value before it is adjusted to the local time zone.

Syntax

DATETIMETRUNC(datetime, component)

Metasymbols

datetime
A datetime value.

component
Indicates the time unit to which to truncate the datetime value.

Component

Description

CC

Truncate to century

Q

Truncate to quarter

YY

Truncate to year

MM

Truncate to month

WW

Truncate to week

DD

Truncate to day

J

Truncate to day

HH

Truncate to hour

MI

Truncate to minute

Examples

The following example echoes 01-JAN-2001 00:00:00.00:

set trunc_date = datetimetrunc(cnvtdatetime("02-Apr-2008 10:55:29"), "CC")
call echo(format(trunc_date,";;q"))

The following example echoes 02-APR-2008 00:00:00.00:

set trunc_date = datetimetrunc(cnvtdatetime("02-Apr-2008 10:55:29"), "dd")
call echo(format(trunc_date,";;q"))

The following example echoes 02-APR-2008 10:55:00.00: 

set trunc_date = datetimetrunc(cnvtdatetime("02-Apr-2008 10:55:29"), "MI")
call echo(format(trunc_date,";;q"))
