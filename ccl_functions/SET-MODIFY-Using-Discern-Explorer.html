The SET MODIFY command allows you to select a modify_option to modify the behavior of Discern Explorer.

Syntax

SET MODIFY modify_option [GO]

Metasymbol

modify_option

Affects the behavior of Discern Explorer. A modify option that is set directly in a back-end session will remain in effect until the session terminates. Modify options set in a program will stay in effect until either the program completes or the session terminates. Good programming practice is to reset the modify option back to the default at the end of the program to prevent the modify option from staying in effect in the parent session. If the program is executed by a server the server is the parent session. The following options are valid:

CNVTAGE([<hours>,]<days>,<weeks>,<months>)

Flex the breakpoints for when the CNVTAGE() function switches to display in hours, days, weeks, months, and years. Default breakpoints are 24 hours, 7 days, 4 weeks, and 24 months. The hours setting is optional. If omitted, the hours breakpoint defaults to 0. If the hours breakpoint is 0 and the difference between the datetime values used by CNVTAGE() is less than 24 hours and the datetime values are from the same day, the age is returned in hours. If the hours breakpoint is 0 and the difference between the datetime values used by CNVTAGE() is less than 24 hours and the datetime values are from different days, the age is returned as 0 Days. To prevent confusion, 24 is the minimum breakpoint that should be used for hours. Use Set Modify CNVTAGE(0,0,0) to reset to the defaults

Override default max size for a variable length string type variable or record structure item. <size> must be an integer data type. Before Discern Explorer Version 8.10.2 the default is 524,288 characters and <size> can be set from 65,536 to 268,435,456 characters. If a program that is executed by a server executes Set Modify MAXVARLEN <size>, subsequent programs executed by that server will retain the new size.
