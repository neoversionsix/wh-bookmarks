The CALL SCROLLUP command controls the way a scroll window reacts when a user presses the UP arrow. This command determines whether Discern Explorer backs up to the previous line in the scroll box, or moves the bottom line of the scroll window out of the box and moves the next available line not in the scroll window to the top of the box.

Note

The CALL SCROLLUP command is only valid in the back end screen application programs that use the Paint command. This type of program was widely used in Cerner Classic, but in general, it is only used for utility programs in Millennium Platform.

Syntax

CALL SCROLLUP (previous_row,current_row,expression)

Metasymbols

current_row
The row that is highlighted as a result of the user pressing the UP arrow.

expression
An expression is the highest-level metasymbol among constants, field names, variables, and so on. The following expressions are valid:

alias.field_name
array_name [dimension]
array_name [dimension1, dimension2]
array_name [dimension1, dimension2, dimension3]
constant
expression arithmetic_operator expression

-expression
(expression)
function(expression{,expression})
parameter
record_name{->field_name}
table_name.field_name
variable

Note

Square brackets are part of the array_name syntax; they do not indicate an optional part of the syntax.

previous_row
The last row that was highlighted before the user pressed the UP arrow.

Examples

The following example would be part of a CASE statement or a complex IF statement that checks for the value of CURSCROLL, a variable used by Discern Explorer to keep track of which keys are pressed when a scroll box is displayed. (See the SELECT command and the CALL ACCEPT command for more information on CURSCROLL).

IF (CNT > 1)
SET CNT = CNT - 1
IF (AROW = 1)
CALL SCROLLUP(AROW,AROW,BUFFER[CNT]) ;scroll top of window
ELSE
SET AROW = AROW - 1
CALL SCROLLUP(AROW+1,AROW,BUFFER[CNT]) ;scroll below top of window
ENDIF
ENDIF

The following example performs the following actions:

  • It backs up a line in the box, usually the next line (the first command).

  • It backs up the last line of information in the box out of the box, moves all lines above that last line down one line, and then displays the next line previously not shown in the box in the first line of the box (the second command).

    CALL SCROLLUP(AROW+1,AROW,BUFFER[CNT])
CALL SCROLLUP(AROW,AROW,BUFFER[CNT])

    Several variables have been defined for this example in earlier parts of the program.

  • CNT is the current array occurrence being displayed in the scroll box.
  • AROW is the current row number in the scroll box.
  • NUMSROW is the maximum number of rows in the window.
  • BUFFER is an array that was filled in by selecting information from a table in the Discern Explorer Data Dictionary using the SELECT command earlier in the executable program.
