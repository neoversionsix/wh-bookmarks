The CALL SCROLLTEXT command displays information on a row in a scroll window in normal video mode. For screen displays, normal mode is bold letters (intense) displayed in the default font and pitch for your terminal. The most common use of this command is to refresh a scroll window with new information once the PF1 or NEXT SCREEN key is pressed.

Note

The CALL SCROLLTEXT command is only valid in the back end screen application programs that use the Paint command. This type of program was widely used in Cerner Classic, but in general, it is only used for utility programs in Millennium Platform.

Syntax

CALL SCROLLTEXT (begin_row,expression)

Metasymbols

begin_row
The beginning row number (from 1 to 24) on the screen where the box, reverse-video box, text, line, or scroll window is displayed.

expression
An expression is the highest-level metasymbol among constants, field names, variables, and so on. The following expressions are valid:

alias.field_name
array_name [dimension]
array_name [dimension1, dimension2]
array_name [dimension1, dimension2, dimension3]
constant
expression arithmetic_operator expression
-expression
(expression)
function(expression{,expression})
parameter
record_name{->field_name}
table_name.field_name
variable

Note

Square brackets are part of the array_name syntax; they do not indicate an optional part of the syntax.

Examples

The following example refreshes the display with new information from an array of data when the PF1 or NEXT SCREEN key is pressed. PF1 and NEXT SCREEN are VT terminal emulation keys.

Several variables have been defined for this example in earlier parts of the program:

  1. CNT is the current array occurrence being displayed in the scroll box.
  2. MAXCNT is the total number of occurrences in the array.
  3. AROW is the current row number in the scroll box.
  4. NUMSROW is the maximum number of rows in the window.
  5. BUFFER is an array that was filled in by selecting information from a table in the Data Dictionary using the SELECT command earlier in the executable program.

The following example would be part of a CASE statement or a complex IF statement that checks for the value of CURSCROLL, a variable used by Discern ExplorerÂ to keep track of which keys are pressed when a scroll box is displayed. See the SELECT command and the CALL ACCEPT command for more information on CURSCROLL.

IF (NUMSROW < MAXCNT)

SET CNT = CNT + NUMSROW - 1
IF (CNT+NUMSROW > MAXCNT) SET CNT = MAXCNT-NUMSROW ENDIF

SET AROW = 1

WHILE (AROW <= NUMSROW)

SET CNT = CNT + 1
CALL SCROLLTEXT(AROW,BUFFER[CNT])
SET AROW = AROW + 1

ENDWHILE

SET AROW = 1

SET CNT = CNT - NUMSROW + 1

ENDIF
