The SET TRACE trace_option command turns on debugging tools when a program executes. Use SET TRACE NOtrace_option to deactivate the trace. The default is OFF. The SET TRACE command can display messages when the program executes, and save messages in the Run-Time Library (RTL) log file.

GO is required at the end of the command when the SET TRACE trace_option command is executed outside of the CREATE PROGRAM command.

Syntax

SET TRACE [=] trace_option [GO]

Metasymbol

trace_option
Defines the type of trace. Add the optional prefix of [NO] to a trace option to deactivate the trace. The following options are valid:

Trace Option

Description

_[NO]TRACE

The _TRACE was introduced in Discern Explorer Version 8.11.2. The _TRACE is reserved for server startup scripts and is not to be embedded in production scripts. When the SET TRACE LOCK is in effect, the only traces allowed are ERRORCLEAR, ERRORCLEARCOM, RECPERSIST. The _ prefix is used to set a trace and this individual trace will be locked and if used in another Discern Explorer program will raise a warning. The SET TRACE LOCKRESET can be issued outside of a program to reset the individual locked traces. This is useful to set the traces in the ccldomain.dat to lock an individual trace without the need to lock all future traces.
Examples:

CCLDEFAULT_0051="NNNN,TRACE(_DEPRECATED,_COST),INSTANCE(1)" 
CCLDEFAULT_0056="NNNN,TRACE(DEPRECATED,COST,LOCK),INSTANCE(1)"

[NO]DTTMCALC [dttmcalc_mode]

This trace option was introduced in Discern Explorer version 8.7.2. It flexes the behavior of the CNVTLOOKAHEAD, CNVTLOOKBEHIND, DATETIMEADD, DATETIMEDIFF, and TIMESTAMPDIFF functions when the mode parameter is not used in the function call. If the mode parameter is specified in the function call, the specified mode is used. If mode is not specified in the function and DTTMCALC is not enabled, the functions default to use Mode 0. If mode is not specified in the function, and DTTMCLAC is enabled and the calculation is greater than one day, the functions default to use Mode 1. Beginning with Discern Explorer versions 8.13.2 and 9.01.2, DTTMCALC is enabled by default.

Discern Explorer versions 8.13.1 and 9.01.1 introduced the dttmcalc_mode option.

dttmcalc_mode 1 (default): If the calculation performed by the CNVTLOOKAHEAD, CNVTLOOKBEHIND, DATETIMEADD, DATETIMEDIFF, or TIMESTAMPDIFF function is less than the DTTMCALCFLEX hours, use the Coordinated Universal Time (UTC) for the calculation; otherwise, convert to the local time zone before performing the calculation.

dttmcalc_mode2: The same as dttmcalc_mode 1, but for BSD extension.

Discern Explorer versions 8.13.1, 8.13.2, 9.01.1 and 9.01.2 may not return the correct date and time when using years or months with the the CNVTLOOKAHEAD or CNVTLOOKBEHIND function and the DTTMCALC trace is enabled. This issue was resolved in Discern Explorer versions 8.13.3 and 9.01.3.

[NO]DTTMCALCFLEX hours
This trace option was introduced in Discern Explorer versions 8.13.1 and 9.01.1 to flex the default number of hours for when the CNVTLOOKAHEAD, CNVTLOOKBEHIND, DATETIMEADD, DATETIMEDIFF, and TIMESTAMPDIFF functions use the local or UTC time zone for calculations. The default number of hours is 24.

Examples

The examples below illustrate different ways to use SET TRACE.

To turn tracing on for errors relating to expressions during a Discern Explorer session, enter the following command:

SET TRACE ERROR GO

To specify a maximum record length for a memory sort, enter the following command:

SET TRACE MEMSORT 2000 GO

To begin displaying warning messages, enter the following command:

SET TRACE WARNING GO

To stop displaying warning messages, enter the following command:

SET TRACE NOWARNING GO

The following example creates a simple program, turns on the RDBDEBUG and RDBBIND traces and then executes the program:

DROP PROGRAM CCL_TRACE_DEBUG_BIND_EXAM GO
CREATE PROGRAM CCL_TRACE_DEBUG_BIND_EXAM

SELECT INTO "NL:"
	P.PERSON_ID
	,P.UPDT_DT_TM
	,P.NAME_LAST_KEY
FROM PERSON P
WHERE P.UPDT_DT_TM BETWEEN CNVTDATETIME("03-JUN-2009")
			AND CNVTDATETIME("04-JUN-2009")
		AND P.NAME_LAST_KEY = "TEST*"
WITH NOCOUNTER


END
GO

SET TRACE RDBDEBUG GO
SET TRACE RDBBIND GO
CCL_TRACE_DEBUG_BIND_EXAM GO
