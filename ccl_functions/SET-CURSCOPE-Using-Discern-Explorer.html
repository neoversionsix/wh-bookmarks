The SET CURSCOPE command defines the scope of variables, record structures, or subroutines.

Syntax

SET CURSCOPE scope [GO]

Metasymbol

scope

Value

Description

PUBLIC

Public is the default. The variable, record structure, or subroutine will be known to all child processes.

PRIVATE

The private scope limits the scope to this level only. The variable, record structure, or subroutine will not be known to any child processes.

PROTECT

Use the protect scope to declare a variable or record structure in a child process with the same name as a variable or record structure that is defined in a parent process. All subsequent children will know the child variable or record structure. When control returns to the parent, the child variable or record structure will no longer be known and the parent variable or record structure is restored.

Examples

The following example sets VAR1 and VAR2 as private variables and VAR3 as public. The program, SCOPE_EX1, is run and sets VAR1 and VAR3. The CALL ECHO( ) displays "VAR1=PG1" for VAR1 and "VAR3=PG2" for VAR3. Because VAR3 is public, the value that is set in the program SCOPE_EX1 is recognized when coming back to SCOPE_EX.

DROP PROGRAM SCOPE_EX GO
CREATE PROGRAM SCOPE_EX

SET CURSCOPE PRIVATE
	DECLARE VAR1 = C10
	DECLARE VAR2 = C2
SET CURSCOPE PUBLIC
	DECLARE VAR3 = C10

SET VAR1 = "VAR1=PG1"
SET VAR3 = "VAR3=PG1"

EXECUTE SCOPE_EX1

CALL ECHO(VAR1)
CALL ECHO(VAR3)

END
GO
