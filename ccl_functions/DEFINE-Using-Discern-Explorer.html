The DEFINE command overrides the default directory and file name for a database. Define can be used to associate a Discern Explorer table definition with an ASCII text file. It affects any future Discern Explorer command that implicitly or explicitly references the database name in the current session. The file name must be enclosed in quotes and follow the operating system's file-naming conventions. The extension .DAT is assumed, but the actual extension can be specified if different (for example, DEFINE MYTABLE IS "CCLUSERDIR:MYTABLE.SAV" GO).

If a database_name is already associated with a file, you must use the FREE DEFINE command before using the DEFINE command to override the previous file name or options. Before you can issue a DELETE, INSERT, or UPDATE command to modify a non-RDBMS table or user-owned file, you must issue a DEFINE command with the MODIFY option to allow modification of the file. Only authorized Oracle personnel should modify Oracle production tables or files.

Syntax

DEFINE database_name [IS] ["file_name"]
[WITH define_option]
GO

Metasymbols

database_name
The name of the database file that Discern Explorer reads from the Discern Explorer Data Dictionary.

file_name
The name of a file. The file extension .DAT is assumed if an extension is not specified. When combined with the file's location (node, disk drive, and directory) file_name forms the full file name or file specification. If an extension is specified, and a file with a name that is the same as the specified file name without the extension exists, the file without the extension is used. Prior version 8.9.4 of Discern Explorer the full file name is limited to 31 characters.

define_option
Identifies how a file is opened. The following options are valid:

Option

Description

MODIFY

Opens the database with read and write access.

NOMODIFY

Opens the database with read access.

WORK

The database is a working-storage type; it is stored in memory.

NOWORK

Looks for the database in the directory CCLDIR, not in working storage (for test purposes).

Examples

The following examples illustrate different ways to use DEFINE:

To select all rows and columns from the PERS_TEMP in directory E, enter the following:

FREE DEFINE PERS_TEMP GO
DEFINE PERS_TEMP IS "E:PERS_TEMP" GO
SELECT * FROM PERS_TEMP GO

To INSERT a row into the user-defined RMS file above:

FREE DEFINE PERS_TEMP GO
DEFINE PERS_TEMP IS "E:PERS_TEMP"  WITH MODIFY GO
INSERT INTO PERS_TEMP P
	SET P.NAME_LAST = "SMITH" GO

To select each line from a sequential file in the CCLUSERDIR: directory named RTL0175.CCL:

FREE DEFINE RTL GO
DEFINE RTL IS "CCLUSERDIR:RTL0175.CCL" GO
SELECT R.* FROM RTLT R GO

Either of the below examples are used to select each line from a sequential file in the CCLUSERDIR: directory named myfile.dat:

free define rtl go
define rtl is "myfile" go  ;.dat is assumed if not specified
select r.* from rtlt r go
free define rtl go
define rtl is "myfile.dat" go
select r.* from rtlt r go

If a file named myfile and a file named myfile.dat exist in the CCLUSERDIR: directory, the below example uses myfile. It does not use myfile.dat:

free define rtl go
define rtl is "myfile.dat" go ;if both myfile and myfile.dat exist, myfile is used
select r.* from rtlt r go


RTL is a predefined database that contains the RTLT table definition. The RTLT table contains one 132-character field named LINE. The RTL definition can be used to read files that have a line length of 132 characters or less.

RTL2 is a predefined database that contains the RTL2T table definition. The RTL2T table contains one 2000-character field named LINE. The RTL2 definition can be used to read files that have a line length of 2000 characters or less.

RTL3 is a predefined database that contains the RTL3T table definition. The RTL3T table contains one 32,768-character field named LINE. The RTL3 definition can be used to read files that have a line length of 32,768 characters or less.

Note

These definitions remain in effect until the Discern Explorer session is terminated, or until the definitions are redefined or are removed using the FREE DEFINE command.

Prior version 8.9.4 of Discern Explorer there is a 31-character limit on the length of a file name. The following example demonstrates using the SET LOGICAL command to create a logical that points to a file name that exceeds the 31 character limit and uses the logical in the define command.

set logical xyz "cer_install:bb_rdm_import_isbt_attributes.csv" go

define rtl2 is "xyz" go
