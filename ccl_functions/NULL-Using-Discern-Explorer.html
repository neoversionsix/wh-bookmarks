NULL () is only valid in a Report Writer clause of a SELECT that is using an ORJOIN, or the OUTERJOIN or DONTCARE control options. It returns 0 if the field contains a valid value, 1 if the field is NULL or undefined. The DECODE() function is preferred, and it should be used instead of NULL().

Syntax

NULL(field_name)

Metasymbols

Field_name
A field from a table used in the ORJOIN, OUTERJOIN, or DONTCARE control options.

Examples

In the following example, ISNULL is equal to 1 if there is no matching row on the ENCOUNTER table.

SELECT INTO $OUTDEV
	P.PERSON_ID
	, E.ENCNTR_ID
FROM
	PERSON   P
	, ENCOUNTER   E
	, DUMMYT D
PLAN P WHERE P.UPDT_DT_TM > CNVTDATETIME(CURDATE-7,0)
JOIN D
JOIN E WHERE E.PERSON_ID = P.PERSON_ID

DETAIL
	COL 0 P.PERSON_ID
	COL +1 E.ENCNTR_ID
	ISNULL = NULL(E.ENCNTR_ID)
	COL +1 ISNULL
	ROW +1
WITH OUTERJOIN = D
