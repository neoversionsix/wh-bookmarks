The COST( ) function allows you to view the cost of a program in terms of system resources. The following example shows the return of the COST( ) function and explanations of its components.

72704:112151 CCLHELP Cost 10.97 Cpu 3.14 Ela 46 Bio 1041 Dio 203 S1,44 I0,0 U0,0 D0,0 O0M0R0 P2 Q13285 R13408

Component

Definition

72704

Julian date.

112151

Julian time.

CCLHELP

Program name.

Cost

Rates program efficiency.

Cpu

Discern Explorer processing time in seconds.

Ela

Total elapsed time in seconds.

Bio

Buffed system reads. Removed in Discern Explorer version 8.11.5.

Dio

Direct system reads.

S

Number of Discern Explorer Selects. Number of Relational Database Management System (RDBMS) Selects.

I

Number of Discern Explorer Inserts. Number of RDBMS Inserts.

U

Number of Discern Explorer Updates.  Number of RDBMS Updates.

D

Number of Discern Explorer Deletes. Number of RDBMS Deletes.

O

Number of operating system sorts

M

Number of memory sorts

R

Number of RDBMS sorts

P

Number of programs run

Q

Number of qualifying rows

R

Number of rows in Discern Explorer

Syntax

COST(option)

Metasymbol

option
An expression that is an integer.

Option

Description

1

This option returns the last cost message as 132-byte string.

2

This option starts the cost.

3

This option ends the cost and returns the last cost message as 132-byte string.

Examples

The following example returns the cost of the last program run:

CALL ECHO(COST(1))
