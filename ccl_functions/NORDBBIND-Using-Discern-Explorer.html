Beginning with Discern Explorer version 8.10.2, NORRDBBIND() is ignored. Before Discern Explorer version 8.10.2, NORDBBIND( ) was used to override the default behavior of bind parameters in the queries sent to the RDBMS based on the following rules:

  • If optimizer mode is RBO, NORDBBIND() is ignored and bind variables are used in the query that is sent to the RDBMS.
  • If optimizer mode is CBO and any non-RDBMS tables are used in the query, NORDBBIND() is ignored and bind variables are used in the query that is sent to the RDBMS.
  • If optimizer mode is CBO and only RDBMS tables are used in the query, a constant is used instead of a bind variable in the query that is sent to the RDBMS.

In environments using CBO, literals used in qualifications are passed as a constant in the query that is sent to the RDBMS by default.

NORDBBIND( ) was introduced in version 8.7.0 of Discern Explorer.

Syntax

RDBBIND(expr)

Metasymbol

expr

A value of any data type.

Examples

select p.person_id
	,p.updt_dt_tm
from person p
where p.person_id = id_var
with format, separator = " "

Executing the above example in a CBO environment causes the following query to be passed to the RDBMS:

 SELECT P.PERSON_ID, P.UPDT_DT_TM FROM  PERSON P  WHERE P.PERSON_ID = :1

The following adds the NORDBBIND() function:

select p.person_id
	,p.updt_dt_tm
from person p
where p.person_id = NORDBBIND(id_var)
with format, separator = " "

If id_var is equal to 12345.0, executing the above example in a CBO environment causes the following query to be passed to the RDBMS:

 SELECT P.PERSON_ID, P.UPDT_DT_TM FROM  PERSON P  WHERE P.PERSON_ID = 12345.0
