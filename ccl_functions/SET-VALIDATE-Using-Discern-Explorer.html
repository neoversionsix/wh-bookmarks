The SET VALIDATE command validates user input in a CALL ACCEPT input field; it ensures that no invalid data is entered in an input field. The command should be issued before the CALL ACCEPT. The default for the SET VALIDATE is 0.

Use the SELECT option with the SET VALIDATE command to read any table and validate the input against values in that table. If you use the SELECT option, you must follow it with another SET VALIDATE using the qual_num option. The qualifying number identifies how Discern Explorer validates the data. If a qualifying number is not specified following a SET VALIDATE with SELECT, the default is zero. Always follow a SET VALIDATE = SELECT with a SET VALIDATE = qual_num to ensure that validation occurs properly.

Use GO at the end of the command to make it valid throughout a Discern Explorer session. If you do not use GO, the SET VALIDATE command is valid only during the execution of the program that called it and must be embedded in an executable program (between the CREATE PROGRAM and END commands).

Syntax

SET VALIDATE = validate_option [GO]

Metasymbol

validate_option

Determines the type of validation that should be performed on the accepted data. The following values are valid:

Value

Description

qual_num

A qualifying number. When the SELECT option is used with the SET VALIDATE command, a second SET VALIDATE should be issued to identify what action to take when input is entered for a CALL ACCEPT command. The following options are valid:
0 - Validate the data if no matches are found on the table. This effectively means that any value can be entered; no validation is performed. The zero option typically is used to perform validation with the INSERT command. If you are adding a record to a file, this option allows you to add a record if it does not exist. This is the default.

  • 1: Validate the data when only one record on the table matches the entered value.
  • 2: Validate the data if it occurs one or more times on the table. This option typically is used more often than 0 or 1 to ensure that validation is performed against any file.

OFF

Turns validation off.

OPTIONAL

The user can press RETURN through the prompt.

REQUIRED

The user must respond to the prompt with a valid value.

SELECT

Any valid Discern Explorer query reading tables from the data dictionary.

Examples

The example below illustrates the commands that would be used in an executable program to validate a user's response in a CALL ACCEPT input field. The CALL ACCEPT only accepts a valid encounter type. If the user enters a value that does not match a value in the CDF_MEANING column on the CODE_VALUE table where the CODE_SET column is also equal to 69, the input is not accepted. Discern Explorer will return an Invalid Input message. The CALL ACCEPT command becomes active and the program waits for the user to input a valid value. The CALL TEXT command displays the input value at the end of the example.

DROP PROGRAM SCR_APP GO
CREATE PROGRAM SCR_APP

PAINT
CALL TEXT(5,5,"ENTER ENCOUNTER TYPE: ")

SET VALIDATE = SELECT INTO "NL:"
	CV.CDF_MEANING
FROM 	CODE_VALUE CV
WHERE	CV.CDF_MEANING = CURACCEPT AND
	CV.CODE_SET = 69

SET VALIDATE = 2
CALL ACCEPT(5,28,"A(30);CU")
SET V_ENCNTR_TYPE = CURACCEPT
SET VALIDATE OFF
CALL TEXT(7,5,V_ENCNTR_TYPE)

END
GO
