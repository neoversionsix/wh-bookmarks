CNVTDATE( ) converts an integer description of a date or an rdb_dt_tm to a Oracle Internal System Date.

By default, CNVTDATE() returns the internal system date as an integer. CNVTDATE() returns an F8 value when it is used to set a variable or record structure item that is defined as an F8. CNVTDATE() returns a DQ8 value when it is used to set a record structure item that is defined as a DQ8. 

Before Discern Explorer versions 8.14.1 and 9.02.1, a variable that is declared as a DQ8 is implicitly cast to an F8, and CNVTDATE() returns an F8 value when it is used to set a DQ8 variable. Beginning with Discern Explorer versions 8.14.1 and 9.02.1, a variable that is declared as a DQ8 is cast as a DQ8. In Discern Explorer versions 8.14.1 and 9.02.1, CNVTDATE() returns 0.0 when setting a variable that is declared as a DQ8. Beginning with Discern Explorer versions 8.14.2 and 9.02.2, CNVTDATE() returns a F8 value when setting a variable that is declared as a DQ8.

Syntax

CNVTDATE(integer_date[,ADATE])

CNVTDATE(rdb_dt_tm)

Metasymbols

integer_date 

An integer description of a date.

If ADATE is omitted, the integer_date is expected as MMDDYY or MMDDYYYY.
If ADATE is specified and the ADATE logical is A, integer_date is expected as MMDDYY or MMDDYYYY.
If ADATE is specified and the ADATE logical is C, integer_date is expected as YYMMDD or YYYYMMDD.
If ADATE is specified and the ADATE logical is E, integer_date is expected as DDMMYY or DDMMYYYY.

rdb_dt_tm

A column that stores the date and time.

ADATE

Only use ADATE as the second parameter. ADATE is a logical that provides a simple method to flex the six-digit date (mmddyy) between American, Canadian, and European formats. See ADATE Using Discern Explorer for more information about the ADATE logical.

Examples

The below examples illustrate the use of the CNVTDATE( ) function.

In the following example, DATE_1 is equal to 72350:

 DATE_1 = CNVTDATE(01301998)

In the following example, persons with birthdays greater than or equal to January 30, 1998 qualify:

WHERE P.BIRTH_DT_TM >= CNVTDATETIME(CNVTDATE(013098),0)

See CNVTDATETIME Using Discern Explorer for more information about the CNVTDATETIME( ) function.

In the following example, if ORDER_DT_TM is 09-JAN-1998 09:05:36, DATE_2 is 72329.

DATE_2 = CNVTDATE(O.ORDER_DT_TM)

In the following example, DATE_VAR should be set equal to 77904.

case(logical("ADATE")) 
	of "A" : set date_var = cnvtdate(04152013,adate)
	of "C" : set date_var = cnvtdate(20130415,adate)
	of "E" : set date_var = cnvtdate(15042013,adate)
	else set date_var = cnvtdate(041520123,adate)
endcase
