The SET LOGICAL command creates a logical that you can use during a Discern Explorer session or in a Discern Explorer program.

Enter GO at the end of the command to make it valid throughout a Discern Explorer session. If you do not use GO, the SET LOGICAL command is valid only during the execution of the program that called it and must be embedded in an executable program (between the CREATE PROGRAM and END commands).

Beginning with Discern Explorer versions 8.15.0 and 9.03.0 setting the logical to two quote marks with nothing between them (""), unsets the logical. Before Discern Explorer versions 8.15.0 and 9.03.0, the logical is set to an empty string.

Syntax

SET LOGICAL [=] logical string [GO]

Metasymbols

logical
The logical name to which you want to assign the string. The logical name cannot exceed 31 characters, and cannot contain spaces or special characters such as commas or semicolons.

string
The string to assign to the logical. Normally, the string is the node, disk drive, directory, subdirectory, and file name to which the logical name is to be assigned. Before Discern Explorer versions 8.14.3 and 9.02.3, embedded spaces are not allowed in the string. Beginning with Discern Explorer versions 8.14.3 and 9.02.3, embedded spaces are allowed in the string.

Examples

The following example creates a logical that is valid during the current Discern Explorer session:

SET LOGICAL = HOMEDIR "home/user/sam" GO

The following example creates a logical named WORKDIR using a string that is first stored in a variable. The output of the CALL ECHO() command is logical: [/cerner/test/folder/not_production/].

DECLARE Log_Str = VC 
SET Log_Str = '/cerner/test/folder/not_production/' 
SET LOGICAL = WORKDIR Log_Str 
CALL ECHO(BUILD("logical: [", LOGICAL("WORKDIR"), "]"))

The following example requires Discern Explorer version 8.14.3, 9.02.3, or later.

SET LOGICAL HAS_A_SPACE "logical for 8143" go
CALL ECHO(BUILD("logical: [", LOGICAL("HAS_A_SPACE"), "]")) go
