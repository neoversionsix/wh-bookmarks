Use the CALL TRACE command to display debug information in the listing or log file. Use CALL TRACE commands only for debugging and testing. Remove CALL TRACE once testing is complete. To use CALL TRACE, the SET TRACE COST or SET TRACE CALLECHO option must be turned on.

Syntax

CALL TRACE(trace_option_num)

Metasymbol

trace_option_num

One of the following option numbers:

Trace Option NumberDescription

1

Set ccloutput to stdout. Only applies to back-end interactive sessions. ccloutput is output to the screen.

2

Set ccloutput to ccllogfile. Only applies to back-end interactive sessions. ccllogfile is output to the session log file. Entering %E without a file name opens the log file for the session.

3

Show active subroutines without tree.

4

Show active subroutines with tree.

5

Show environment.

6

Pool check.

7

Show cache.

8

Set ccloutput and ccllogfile to stdout. Only applies to back-end interactive sessions.

9

Debug a script break routine.

10

Pool check with print detail memory info.

11

Get thread stack size.

12

Forced logfile flush.

13

Pool check with print summary memory info.

14

Print current symbol table.

15

Check all active record structures and check all memory objects.

16

Display the current count of an object type.

17

Display all active label names and levels.

18

Same as trace_option_num 5 (show env) but excludes the show of uar declarations.

19

Show symbol information for call trace(19,<symbol_num>). Use call trace(14) to get symbol_num.

Show env with trace and settings only.

42
Get date sync milliseconds. This functionality was introduced in Discern Explorer version 8.12.1.
43
Show current system memory data, stack and used pages. This functionality was introduced in Discern Explorer version 8.12.1.
44

Print all memory information. This functionality was introduced in Discern Explorer version 8.12.1.

CCL sessions apply limits on the maximum file size and process memory that can be allocated for the process.  This applies to all Millennium 2018.01+ releases, or higher 2015.01 releases (CCL Language v8.12.6|9.00.0 JUL2017 and later).  A program attempting to process a large amount of data and creating a file larger than 4GB for example will terminate abnormally with 'File size limit exceeded' or a similar error message.  The file limit applies to SELECT INTO <file>, or when temporary sort files are created during query processing ($CCLSCRATCH/cclt*.dat files).

These limits are NOT applied to the root account.  Executing the same program through a server will not encounter the limits.

Executing the CALL TRACE(44) command in a CCL session will display the process limits being applied (FSIZE:4294967294 = 4 GB).

45
Print all SET MODIFY options. This functionality was introduced in Discern Explorer version 8.12.6.
46
Print dictionary segments size.

Examples

The following example demonstrates using call trace(15) to check all active record structures and check all memory objects.

drop program  test_call_trace go
create program  test_call_trace
call trace(15)
record  rec (
  1 list [*]
    2 id = f8
    2 name = vc
)
call trace(15)
end go

test_call_trace go

The following example uses call trace(5).

drop program  test_call_trace5 go
create program  test_call_trace5

     declare v1 = vc
      declare sub1(p1=vc) = vc
      set v1 = sub1(build(123456789012.0,":"))
      call echo(build("size=",size(v1),",var=",v1))
      subroutine sub1(p_var1)
        call trace(5)

        return(build(p_var1,"test"))
      end
end go

test_call_trace5 go
