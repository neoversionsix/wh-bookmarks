The SET BREAK command sets a break point on a command, row, label, or report. A break point is a place where you can suspend the execution of a program being debugged and regain control. While execution is paused, you can issue debugger commands at the command line (for example, EVALUATE or DEPOSIT a variable). To continue executing the program, enter the GO command at the debugger command line.

All EXECUTE commands use a debugger break point automatically (including executing a subprogram in a main program). Use the SET BREAK command to set additional break points. You can set a maximum of five break points. To cancel a break point, use the CANCEL command.

Note

You cannot set a break point in a SELECT command unless it contains a Reportwriter section.

Syntax

S[ET] B[REAK] break_option

Metasymbol

break_option
Determines where a break point occurs. The following options are valid:

OptionDescription

Command

The command (for example, SELECT) at which a break point should be set.

Label

The name of a label at which a break point should be set.

REPORT

Sets a break at the start of the REPORT.

Row

The number of a row in the Reportwriter section at which a break point should be set.

WHERE

Sets a break point at the specified qualification or condition. The full syntax of this option is WHERE qualification.

Examples

The following example sets a break at the first Report statement of the Reportwriter:

SET BREAK REPORT

The following example sets a break at the SELECT command:

SET BREAK SELECT

The following command sets a break at the tenth line in a Discern Explorer program:

SET BREAK 10

The following command sets a break at the specified label:

SET BREAK #CORRECT_YN

The following command sets a break point in a FOR loop when the loop variable X reaches the value of 10:

SET BREAK WHERE X = 10
