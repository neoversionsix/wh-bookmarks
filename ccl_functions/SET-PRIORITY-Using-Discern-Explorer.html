The SET PRIORITY command changes the priority of the Discern Explorer session. You can change your priority lower than your default priority only when you enter the session. Your process priority cannot be higher than the base priority assigned to your sign-on. If your base priority is 3, you will enter Discern Explorer with a process priority of 3.

You can adjust the priority one number at a time either up or down. To lower your priority by two numbers, you must execute the command twice.

With the SET PRIORITY command, you can lower the process priority so that the Discern Explorer program does not use CPU time before more important programs that might be running at the same time. This is particularly useful if you are running programs during peak usage times or if you are running a program during nightly operations but do not want it to interfere with other operations programs.

Use GO at the end of the command to make the priority valid throughout a Discern Explorer session. If you do not use GO, the SET PRIORITY command is valid only during the execution of the program that called it and must be embedded in an executable program (between the CREATE PROGRAM and END commands).

Syntax

SET PRIORITY = priority [GO]

Metasymbols

priority

Increase or decrease the priority from its current value by one. The following values are valid:

Value

Description

BATCH

Equivalent to level 3.

DOWN

Decrease priority by one.

HIGH

The highest priority that can be set using the UP option.

INTERACTIVE

Equivalent to level 4; same as the HIGH option.

LOW

The lowest priority that can be set using the DOWN option.

SHOW

Display the current priority setting.

UP

Increase priority by one.

Examples

The following example lowers the priority from 4 to 3 for the current Discern ExplorerÂ session:

SET PRIORITY = DOWN GO
