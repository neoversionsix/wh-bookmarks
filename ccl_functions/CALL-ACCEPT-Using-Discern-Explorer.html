The CALL ACCEPT command accepts input from a user in a Discern Explorer program that controls the screen display. The value accepted as a result of the CALL ACCEPT is stored in a special variable called CURACCEPT. CALL ACCEPT takes advantage of VT terminal emulation keys, such as Help, PF1, and PF2.

The user-defined CALL ACCEPT allows the Discern Explorer program to define the length of the input field, the type of data that can be entered into each character of the field, and additional format options. This combination of information is sometimes referred to as the edit mask. Most Discern Explorer users, excluding Oracle personnel, use this form of the CALL ACCEPT.

If you perform an accept for a scroll window, you can capture the value of a key pressed using the variable CURSCROLL. To use CURSCROLL in an accept, set modifier to S or issue the SET ACCEPT = SCROLL command before the accept. KEYPAD mode also must be turned on (by pressing PF4 on the numeric keypad). Note

The CALL ACCEPT command is only valid in back-end screen application programs that use the Paint command. This type of program was widely used in HNAC, but in general, it is only used for utility programs in Millennium Platform.

Syntax

The syntax of the CALL ACCEPT command is below. The command can fit on one line or run over multiple lines.

CALL ACCEPT (begin_row,begin_col,
	"{picture}[;{modifier}]" [,default]
[WHERE qualification])

Metasymbols

begin_col
The beginning column number (from 1 to 80 for an 80-column mode, and from 1 to 132 for a 132-column mode) on the screen on which the reverse-video box, scroll window, line, box, or text will be displayed.

begin_row
The beginning row number (from 1 to 24) on the screen on which the box, reverse-video box, text, line, or scroll window will be displayed.

default
The value that is displayed and entered automatically if the user presses <Return> without entering any information in the input field.

modifier
Options that enable the user to change the accepted input into another format. You must use a modifier if your picture contains anything other than 9 or N. If you are going to use the accepted information as a character field, you must use a modifier. One example of using a modifier is to change lowercase letters into uppercase letters. The following values are valid:

Value

Description

C

Character string (if 1024 is entered for the picture 9999, accept this as a character string).

D

Use the corresponding default character in the default clause and skip over for accept.

E

Erase input as it is entered.

F

Force help from the current SET HELP command to invoke the Help function automatically for the CALL ACCEPT without the user pressing the HELP key.

H

Hide the default. The default value is not displayed, but pressing <Return> uses that default anyway.

P

Prompt the user for a starting value for Help on this input field.

S

Scroll window accept. An existing scroll window allows the user to use the UP and DOWN arrows, or the <Prev Screen>, <Next Screen>, <PF1>, or <PF2> keys to move through the Help window.

U

Force to uppercase.

picture

The edit mask for one character that is accepted from the user. The following values are valid:

Value

Description

9

Numeric.

9 (n)

Repeat the 9 mask for the number of times indicated by (n).

A

Alpha (uppercase and lowercase letters).

A(n)

Repeat the A mask for the number of times indicated by (n).

D

Display. Use the corresponding default character in the default clause and skip over for accept.

N

Numeric or punctuation (0-9, period, or minus sign).

N(n)

Repeat the N mask for the number of times indicated by (n).

P

Any printable ASCII character.

P(n)

Repeat the P mask for the number of times indicated by (n).

X

Alphanumeric (letters or numbers). For example, to accept a ten-character name, the picture is "AAAAAAAAAA". To accept a printer name like "LP01", the picture is "AA99"; to accept a 30-byte character field, the picture is "P(30)".

qualification
A condition that limits the number of records to be displayed as valid matches. Qualifications can include IF, IN, and CASE statements, and can be combined using the qualifiers AND, OR, NOT, and BETWEEN. Relational operators are used to define the condition. The following is syntax for qualification:

(qualification)
NOT qualification
qualification AND qualification
qualification OR qualification
expression relational_operator expression

relational_operator
Identifies the condition in a qualification. To view valid operators, see Relational Operators.

Examples

The following examples illustrate different ways to use CALL ACCEPT:

The first example accepts data starting on row 5, column 52, and accepts up to 12 characters. To change the input field defaults, use the SET ACCEPT command. These characters can be any printable characters; they are converted to a character string. All letters are converted to uppercase.

CALL ACCEPT(05,52, "P(12);CU")

The second example accepts data starting on row 6, column 52, and accepts four characters. They can be any printable character; and are converted to a character string. All letters are converted to uppercase. If you press <Return>, "MINE" is accepted as the default. The only values that are accepted are defined in the WHERE portion of this command. Valid options are MINE, LP01, LP02, or LP03.

CURACCEPT is a variable defined by Discern Explorer as the information accepted when the input field is displayed for this program.

CALL ACCEPT(06,52, "PPPP;CU", "MINE"
WHERE CURACCEPT IN ("MINE","LP01","LP02","LP03"))
