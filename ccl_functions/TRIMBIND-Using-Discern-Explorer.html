The TRIMBIND function can be used to optionally control the bind parameter string length and maximize binding in the shared pool cache since the RDBMS creates a new shared pool cache instance based on the length. This reduces the RDBMS bind mismatch due to varying string lengths.

Syntax

TRIMBIND(expression, bindmode)

Metasymbol

expression
A character data type value.

bindmode

Mode

Description

1

Use small bind length to pad

2

Use medium bind length to pad

3

Use medium large bind length to pad

4

Use large bind length to pad

-length

Use absolute length for the bind length

Examples

The below example displays using different bindmodes with the TrimBind() function. Using Set Trace RDBDEBUG echoes the query that is passed to the RDBMS from Discern Explorer. For each of the example selects, the following query is passed to the RDBMS:

SELECT  R.DESCRIPTION FROM  REQUEST R  
WHERE R.DESCRIPTION =     RTRIM( :1 )

In the above query, :1 is a bind variable. Using Set Trace RDBBIND echoes information about the bind variables that are passed in the query to the RDBMS. The TrimBind() function causes the length of the bind variable to change based on the specified bindmode.

In the following query, using bindmode -3, :1 is a Char[3]:

	
	set trace rdbdebug go
	set trace rdbbind go

	SELECT INTO "nl"
		r.description
	FROM request   r
	WHERE r.description = trimbind("ABC",-3)
	WITH nocounter 
	GO

In the following query, using bindmode 1, :1 is a Char[32]:

	SELECT INTO "nl"
		r.description
	FROM request   r
	WHERE r.description = trimbind("ABC",1)
	WITH nocounter
	GO

In the following query, using bindmode 2, :1 is a Char[128]:

	SELECT INTO "nl"
		r.description
	FROM request   r
	WHERE r.description = trimbind("ABC",2)
	WITH nocounter 
	GO
