The SET HELP command identifies the type of help that is called when the user presses HELP at a CALL ACCEPT input field. When possible, the help window is displayed to the left or right of the input field so the user can view both the CALL ACCEPT field and the help information. The help window overlays the CALL ACCEPT input field if it is too large to be displayed to one side of the field.

Three types of help are available: message help, fixed (hard-coded) help, and table-driven help. The help_option selected determines the type of help used.

Help is automatically scrollable using the ARROW keys, PF1, PF2, NEXT SCREEN, and PREV SCREEN. Press ENTER, HOME, or PF3 to close the help window and return the cursor to the CALL ACCEPT input field. Selecting a line in the help window and pressing ENTER sends the information from help into the CALL ACCEPT input field. Pressing PF3 returns the cursor to the CALL ACCEPT input field without bringing any information from the help window. Use the SET HELP = POS option to override the default location of the help window. (For one-line textual help, information is displayed at the bottom of the screen. This type of help is not scrollable, and does not return values to the CALL ACCEPT input field.)

In the SET HELP with SELECT command, hidden values can be returned as well as displayed items. The CURHELP variable stores the number of the item selected, making other fields associated with that item accessible to the program. For example, if a user selects a specific patient from a help window, the help also could return information associated with the patient that is not displayed, such as medical record number. The default control options used for a SET HELP with SELECT are: WITH NOCOUNTER, HEADING, SEPARATOR = " ", and NOFORMFEED.

Use GO at the end of the command to make it valid throughout a Discern Explorer session. If you do not use GO, the SET HELP command is valid only during the execution of the program that called it, and it must be embedded in an executable program (between the CREATE PROGRAM and END commands).

Syntax

SET HELP = help_option [GO]

Metasymbols

help_option

The type of information to be displayed in the help window. The following options are valid:

Option

Description

FIX (fixstring)

FIX is a function to be used only with the SET HELP command, where fixstring represents any character string with each individual help value separated by commas. The entire string should be surrounded by single quotation marks.

OFF

No help is available.

POS

Determines the position of the documentation window on the screen according to the syntax: POS(begin_row,begin_col,num_row,num_col). The default position is column 1, row 3.

PROMPTMSG (prompt)

Defines the text of a prompt message, where prompt represents the actual text of the prompt. The default text is, "Enter Help Start=>".

SEARCHMSG (search)

Defines the text of a search message, where search represents the actual text of the message. The default text is, "Enter Help Search=>".

SELECT

Any valid Discern Explorer query reading tables from the data dictionary. The extracted data then is shown in the help window as valid values for the CALL ACCEPT input field. The first field selected corresponds to the value that will be returned in the accepted input field.

text_string

Any characters embedded in quotes.

Examples

The following examples illustrate different ways to use SET HELP:

To create a one-line help string that is displayed at the bottom of the screen, enter the following command:

SET HELP = "Valid printers are: LP01, LP07, LP09"

The following example creates a help window that displays four valid choices. When the user highlights a choice and presses RETURN, the selected data is placed in the CALL ACCEPT input field.

SET HELP = FIX('LP02,LP03,LP04,LP07')

You also can enter a line number at the Select line # prompt displayed at the bottom of the scroll window or scroll through the window to select a line. When you create a help window, the message, Select line #, is displayed at the bottom of the window automatically.

The following example uses the CONCAT( ) function along with the FIX option to create a help message that can extend over as many lines as you need:

SET FIX_MSG =
CONCAT(	'LP02',
		'LP03',
		'LP04',
		'LP07')
SET HELP = FIX(VALUE(FIX_MSG))

To simulate a help window with three values along with a description of each value (using the FIX function), enter the following command:

SET HELP = FIX('LP01"LAB PRINTER",LP02"COMP ROOM",LP03"OFFICE"')

To display the valid Encounter Types from the CODE_VALUE table, in the help window, enter the following command:

SET HELP  =  SELECT INTO "NL:"
	CV.CDF_MEANING
FROM	CODE_VALUE CV
WHERE	CV.CODE_SET = 69
CALL ACCEPT(4,28,"A(12);CU")
SET V_ENCNTR_TYPE = CURACCEPT
CALL TEXT(5,5,V_ENCNTR_TYPE)

To not display a help window, enter:

SET HELP = OFF

The following example asks the user to enter a start value for the help on the encounter type. The "P" and "F" modifiers in the CALL ACCEPT force the help window to open and prompt the user for a starting value. The POS and PROMPTMSG options are used to override the default placement of the help window and the default prompt message.

call text(4,5,"Enter Encounter Type: ")
SET HELP  = POS(5,25,15,40)
SET HELP  = PROMPTMSG("ENTER STARTING VALUE FOR HELP")
SET HELP  =  SELECT DISTINCT INTO "NL:"
	CV.CDF_MEANING
FROM	CODE_VALUE CV
WHERE	CV.CODE_SET = 69 AND
	CV.CDF_MEANING >= CNVTUPPER(CURACCEPT)
ORDER	CV.CDF_MEANING
CALL ACCEPT(4,28,"A(12);CUPF")
SET V_ENCNTR_TYPE = CURACCEPT
CALL TEXT(5,5,V_ENCNTR_TYPE)

The ORDER clause determines the order in which the help information is sorted. If the ORDER clause is not included, the sequence of the help information is based on the key the SELECT statement qualified on.
