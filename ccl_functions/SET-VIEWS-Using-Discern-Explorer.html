The SET VIEWS command turns views on or off in a Discern Explorer session or in the execution of a Discern Explorer program. Views are logical relationships between two or more tables that automatically joins those tables and displays selected fields from those tables. Creating views causes additional overhead when running Discern Explorer programs; therefore, the VIEWS option is turned off by default.

EnterÂ GO at the end of the command to make it valid throughout a Discern Explorer session. If you do not use GO, the SET VIEWS command is valid only during the execution of the program that called it, and it must be embedded in an executable program (between the CREATE PROGRAM and END commands).

Before Discern Explorer versions 8.14.2 and 9.02.2, enabling Set Views Check and using a view name that has the same name as a program generates errors. This issue is resolved in Discern Explorer versions 8.14.2 and 9.02.2.

Syntax

SET VIEWS = view_option [GO]

Metasymbol

view_option
Determines whether views are on or off. The following options are valid:

Option

Description

CHECK

Turn views on.

NOCHECK

Turn views off.

Examples

The following examples illustrate different ways to use SET VIEWS:

To turn views on for a Discern Explorer session, run the below command.

SET VIEWS CHECK GO

To turn views on during the execution of a Discern Explorer program, run the below command.

SET VIEWS CHECK

To turns views off for a Discern Explorer session, run the below command.

SET VIEWS NOCHECK GO
