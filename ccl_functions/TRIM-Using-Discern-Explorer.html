TRIM () returns a result of type String that is a specified text expression stripped of any leading, embedded, or trailing characters that are less than or equal to spaces. TRIM() is a varchar function.

Syntax

TRIM(text_expr[,trim_option])

Metasymbols

text_expr
A text expression.

trim_option
An integer that determines which spaces are stripped from an expression. If omitted, Option 1 is used by default. The following options are valid:

Note

Space references Char(32); the pattern-matching character ? is Char(1), the pattern-matching character * is Char(2), and so on. Use the Check() function around the expression if you want to remove the pattern-matching characters when using Options 1 through 3.

See RDBMS documentation for information on the application of RTRIM() and LTRIM(). Generally, RTRIM() trims trailing spaces and LTRIM() trims leading spaces.

Examples

In the below example, the printed result is: * ABCD *.

CONCAT("  *  " , TRIM("AB    " ,1) , TRIM("CD  ") , " *  ")

The below example is displayed: Last Name, First Name (for example, Smith, James). The SUBSTRING( ) function is used to limit the number of spaces used, while the CONCAT( ) function is used to combine the two columns.

NAME = SUBSTRING(1,40,CONCAT(TRIM(P.NAME_LAST_KEY),", ",( P.NAME_FIRST_KEY)))

The following example trims the embedded spaces and returns: "ABCDEFGHI":

STRING = TRIM("ABC  DEF  GHI",4)

When used to create Select expressions, the TRIM() function has no effect unless it is used in the context of another function like CONCAT(). In the below example, the Select expressions, TLNAME and LNAME are both displayed as fixed-width columns of the same width. The Select expression CNAME has trailing spaces between the first and last names, whereas CTNAME has a single space between the first and last names, and a trailing space after the last name. Discern Explorer uses a temporary table with fixed column widths to display the output of a Select command that does not use Report Writer clauses. The TRIM() function has no effect on the width of the column.

SELECT INTO $OUTDEV
	TLNAME = TRIM(P.NAME_LAST)
	, LNAME =  P.NAME_LAST
	, CNAME = CONCAT(P.NAME_FIRST, " ", P.NAME_LAST)
	, CTNAME = CONCAT(TRIM(P.NAME_FIRST), " ", TRIM(P.NAME_LAST))

FROM
	PERSON   P

WITH MAXREC = 100, FORMAT, SEPARATOR = " "

The below example removes the pattern-matching characters "?" and "*", and any leading or trailing characters that are less than or equal to a space from $LNAME before that value is compared to the P.NAME_LAST field. Any leading or trailing characters that are less than or equal to a space are removed from $FNAME, but the pattern-matching characters are not removed.
