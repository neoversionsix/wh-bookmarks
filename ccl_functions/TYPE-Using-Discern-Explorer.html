The TYPE( ) function is used to declare the datatype of a field in a table that is created using the SELECT INTO TABLE table_name command.

Syntax

Type(data_type)

Metasymbols

Data_Type

Description

<len>

Fixed or maximum length for character

C

Character

V

Variable

G

graphical/raw/long raw

Z

clob/blob

I1

integer

I2

integer

I4

integer

F4

real number

F8

real number

DM12time stamp
DM14time stamp with time zone

DQ8

datetime

Q8

datetime

W8

long integer

The V and Z data type must be used in combination with the G, C, or C<len> types. Any combination of C, V, G, or Z can be used.

Prior to Discern Explorer Version 8.5.6, a C<len> of 32000 or greater must be used in combination with the Z to create a clob/blob field; otherwise, a varchar2 field is created. In Discern Explorer Version 8.5.6, a C<len> of 6000 or greater must be used in combination with the Z to create a clob/blob field; otherwise, a varchar2 field is created. Generally, the RDBMS limits a VC data type to a max of 4000. If the field needs to store larger values, it should be created as a clob or blob using "ZVC32000" or "ZGVC32000" as the datatype. If <len> is not specified when using VC, a length of 100 is used by default. "W8" requires Discern Explorer Version 8.10.2 or higher. Prior to Discern Explorer Version 8.11.4, using I1 and I2 created a field with an I4 datatype. Beginning in Discern Explorer Version 8.11.4, using I1 and I2 actually creates a field with an I1 or I2 datatype.

Discern Explorer limits the combined length of all the fields in the table definition to 35000 or less.

When creating RDBMS tables, the aforementioned data types are converted to a data type used by the RDBMS. To see the actual data types that are used at the RDBMS level, use SET TRACE RDBDEBUG prior to executing the SELECT INTO TABLE table_name command.

Examples

The following example uses the TYPE( ) function to set the data types of the columns on the custom RDBMS table:

SELECT INTO TABLE CUST_TEMP_EXTRACT                                                                      
 IONE=TYPE("I1"),
 ITWO=TYPE("I2"),
 IFOUR=TYPE("I4"), 
 FFOUR = TYPE("F4"),
 FEIGHT=TYPE("F8"),
 CHAR_TEN=TYPE("C10"), 
 VAR_CHAR=TYPE("VC"), 
 VC_SIXTY=TYPE("VC60"),
 DEIGHT=TYPE("DQ8"),
 QEIGHT=TYPE("Q8")
WITH ORGANIZATION = "TEMPRDB"

In the example below, the following values are created:

  • The VC_FLD is created with a VC100 data type. Since Type("VC") is used, the length defaults to 100.
  • The VC200_FLD is created as a VC200 data type.
  • The CLOB_FLD is created as a CLOB data type since Z is specified in combination with a VC length of 32000.
SELECT INTO TABLE CUST_TEMP_EXTRACT                                                                    
	VC_FLD=TYPE("VC"),
	VC200_FLD=TYPE("VC200"),
	CLOB_FLD=TYPE("ZVC32000") 
WITH ORGANIZATION = "TEMPRDB"

In the following example, BLOB_FLD is created as a BLOB datatype since Z and G are specified in combination with a VC length of 32000:

SELECT INTO TABLE CUST_TEMP_EXTRACT                                                                       
	BLOB_FLD=TYPE("ZGVC32000") 
WITH ORGANIZATION = "TEMPRDB"
