CNVTSTRING ( ) converts a numeric value to a character value.

Note

The CNVTSTRING( ) output defaults to 11 characters. Extraneous leading or trailing spaces can be deleted by using the TRIM( ) function. If the numeric value contains more than 11 digits, a length must be specified to prevent the result from truncating to 11 characters. Beginning in Discern Explorer Versions 8.17.5 and 9.05.5, a warning is generated if the result is truncated. If the numeric value contains digits to the right of the decimal point, a length and precision must be specified to prevent those digits from being dropped.

Consider using CNVTSTRINGCHK() instead of CNVTSTRING(), since CNVTSTRINGCHK() checks for overflow on the numeric expression and fills it with asterisks (*) if a truncation occurs in the display field.

Syntax

CNVTSTRING(numeric_value[,length[,precision,justify]])

Metasymbols

justify
Defines whether the converted string is left or right-justified. When this option is not specified, left justification is used by default. Length and precision must also be specified. The following option is valid:

R Right-justified

length
An integer that defines the number of characters in the returned string. If left unspecified, a default value of 11 is used.

numeric_value
An expression that is a numeric value.

precision
An integer representing the number of digits to the right of the decimal. Length must also be specified. The string that is returned is rounded to the precision following IEEE 754 recommendations, which rounds a trailing digit of 5 up in some cases and down in other cases. The ROUND() function can be used in CNVTSTRING() to always round a trailing digit of 5 up.

Examples

In the following example, NEWSTRING is set to 789:

NEWSTRING = CNVTSTRING(789,3)

In the following example, NEWSTRING1 is set to 1234567890123.456:

NEWSTRING1 = CNVTSTRING(1234567890123.456 ,17,3)

In this example, NEWSTRING2 is set to 12345678901. Since the numeric expression exceeds 11 characters, and length and precision are not used, the resulting character string is truncated to 11 characters.

NEWSTRING2 = CNVTSTRING(1234567890123.456)

In this example, NEWSTRING3 is set to 123. Since the length and precision are not used, the resulting character string does not include the digits that are to the right of the decimal.

NEWSTRING3 = CNVTSTRING(123.456)

The following example uses the ROUND() function in CNVTSTRING() to always round a trailing digit of 5 up:

call echo(cnvtstring(round(31.025,2),17,2)) go
call echo(cnvtstring(round(31.015,2),17,2)) go
