The CUBE() function is valid only in the Group By clause. When the CUBE function is used in the Group By clause, the query returns subtotals for all combinations of the specified Select expressions. Beginning in Discern Explorer version 8.12.3, the CUBE function must be declared before it is called.

Syntax

CUBE(select_expression{,select_expression})

Metasymbols

select_expression
A field from the selection list of a Group By query. The field must be explicitly listed in the Select clause.

Examples

The following is an example of a simple Group By query:

declare cube() = f8 go
select
    e_encntr_type_class_disp = uar_get_code_display(e.encntr_type_class_cd)
    , e.encntr_type_class_cd
    , e_encntr_type_disp = uar_get_code_display(e.encntr_type_cd)
    , e.encntr_type_cd
    , cnt = count(*)

from
    encounter   e

where e.reg_dt_tm between cnvtlookbehind("1,w") and cnvtdatetime(curdate, curtime3)
    and e.encntr_type_class_cd = er_var
    and e.encntr_type_cd != 0.0

group by
    e.encntr_type_class_cd
    ,e.encntr_type_cd


order by
    e.encntr_type_class_cd
    , e.encntr_type_cd
go

The following example uses CUBE(e.encntr_type_class_cd, e.encntr_type_cd):

declare cube() = f8 go
select
    e_encntr_type_class_disp = uar_get_code_display(e.encntr_type_class_cd)
    , e.encntr_type_class_cd
    , e_encntr_type_disp = uar_get_code_display(e.encntr_type_cd)
    , e.encntr_type_cd
    , cnt = count(*)

from
    encounter   e

where e.reg_dt_tm between cnvtlookbehind("1,w") and cnvtdatetime(curdate, curtime3)
    and e.encntr_type_class_cd = er_var
    and e.encntr_type_cd != 0.0

group by
    CUBE(e.encntr_type_class_cd
    ,e.encntr_type_cd)


order by
    e.encntr_type_class_cd
    , e.encntr_type_cd
go
