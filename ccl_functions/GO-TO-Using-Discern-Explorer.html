The GO TO command directs program control to the set of Discern Explorer instructions under a specified label. The GO TO command begins an unconditional branch so that program execution moves to the label and begins processing commands starting at that label. Program execution does not automatically return to the point where the branch occurred. (To perform a branch that returns, use the EXECUTE  FROM command.)

When you use the GO TO command in a program, the SET HELP, SET DOC, and SET VALIDATE options are not changed automatically. For example, if SET HELP is turned off, it remains off when the GO TO command is performed. You can change the setting of the option, however, in the range of instructions executed by the GO TO command.

GO TO is not directly supported in a Reportwriter clause in a Discern Explorer Select command. Beginning in Discern Explorer version 9.6.0 GO TO can be used in a Subroutine that is called in a Reportwriter clause. Prior to Discern Explorer version 9.6.0, using GO TO in a Subroutine that is called in a reportwriter clause will likely have unexpected results. 

GO TO has two forms, a basic directional form and an arithmetic form that can be used to direct program control based on a user's response to a prompt.

Syntax

Basic GO TO:

GO TO label_name

Arithmetic GO TO:

GO TO (label_name{,label_name}) variable

Metasymbols

label_name

The name of a label that identifies a set of Discern Explorer instructions that can be branched to using a SET HOME or GO TO command. See The Label Directive for more information on creating labels.

variable

The name to which an expression is stored.

Examples

The following examples illustrate different ways to use GO TO:

The following partial program uses the GO TO command to pass control to a label named Exit_Program if no records qualify in the Select.

SELECT INTO "NL:"
PA.PERSON_ID,
PA.ALIAS,
TYPE = UAR_GET_CODE_DISPLAY(PA.PERSON_ALIAS_TYPE_CD)
FROM	PERSON_ALIAS PA
WHERE	PA.PERSON_ALIAS_TYPE_CD = SSN_VAR
AND PA.PERSON_ID = PERSON_ID_VAR
WITH	COUNTER

IF(CURQUAL = 0)
	GO TO EXIT_PROGRAM
ENDIF

The following example is a fragment of a larger program. The example prompts the user to indicate whether the information entered previously is correct. If the user enters N the program prompts for the line number. The arithmetic format of the GO TO command uses that line number to direct program control to the block of commands corresponding to a label for that line number.

#CORRECT_YN
CALL TEXT(24,1,"CORRECT? Y/N")
CALL ACCEPT(24,20, "P;CU", "Y" WHERE CURACCEPT IN ("Y","N"))
IF(CURACCEPT = "N")
	CALL CLEAR(24,1)
	CALL TEXT(24,1,"LINE #")  ;prompt for line number
	CALL ACCEPT(24,10,"9",1 WHERE CURACCEPT BETWEEN 1 AND 3)
	GO TO (LINE1,LINE2,LINE3) CURACCEPT
ENDIF
