This function is only for servers and internal system usage. It enables the server to execute Discern Explorer programs using the username that was used to log in to a front-end application instead of the default username of the server. The function returns 1 if valid, 0 if invalid. Beginning with Discern Explorer versions 8.15.0 and 9.03.0, interactive back-end sessions use PROXYUSER to set the Reserved Variable CURUSER to the CCLSECLOGIN username.

Syntax

 ProxyUser(expression)

Metasymbol

Expression - 0 (zero) or 1 (one)

0

Turn off ProxyUser and revert back to server CURUSER

1

Turn on ProxyUser and set CURUSER to the username that was used to login to the front end application. If the front end username is longer than 12 characters it is truncated to 12 characters when setting CURUSER

Examples

set stat = proxyuser(1)
set stat = proxyuser(0)
