The SET TRANSACTION command provides access to RDB. This command typically is issued before DELETE, INSERT, and UPDATE commands.

Use GO at the end of the command to make it valid throughout a Discern Explorer session. If you do not use GO, the command is valid only during the execution of the program that called it and must be embedded in an executable program (between the CREATE PROGRAM and END commands).

Syntax

SET TRANSACTION = tran_option
[WITH table_name = tran_mode {,table_name = tran_mode}] [GO]

Metasymbols

tran_option

Determines the type of transaction to be established. The following options are valid:

Option

Description

READ_WAIT

Read only transaction with wait mode

READ_NOWAIT

Read only transaction, no wait mode

WRITE_WAIT

Read/write transaction with wait mode

WRITE_NOWAIT

Read/write transaction, no wait mode

table_name

The name of a specific table to which access can be restricted.

tran_mode

The type of access provided for the specified table. The following options are valid:

  • EXCLUSIVE_READ
  • EXCLUSIVE_WRITE
  • PROTECTED_READ
  • PROTECTED_WRITE
  • SHARED_READ
  • SHARED_WRITE

Examples

The examples below illustrate different ways to use SET TRANSACTION.

To indicate to RDB that read-only mode should be used, enter the following command:

SET TRANSACTION READ_NOWAIT

To indicate to RDB that read-write mode should be used, enter the following command:

SET TRANSACTION WRITE_NOWAIT WITH JOB_HISTORY = SHARED_WRITE
