The CALL PARSER command parses the contents of a dynamic Discern Explorer command. Consecutive CALL PARSER commands terminated by a "GO" can be issued. Before Discern Explorer version 8.7.2 the CALL PARSER command was limited to 32,000 characters.Â  Discern Explorer version 8.7.2 increased the limit to 100,000 characters.

Syntax

CALL PARSER (expression [,mode])

Metasymbols

expression
An expression is the highest-level metasymbol among constants, field names, variables, and so on. The following expressions are valid:

alias.field_name
array_name [dimension]
array_name [dimension1, dimension2]
array_name [dimension1, dimension2, dimension3]
constant
expression arithmetic_operator expression
-expression
(expression)
function(expression{,expression})
parameter
record_name{->field_name}
table_name.field_name
variable

Note

Square brackets are part of the array_name syntax; they do not indicate an optional part of the syntax.


mode

Introduced in Discern Explorer version 8.12.6.

ModeDescription
0,1Default behavior.
2Mark and release tree memory for a parser command that begins and ends in one parser call.
3

Mark and release symbol and tree memory for a parser command that begins and ends in one parser call. Introduced in Discern Explorer versions 8.15.4 and 9.03.4.


Examples

The following example parses the contents of the array PARSER_BUFFER, which builds a query:

SET PARSER_BUFFER[10] = FILLSTRING(132," ")
SET X = INITARRAY(PARSER_BUFFER," ")
SET PARSER_BUFFER[1] = "SELECT P.NAME_FULL_FORMATTED"
SET PARSER_BUFFER[2] = " FROM PERSON P"
SET PARSER_BUFFER[3] = " WITH NOCOUNTER GO"
SET CNT = 1 GO
WHILE (CNT <= 3)
	CALL PARSER(PARSER_BUFFER[CNT])
	SET CNT = CNT + 1
ENDWHILE

The following example builds a select command and executes it using the CALL PARSER command:

set outname = $outdev
set cmd = concat("select into value(outname) p.person_id",
		" from person p where p.updt_dt_tm > cnvtlookbehind('1,D') with maxrec = 100,",
		" format, separator = ' ' go")
call parser(cmd)

Note

Before version 7.7 of Discern Explorer, the Call Parser() syntax allowed a second optional parameter. Beginning in version 7.7 of Discern Explorer, the second parameter is ignored.
