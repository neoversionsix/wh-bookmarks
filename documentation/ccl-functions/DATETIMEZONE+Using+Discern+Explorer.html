The DATETIMEZONE( ) function returns the adjusted datetime based on the Time Zone index.

Use DATETIMEZONEFORMAT() to display a datetime value adjusted to a timezone other than the local timezone. Use DATEBIRTHFORMAT() or DATEDECEASEDFORMAT() to display the birth date or deceased date.

When using the Format( ) function or a Display Option to display the adjusted datetime value, the Display Qualifier 4 must be used.

Syntax

DATETIMEZONE(<datetime>,<timezone_index>[,<option>])

Metasymbols

datetime
    A datetime value.

timezone_index
    An integer representing the time zone. If timezone_index is zero, then the application time zone is returned.

Note

Oracle has adopted a Windows standard that has defined time zone names and their associated index numbers.

option
    1 - To convert the Coordinated Universal Time (UTC) datetime to local datetime using the index (default if omitted).
    2 - To convert the local datetime to UTC datetime using the index.

Note

If UTC mode is off, then the function returns the datetime as is and performs no conversion.

Examples

When the following example is executed in an environment set to CST(Central Standard Time) with UTC activated, the Call Echo displays "05-MAY-2002 10:00:00.00", since 43 is the time zone index for Pacific Standard Time:

SET P_DT_TM = DATETIMEZONE(CNVTDATETIME("05-MAY-2002 12:00"),43,1) GO
CALL ECHO(FORMAT(P_DT_TM,";4;Q")) GO

In the following example, APP_TIME displays the updated time converted to the time zone of the application. SYS_TIME displays the updated time converted to the system's time zone:

SELECT
      APP_TIME = DATETIMEZONE( P.UPDT_DT_TM ,CURTIMEZONEAPP,1)"HH:MM;4;D",
      SYS_TIME = DATETIMEZONE( P.UPDT_DT_TM,CURTIMEZONESYS,1) "HH:MM;4;D"
FROM PERSON P
WITH MAXREC = 100
