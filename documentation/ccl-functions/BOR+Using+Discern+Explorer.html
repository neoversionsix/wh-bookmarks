The BOR( ) function returns an integer expression that is the result of a bit_number to bit_number or a comparison of two integers in base-2. The bit_number is the position of a bit in a field.

Syntax

BOR(integer,integer)

Metasymbols

Integer
Any whole number.

The key to understanding how the BOR( ) function works is to first convert the two numbers being compared into base-2 numbers.

In base-2, numbers are represented by placing a zero or one in a bit number position and a subscript value of 2. In the base-10 system, the number base in which we usually count, we count from one to nine, carry the one, and start over at zero. The below examples display conversions from base-10 to base-2.

The BOR( ) function identifies if the conversion of either integer to base-2 results in a one being present in any bit_number position. If a one is present, a one is present in the corresponding bit_number position in the result of the BOR( ) function. 

Examples

The following example displays the first number as three and the second number as five:

RESULT=BOR(3,5)

310=000000112

510=000001012

Looking from right to left, you can see that 000000112 has a one in the zero and first bit_number positions and that 000001012 has a one in the zero and second bit_number positions. Thus the RESULT value has a one in the zero, first, and second bit_number positions.

In base-2, the following result is displayed:

RESULT=000001112

When you convert this back to base-10, the following result is displayed:

RESULT=7

The following example displays the integers of 12 and 10:

RESULT=BOR(12,10)

1210=000011002

1010=000010102

RESULT=000011102

When you convert this back to base-10, the following result is displayed:

RESULT=14
