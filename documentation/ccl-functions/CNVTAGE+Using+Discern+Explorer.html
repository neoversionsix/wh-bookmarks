CNVTAGE( ) calculates an age in years, months, days, or hours, and returns a 12-character string based on the following formula:

IF (AGE < 1 DAY) "HH:MM HOURS"
ELSEIF (AGE < 1 WEEK) "999 DAYS"
ELSEIF (AGE < 1 MONTH) "999 WEEKS"
ELSEIF (AGE < 2 YEAR) "999 MONTHS"
ELSE "999 YEARS"
ENDIF

The default break point for days, weeks, and months are:
Days 7
Weeks 5
Months 24

There are two ways to flex the break point when using the CNVTAGE( ) function. Use the CCLCNVTAGE logical or system variable to flex the defaults for all programs or use the SET MODIFY CNVTAGE( ) command to flex the defaults in individual programs.

Syntax

CNVTAGE(rdb_date_time[,rdb_date_time,0])

or

CNVTAGE(internal_date, internal_time[,internal_date,internal_time|,internal_date,internal_time])

Metasymbols

rdb_date_time
An expression that contains the date and time as one value

0
Used internally to indicate that the first two arguments are date_time expressions

internal_date
A Oracle Internal System Date

internal_time
An integer value representing the time in HHMM format

Examples

The following example calculates the person's current age. Because only one date time value is passed to the CNVTAGE( ) function, the current date and time are used as the ending date time value:

AGE = CNVTAGE(P.BIRTH_DT_TM)

The following example calculates the person's age at the time the encounter was registered:

ADM_AGE = CNVTAGE(P.BIRTH_DT_TM, E. REG_DT_TM,0)

The following example passes Oracle internal dates and integer times to the CNVTAGE( ) function:

SET BEGIN_DATE = CNVTDATE(10261990)
SET BEGIN_TIME = 1300
SET END_DATE = CNVTDATE(10261990)
SET END_TIME = 2230
SET AGE = CNVTAGE( BEGIN_DATE, BEGIN_TIME, END_DATE, END_TIME )
