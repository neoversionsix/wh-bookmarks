The CNVTINT( ) function converts an expression containing numeric characters or a real number to an integer data type. This is useful for comparing or joining two columns when one column is a numeric column and the other is a character column. In Discern Explorer versions 8.3.0 and higher, if the result of converting the expression results in a value that exceeds the maximum integer value, CNVTINT() returns a float (F8) value.

Syntax

CNVTINT(expression)

Metasymbol

expression

A value of any data type.

Examples

In the following example, I is set equal to the integer 123:

STR = "0123"
I = CNVTINT(STR)


In the following example, I is set equal to the integer 456 since digits to the right of a decimal place are ignored:

I = CNVTINT("456.842")


In the following example, I is set equal to the integer 789 since digits to the right of a decimal place are ignored:

I = CNVTINT( 789.567 )


In the following example, I is set equal to 0 because "aaa" is not a numeric value:

I = CNVTINT("aaa2345.123")


In the following example, F is set equal to 2147483648 but has an F8 data type because 2147483648 exceeds the maximum integer value:

SET F = CNVTINT("2147483648")
{"serverDuration": 760, "requestCorrelationId": "c8ae0b807e242f74"}
