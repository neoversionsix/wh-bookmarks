BXOR( ) returns an integer expression that is the result of a bit_number to bit_number exclusive or comparison of two integers in base-2. The bit_number is the position of a bit in a field.

Syntax

BXOR(integer,integer)

Metasymbols

Integer
Any whole number.

The key to understanding how the BXOR( ) function works is to first convert the two numbers being compared into base-2 numbers.


In base-2, numbers are represented by placing a zero or one in a bit number position and a subscript value of 2. In the base-10 system, the number base in which we usually count, we count from one to nine, carry the one, and start over at zero. The below examples display conversions from base-10 to base-2.

The BXOR( ) function identifies if the conversion of both integers to base-2 results in a one being present in either integer's specified bit_number positions. If a one is present in either of the bit_number positions, not both, then the result has a one. If a one or zero is present in both bit_number positions, then the result of the BXOR( ) function is a zero.

Examples

The following example displays the first number as three and the second number as five:

RESULT=BXOR(3,5)

310=000000112

510=000001012

Looking from right to left, you can see that 000000112 has a one in the zero and first bit_number positions and that 000001012 has a one in the zero and second bit_number positions. Thus the RESULT value has a one in the zero bit_number position.
