CNVTMIN( ) converts a time value to an integer that is the total number of minutes for that value within a 24-hour period. It returns the number of minutes as an integer. This is useful when performing arithmetic operations on time values. The conversion can be performed in the other direction by using the CNVTTIME() function.

Syntax

CNVTMIN(time[,time_type])
CNVTMIN(rdb_dt_tm[,time_type])

Metasymbols

time
An expression that typically is in the format HHMM.

time_type
An integer representing the time format specified in the time expression. The following are valid options:

Option

Time Expression

Description

1

Time column in HHMM format

Default

2

Time column in HHMMSS format


3

Time in 10-millisecond units


4

rdb_dt_tm, convert date and time

Default for rdb_dt_tm
Using this time_type, before 01-Mar-1800 the total number of minutes since 01-Jan-1800 is returned. Between 01-Mar-1800 and 28-Feb-1900 the total number of minutes since 01-Jan-1800 plus 1440 is returned. After 28-Feb-1900 the total number of minutes since 01-Jan-1800 plus 2880 is returned..

5

rdb_dt_tm, convert time only


rdb_dt_tm
A column that stores the date and time.

Examples

In the below example, D1 is set equal to 60.

D1 = CNVTMIN(0100,1)

In the below example, D2 is set equal to 618.

D2 = CNVTMIN(1018,1)

In the below example, if the ORDER_DT_TM column is 01-JAN-1998 10:50:00, D3 is set equal to 650.

D3 = CNVTMIN(O.ORDER_DT_TM,5)

In the below example, if the ORDER_DT_TM column is 01-JAN-1998 10:50:00, D4 is set equal to 104142890.

D4 = CNVTMIN(O.ORDER_DT_TM)
