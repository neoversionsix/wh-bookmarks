The CNVTJSONTOREC() function converts a JSON string to a Discern Explorer record structure. It returns 1 for success and 0 for error. By default if the record structure defined in the JSON string does not exist, it is created and populated. If the record structure already exists it is populated. When called in a Reportwriter Clause, CNVTJSONTOREC() populates an existing record structure but does not create the record structure if it does not already exist.

For information about writing the Discern Explorer record structure as a JSON stream to the displayer, default log or listing file, or to a user-defined file, see CALL ECHOJSON.

For information about converting a Discern Explorer record structure to a JSON string, see CNVTRECTOJSON.

Discern Explorer version 8.12.3 resolves a memory allocation issue when reaching 200 field names in the record definition. 

Before Discern Explorer versions 8.16.0 and 9.04.0, CNVTJSONTOREC() does not recognize \\\" as an escaped backslash followed by and escaped double quote.

Before Discern Explorer versions 8.16.0 and 9.04.0, CNVTJSONTOREC() ignores an empty list in the JSON string. For example, list_name:[] is ignored. Beginning in Discern Explorer versions 8.16.0 and 9.04.0 an empty list in the JSON string is converted to an empty list in the Discern Explorer record structure.

Syntax

CNVTJSONTOREC(expression,[mode[,casemode[,datemode[,flexmode]]]])

Metasymbols

expression
The name of an expression that contains the JSON string.

mode
Modes 0 to 5 only make a pass through the first occur of lists to generate the Discern Explorer record structure definition.

Modes 6 to 11 make a full pass through all occurs to generate the Discern Explorer record structure definition.

Mode

Description

0,6

Create the Discern Explorer record structure definition with nopersist, and populate it from the JSON string. (the default is 0).

1,7

Create the Discern Explorer record structure definition with nopersist, but do not populate it from the JSON string; initialize all lists to 0 occurs.

2,8

Create the Discern Explorer record structure definition with persist, and populate it from the JSON string.

3,9

Create the Discern Explorer record structure definition with persist, but do not populate it from the JSON string; initialize all lists to 0 occurs.

4,10

Create the Discern Explorer record structure definition with persistscript, and populate it from the JSON string.

5,11

Create the Discern Explorer record structure definition with nopersistscript, but do not populate it from the JSON string; initialize all lists to 0 occurs.

The mode Options 0 and 1 were introduced in Discern Explorer Version 8.6.4. The mode Options 2 thorough 11 were introduced in Discern Explorer Version 8.8.3.

casemode
Introduced in Discern Explorer version 8.10.0. By default, the record name, list names, and field names in the JSON string are converted to uppercase in the Discern Explorer record structure definition. Casemode allows list names and field names to keep their case from the JSON string. Before Discern Explorer versions 8.16.0 and 9.04.0, if the record name is not uppercase in the JSON string, and casemode 1 is used, the Discern Explorer record structure is not created because record names must be uppercase. Beginning in Discern Explorer versions 8.16.0 and 9.04.0 the record name is implicitly converted to uppercase when casemode 1 is used.

Casemode

Description

0

Convert list names and field names to uppercase (default).

1

List names and field names retain the case from the JSON string.

datemode
Introduced in Discern Explorer version 8.10.3 to implicitly recognize the International Organization for Standardization (ISO) date string if Date() is omitted.  In Coordinated Universal Time (UTC)-enabled environments, the date is converted from UTC to local time. In non-UTC-enabled environments, the date is used as is. Discern Explorer versions 8.13.0 and 9.01.0 added datemodes 2 and 3.

Datemode

Description

0

Do not cast an ISO date string to DQ8 (default)

1

Cast an ISO date string to DQ8

2Do not cast an ISO date string to DQ8, and convert UTC to local regardless of UTC
3Cast an ISO date string to DQ8, and convert UTC to local regardless of UTC


flexmode

Introduced in Discern Explorer versions 8.14.0 and 9.02.0. Flexmode is an integer value representing the bitwise BOR of the following options:

OptionDescription
0The default mode.
1Retain string, no conversion.
2

Retain binary values, and do not convert to backslash (\).

4

Convert \udddd to an ASCII character. Introduced in Discern Explorer versions 8.14.1 and 9.02.1.

8

Suppress errors. Introduced in Discern Explorer versions 8.14.2 and 9.02.2.


Examples

Assuming that the JSON_STR variable is equal to the following:

{"PERSON_REC":
{
     "firstName": "John",
     "lastName": "Smith",
     "address": [{
         "streetAddress": "2800 Rockcreek Parkway",
         "city": "Kansas City",
         "state": "MO",
         "postalCode": "64117"
     },
     {
         "streetAddress": "10236 Park Drive",
         "city": "Kansas City",
         "state": "MO",
         "postalCode": "64137"
     }],
     "phoneNumbers": [
         { "type": "home", "number": "816 555-1234" },
         { "type": "fax", "number": "816 555-4567" }
     ] } }

The following example creates and populates a Discern Explorer record structure named PERSON_REC:

SET JREC = CNVTJSONTOREC(JSON_STR)

The PERSON_REC record structure has the following definition:

RECORD PERSON_REC (
  1 FIRSTNAME =  VC
  1 LASTNAME =  VC
  1 ADDRESS[*]
   2 STREETADDRESS =  VC
   2 CITY =  VC
   2 STATE =  VC
   2 POSTALCODE =  VC
  1 PHONENUMBERS[*]
   2 TYPE =  VC
   2 NUMBER =  VC
)
Executing CALL ECHORECORD(PERSON_REC) then creates the following:
>>>Begin EchoRecord PERSON_REC   ;PERSON_REC
 1 FIRSTNAME=VC4   {John}
 1 LASTNAME=VC5   {Smith}
 1 ADDRESS[1,2*]
	  2 STREETADDRESS=VC22   {2800 Rockcreek Parkway}
	  2 CITY=VC11   {Kansas City}
	  2 STATE=VC2   {MO}
	  2 POSTALCODE=VC5   {64117}
 1 ADDRESS[2,2*]
	  2 STREETADDRESS=VC16   {10236 Park Drive}
	  2 CITY=VC11   {Kansas City}
	  2 STATE=VC2   {MO}
	  2 POSTALCODE=VC5   {64137}
 1 PHONENUMBERS[1,2*]
	  2 TYPE=VC4   {home}
	  2 NUMBER=VC12   {816 555-1234}
 1 PHONENUMBERS[2,2*]
	  2 TYPE=VC3   {fax}
	  2 NUMBER=VC12   {816 555-4567}
>>>End EchoRecord PERSON_REC varchar=14, varlist=2, reclen=114, reclen2=242

The below example uses the continuation character "\" to create the JSON string. It then uses CNVTJSONTOREC() to convert the JSON string to a record structure.

DROP PROGRAM TEST_CNVTJSONTOREC GO
CREATE PROGRAM TEST_CNVTJSONTOREC

DECLARE JSON_STR = VC
DECLARE JREC = I4
SET JSON_STR = ^{"PERSON_REC":\
{\
     "firstName": "John",\
     "lastName": "Smith",\
     "address": [{\
         "streetAddress": "2800 Rockcreek Parkway",\
         "city": "Kansas City",\
         "state": "MO",\
         "postalCode": "64117"\
     },\
     {\
         "streetAddress": "10236 Park Drive",\
         "city": "Kansas City",\
         "state": "MO",\
         "postalCode": "64137"\
     }],\
     "phoneNumbers": [\
         { "type": "home", "number": "816 555-1234" },\
         { "type": "fax", "number": "816 555-4567" }\
     ]\
 }\
 }^


SET JREC = CNVTJSONTOREC(JSON_STR)
CALL ECHORECORD(PERSON_REC)

END GO

The example below creates and populates Discern Explorer record structures named REC1 and REC2. The Discern Explorer records will have the following definitions:

Record Rec1(
     1 DTTM = DQ8)

Record Rec2(
     1 DTTM = DQ8)

In both REC1 and REC2, the DTTM item is initialized to the datetime value for 2014-10-08 20:43:24.00.

declare jsonbuf1 = vc
declare jsonbuf2 = vc
set jsonbuf1 = ~{"REC1":{"DTTM":"\/Date(2014-10-08T20:43:24.000+00:00)\/"}}~  
set jsonbuf2 = ~{"REC2":{"DTTM":"2014-10-08T20:43:24.000+00:00"}}~  
set stat = cnvtjsontorec(jsonbuf1,0,0,0) 
set stat = cnvtjsontorec(jsonbuf2,0,0,1)

The example below uses the datemode parameter.

declare jsonbuf = vc
set jsonbuf = ~{"REC1":{"DTTM":"\/Date(2014-10-08T20:43:24.000+06:00)\/"}}~  
set stat = cnvtjsontorec(jsonbuf,0,0,0) 
call echo(build("curutc=",curutc))
call echorecord(rec1)

The example above creates and populates a Discern Explorer record structure named REC1 with the following definition:

Record Rec1(
     1 DTTM = DQ8)

In a UTC-enabled environment, the following output is generated from the Call Echorecord(REC1) command:

>>>Begin EchoRecord REC1   ;REC1

 1 DTTM=DQ8   {67777010040000000    (2014-10-08 14:43:24.00) utc(1)}

In a non-UTC-enabled environment the +06:00 offset is ignored, and the following output is generated from the Call Echorecord(Rec1) command:

>>>Begin EchoRecord REC1   ;REC1

 1 DTTM=DQ8   {67777226040000000    (2014-10-08 20:43:24.00) utc(0)}

The example below uses the flexmode parameter.

declare json_str = vc with protect

set json_str = "{'rec_a':{'fld_a': 'ABC\u0026\u003c\u003eDEF'}}"
call echo(json_str) 
set stat = cnvtjsontorec(json_str, 0, 0, 0, 0) 
call echorecord(rec_a) 

set json_str = "{'rec_b':{'fld_b': 'ABC\u0026\u003c\u003eDEF'}}"
set stat = cnvtjsontorec(json_str, 0, 0, 0, 4) 
call echorecord(rec_b)

The following output is generated by the Call Echorecord(rec_a) command:

>>>Begin EchoRecord REC_A   ;REC_A

 1 FLD_A=VC24   {ABC\u0026\u003c\u003eDEF}

>>>End EchoRecord REC_A Varchar=1, Varlist=0, Fixsize=12, Varsize=32


The following output is generated by the Call Echorecord(rec_b) command:

>>>End EchoRecord REC_B Varchar=1, Varlist=0, Fixsize=12, Varsize=17
