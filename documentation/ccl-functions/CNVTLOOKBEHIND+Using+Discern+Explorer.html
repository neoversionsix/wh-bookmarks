The CNVTLOOKBEHIND( ) function returns a DQ8 or DM12 value that is a number of units (hundredths of seconds, seconds, minutes, hours, days, weeks, months, or years) in the past. If the <datetime> is omitted, the current datetime is assumed. Discern Explorer version 8.14.3 or 9.02.3 introduced the ability to pass a DM12 timestamp value as the second parameter to CNVTLOOKBEHIND(). If the second parameter is omitted or a DQ8 datetime value is used for the second parameter, CNVTLOOKBEHIND() returns a DQ8 value. If a DM12 timestamp value is used for the second parameter, CNVTLOOKBEHIND() returns a DM12 timestamp value.

See the DTTMCALC description in the SET TRACE topic for information about flexing the behavior of the CNVTLOOKBEHIND() function.

The CNVTLOOKAHEAD() function is used to return a datetime value that is in the future.

Syntax

CNVTLOOKBEHIND("{units, interval_type}"[,datetime[,mode]])

Metasymbols

units
An integer representing the number of units to look back. A positive integer is expected. The absolute value of a negative the integer is used to look back that number of units.

interval_type
The time interval measured as HUN[SEC] S[EC] MIN H[OUR] D[AY] W[EEK] M[ONTH] Y[EAR].

datetime
The datetime or timestamp value from which to look back. If omitted, the current date and time are assumed. Before Discern Explorer version 8.14.3 or 9.02.3, only a DQ8 datetime value was accepted. Beginning with Discern Explorer version 8.14.3 or 9.02.3, a DQ8 datetime or DM12 timestamp value can be used. Beginning with Discern Explorer version 8.15.0 or 9.03.0, a variable that is declared as a DQ8 and assigned an internal date value can be used.

mode
Introduced in Discern Explorer version 8.7.2.

Mode

Description

0

If a Coordinated Universal Time (UTC) datetime, convert the datetime to local before applying the calculation.

1

If a UTC datetime and the look behind is more than the DTTMCALCFLEX trace, use the UTC for the calculation; otherwise, use the local datetime for the calculation.

2

Same as Mode 1, but for the BSD extension.

If mode is omitted, the DTTMCALC trace flexes the default mode. If DTTMCALC is disabled, Mode 0 is used. If DTTMCALC is enabled and the look behind is more than DTTMCALCFLEX trace, Mode 1 is used; otherwise, Mode 0 is used. By default, the DTTMCALCFLEX is 24 hours.

Examples

The following example echoes 22-DEC-1999 11:30:00.00:

CALL ECHO(FORMAT(CNVTLOOKBEHIND("10,D",CNVTDATETIME("01-JAN-2000 11:30")), ";;Q"))

CNVTLOOKBEHIND( ) can be used in a qualification on a Relational Database (RDBMS) table, provided the return value can be converted to a bind variable in the query that Discern Explorer passes to the RDBMS. The following example returns people who are less than 18 years old at the time the query is executed:

SELECT
     P.NAME_FULL_FORMATTED
     ,P.BIRTH_DT_TM
FROM PERSON P
WHERE P.BIRTH_DT_TM > CNVTLOOKBEHIND("18,Y")
WITH MAXREC = 1000

If the return value cannot be converted to a bind variable in the query, Discern Explorer passes to the RDBMS, the CNVTLOOKBEHIND() can be used in a qualification on the DUMMYT table.

The following example returns encounters that were registered in the past month when the person was 18 years or older at the time of registration:

SELECT
     P.NAME_FULL_FORMATTED
     ,P.BIRTH_DT_TM
     ,E.REG_DT_TM
FROM PERSON P
     ,ENCOUNTER E
     ,DUMMYT D
PLAN E WHERE E.REG_DT_TM BETWEEN CNVTLOOKBEHIND("1,M") AND CNVTDATETIME(CURDATE,CURTIME3)
JOIN P WHERE P.PERSON_ID = E.PERSON_ID AND P.BIRTH_DT_TM IS NOT NULL
/*passing e.reg_dt_tm prevents the return value of cnvtlookbehind() from being converted to a bind variable*/
JOIN D WHERE P.BIRTH_DT_TM < CNVTLOOKBEHIND("18,Y",E.REG_DT_TM)
WITH MAXREC = 1000

The following example uses the optional mode parameter. If the example is executed in an environment in the America/Chicago time zone, the Call Echo commands return 09-MAR-2014 00:00:00.00 and 09-Mar-2014 06:00:00 UTC in both UTC and non-UTC environments.

set start_dt_tm = cnvtdatetime("09-Mar-2014 03:00:00") 

set end_dt_tm = cnvtlookbehind("3, H",start_dt_tm,0) 
call echo(format(end_dt_tm,";;q")) 
call echo( datetimezoneformat(end_dt_tm,datetimezonebyname("UTC")
                       ,"dd-MMM-yyyy HH:mm:ss ZZZ",curtimezonedef) )

The following example uses the optional mode parameter. If the example is executed in an environment in the America/Chicago time zone, the Call Echo commands return 08-MAR-2014 23:00:00.00 and 09-Mar-2014 05:00:00 UTC in both UTC and non-UTC environments.

set start_dt_tm = cnvtdatetime("09-Mar-2014 03:00:00") 

set end_dt_tm = cnvtlookbehind("3, H",start_dt_tm,1) 
call echo(format(end_dt_tm,";;q")) 
call echo( datetimezoneformat(end_dt_tm,datetimezonebyname("UTC")
                       ,"dd-MMM-yyyy HH:mm:ss ZZZ",curtimezonedef) )

The following example uses the optional mode parameter. If the example is executed in an environment in the America/Chicago time zone and UTC is enabled, the Call Echo commands return 08-MAR-2014 23:00:00.00 and 09-Mar-2014 05:00:00 UTC. If the example is executed in an environment in the America/Chicago time zone and UTC is not enabled, the Call Echo commands return 09-MAR-2014 00:00:00.00 and 09-Mar-2014 06:00:00 UTC. In the America/Chicago time zone, daylight saving time began at 2:00 AM local time on 09-Mar-2014.

set start_dt_tm = cnvtdatetime("09-Mar-2014 03:00:00") 

set end_dt_tm = cnvtlookbehind("3, H",start_dt_tm,2) 
call echo(format(end_dt_tm,";;q")) 
call echo( datetimezoneformat(end_dt_tm,datetimezonebyname("UTC")
                       ,"dd-MMM-yyyy HH:mm:ss ZZZ",curtimezonedef) )

The following example requires Discern Explorer version 8.14.3 or 9.02.3 or later. It returns a DM12 timestamp value that is one-hundredth of a second before the current system timestamp:

call echo(format(cnvtlookbehind("1,hun",systimestamp),"dd-mmm-yyyy hh.mm.ss.cc;;d")) go

The following example requires Discern Explorer version 8.15.0, 9.03.0, or later:

declare date_var = dq8 with protect go
set date_var = cnvtdate(3292019) go
call echo(format(cnvtlookbehind("1,d",date_var),";;d")) go
