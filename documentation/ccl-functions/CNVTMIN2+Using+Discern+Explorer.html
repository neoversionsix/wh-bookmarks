CNVTMIN2( ) converts a date value and a time value into a number of minutes. The number of minutes is returned as an integer. This is useful when performing arithmetic operations on dates and times.

Before 01-Mar-1800 the total number of minutes since 01-Jan-1800 is returned. Between 01-Mar-1800 and 28-Feb-1900, the total number of minutes since 01-Jan-1800 plus 1440 is returned. After 28-Feb-1900, the total number of minutes since 01-Jan-1800 plus 2880 is returned.

Syntax

CNVTMIN2(date,time[,time_type])

Metasymbols

date

An expression that represents a  Oracle Internal System Date.

time

An expression that represents a time.

time_type

An integer representing the format of the time expression. The following integers are valid options:

Time Type

Description

1

Time column in HHMM format (default)

2

Time column in HHMMSS format

3

Time in 10-millisecond units

Examples

In the example below, if CURDATE is 70409 (October 7, 1992) and CURTIME is 1618 (4:18 p.m.), D1 is set equal to 101389938.

D1 = CNVTMIN2(CURDATE,CURTIME )
D1 = CNVTMIN2(CNVTDATE(100792), 1618) 


The following examples will both echo 114040180, which is the number of minutes between Jan 1, 1800, and Oct 26, 2016, 13:40:

call echo(cnvtmin2(cnvtdate(10262016),1340)) go 
call echo(cnvtmin2(cnvtdate(10262016),134059,2)) go
