The result of DATETIMECMP() is the number of days between two rdb_date_time values. The time is ignored.

Beginning in Discern Explorer Version 8.10.4, DATETIMECMP(rdb_date_time1,rdb_date_time2) will behave like DATETIMECMPUTC(rdb_date_time1,rdb_date_time2,2) if coordinated universal time (UTC) is enabled for the domain. This adjusts the rdb_date_time1 and rdb_date_time2 from UTC to local time before making the comparison.  However, if the first argument of the DATETIMECMP() is using CNVTDATETIMEUTC(), it will not convert to local before the calculation.

Syntax

DATETIMECMP(rdb_date_time1,rdb_date_time2)

Metasymbols

rdb_date_time1
A datetime value that is the higher Julian datetime value when compared to rdb_date_time2.

rdb_date_time2
A datetime value that is the lower Julian datetime value when compared to rdb_date_time1.

Examples

The example below demonstrates the syntax. If the update date and time is December 2, 1998, and the current date is December 10, 1998, D1 is set equal to 8.

D1 = DATETIMECMP(CNVTDATETIME(CURDATE,0), P.UPDT_DT_TM )
