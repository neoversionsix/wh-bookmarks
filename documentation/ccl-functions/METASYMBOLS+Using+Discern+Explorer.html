The expression metasymbol described below is used in the syntax of most functions.

expression
An expression is the highest level metasymbol among constants, column names, variables, and so on. The following are valid expressions:

alias.column_name
array_name [dimension]
array_name [dimension1, dimension2]
array_name [dimension1, dimension2, dimension3]
constant
expression arithmetic_operator expression
-expression
(expression)
function(expression{,expression})
parameter
record_name{->column_name}
table_name.column_name
variable

Note

Square brackets are part of the array_name syntax; they do not indicate an optional part of the syntax.

In addition to the expression metasymbol, the syntax for every aggregate function and some string functions includes the metasymbols described below.

display_template
Templates are part of display options; a template allows you to define the format that is applied to extracted data. See Display Template Using Discern Explorer for a complete list of display templates.

display_qualifier
Display qualifiers are part of display options; the display qualifier allows you to define how your data will be formatted for display. See Display Qualifier Using Discern Explorer for a complete list of display qualifiers.

display_format
Display format is part of display options; the format allows you to identify how your data is treated. See Display Format Using Discern Explorer for a complete list of display formats.

qualification
A qualification is a condition that limits the number of records to be displayed as valid matches. Qualifications can include IF, IN, and CASE statements and can be combined using the qualifiers AND, OR, NOT, and BETWEEN. Relational operators are used to define the condition. The syntax for qualification is as follows:

(qualification)
NOT qualification
qualification AND qualification
qualification OR qualification
expression relational_operator expression
expression relational_operator subquery

relational_operator
Relational operators identify the condition in a qualification. See Operators in Discern Explorer for a complete list of the relational operators.
