The CHECKPRG function is used to determine if a program exists in the object library. Returns 0 if program does not exist, 1 if program exists and is in the server cache, 2 if program exists but is not currently in the server cache. If the group_name is omitted, the function checks in the user's group library. If not found, it checks in the DBA group library. If the explicit group_name is specified, it checks in that user's group library and if not found, it checks in the DBA group library. This matches the logic used to run a program.

Syntax

CHECKPRG(program_name[:group_name][,type])

Metasymbols

program_name
The name of the program in the Discern Explorer object library. Must be in uppercase and embedded in quotes.

group_name
The name of a library group. Valid library groups are DBA and GROUP1 through GROUP99. Must be in uppercase and embedded in quotes.

type
The object type. Must be in uppercase and embedded in quotes.

"P"

Program (default)

"E"

EKModule (Discern Expert rule)

Examples

The following example checks for the existence of a program named MY_PROGRAM.

SET PRG_EXISTS = CHECKPRG("MY_PROGRAM")

If a user in the DBA group executes the following example, the call echo(checkprg("CCL_TEST:DBA","P")) command would echo 0 because the DBA program was dropped. The call echo(checkprg("CCL_TEST:GROUP1","P")) would echo either 1 or 2 depending on whether or not the GROUP1 program is in the server cache or not. The call echo(checkprg("CCL_TEST")) would echo 0 because the user is in the DBA group and a DBA version of the program does not exist.

DROP PROGRAM CCL_TEST:DBA GO
DROP PROGRAM CCL_TEST:GROUP1 GO
CREATE PROGRAM CCL_TEST:GROUP1
	call echo("hello")
END
GO

call echo(checkprg("CCL_TEST:DBA","P")) go
call echo(checkprg("CCL_TEST:GROUP1","P")) go
call echo(checkprg("CCL_TEST")) go
