Functions are programs provided with Discern Explorer that perform specific tasks. These functions can be divided into the seven categories described below:

Aggregate functions perform calculations on groups of data. They are useful for displaying running totals and subtotals with Reportwriter FOOT clauses. For example, if COUNT(P.NAME_FULL_FORMATTED) is specified in the FOOT REPORT clause, the total number of qualifying rows for the report is displayed at the end of the report. You can also count rows in a table as in, SELECT COUNT(*) FROM PERSON GO.

  • Bit functions: Bit functions perform bitwise operations on binary columns. They are used to test, set, and clear bits of a status word.
  • Math Functions: Math functions perform operations such as logarithms on binary data (constants, variables, and numeric columns).
  • Miscellaneous functions: Miscellaneous functions are functions that do not fit clearly into one of the other groups.
  • String Functions: String functions perform operations on character strings.
  • Temporal functions: Temporal functions perform operations on date and time values.
  • UAR functions: UAR functions automatically get code descriptions from pre-assigned tables, which is more efficient and avoids the need to join to those tables every time you need to get a description.

The following Discern Explorer functions are described in detail at the end of this section (after the discussion of metasymbols). Each description includes a definition of the function, its syntax, an explanation of the metasymbols used (if appropriate), and examples.

JOIN D	WHERE MONTH(P.UPDT_DT_TM) = 9
GO
