The CNVTB10B36( ) function converts a base-10 string to a base-36 string with the maximum length defined by length. This function produces values primarily used in bar coding. Before version 8.8.1, Discern Explorer used the base-36 sequence of ABCDEFGHIJKLMNOPQRSTUVWXYZ0123456789. Beginning in version 8.8.1, the mode option is used to flex the base-36 sequence.

Syntax

CNVTB10B36(base10_string,length,[mode])

Metasymbols

base10_string

An expression that is a base-10 string.

length

An integer that defines the number of bytes in a string.

mode

Available in Discern Explorer version 8.8.1 and higher. Mode is used to flex the base-36 sequence. Modes 2 and 3 were added in Discern Explorer versions 8.13.3 and 9.01.3.

Mode

Base-36 Sequence

0

Use the base-36 sequence of ABCDEFGHIJKLMNOPQRSTUVWXYZ0123456789. Each group of 9 digits is converted separately. This mode is the default and is primarily used for bar codes.

1

Use the base-36 sequence of 0123456789ABCDEFGHIJKLMNOPQRSTUVWXYZ. Each group of 9 digits is converted separately. This mode is primarily used for bar codes.

2Use the base-36 sequence of ABCDEFGHIJKLMNOPQRSTUVWXYZ0123456789 and convert from base-10 to base-36 as one value.
3Use the base-36 sequence of 0123456789ABCDEFGHIJKLMNOPQRSTUVWXYZ and convert from base-10 to base-36 as one value.

Examples

In the following example, C is set to ANY9ZH:

 C = CNVTB10B36("023001019",6)

In the following example, C is set to 0DOZP7:

 C = CNVTB10B36("023001019",6,1)
