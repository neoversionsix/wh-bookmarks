The CNVTDATETIME() function converts date and time expressions to a Binary System Datetime value. A Binary System datetime is a value that represents a specific time on a specific date. A Binary System datetime has a DQ8 data type.

Beginning with Discern Explorer version 8.7.2, CNVTDATETIME() ignores fractional seconds.

Beginning with Discern Explorer versions 8.14.3 and 9.02.3, a DQ8 value passed to CNVTDATETIME() is returned as is.

In Discern Explorer versions 9.03.0 - 9.03.4 and 8.15.0 - 8.15.4, CNVTDATETIME() may incorrectly return a UTC datetime value in a non-UTC enabled environment when passed an internal_date and internal_time or sysdate after CNVTLOOKAHEAD(), CNVTLOOKBEHIND(), or DATETIMEADD() has been executed when SET TRACE DTTMCALC is enabled on a Linux system. This issue is resolved in Discern Explorer versions 9.03.5 and 8.15.5.

Syntax

CNVTDATETIME(rdb_dt_tm)
CNVTDATETIME(character_date [character_time])
CNVTDATETIME(internal_date, internal_time)

Metasymbols

rdb_dt_tm
A column or value that stores the date and time.

character_date
A character representation of a date.

Beginning with Discern Explorer versions 8.14.4 and 9.02.4, if SET TRACE WARNING2 is enabled, an informational message is generated when an invalid character date is passed to CNVTDATETIME().

Before Discern Explorer version 8.10.0, the character_date was required to be in D-MMM-YYYY or DD-MMM-YYYY format. Beginning with Discern Explorer version 8.10.0, the character_date can be in the following formats, where monthabrev is a three- or four-character month abbreviation and monthfull is the full month name. Before Discern Explorer versions 8.16.0|9.04.0 the first three characters of the month abbreviation or full month name must match one of the values in the CURLOCALE("MONTHABBREV") string. CNVTDATETIME() performs a case and accent insensitive comparison of the first three characters of the month abbreviation or full month name to the CURLOCALE("MONTHABBREV") string to determine the month. If a match is not found, the month with the closest match is returned. Beginning in Discern Explorer versions 8.16.0|9.04.0, if the first three characters of the month abbreviation or full month name does not match one of the values in the CURLOCALE("MONTHABBREV") string, the first four characters of the month abbreviation or full month name is compared to the values in the CURLOCALE("MONTHABBREV2") string. If a match is not found, the full month name is compared to the values in the CURLOCALE("MONTFULL") string. If a match is still not found, the month with the closest match is returned.

1-Jan-15

Beginning with Discern Explorer version 8.12.0, the leading or trailing space no longer is required.

character_time
A character representation of a time. The character_time is optional. If omitted a character_time of 00:00:00 is assumed. Before Discern Explorer version 8.10.0, the character_time was required to be in the HH:MM:SS format. Only one space was required between the character_date and the character_time. Beginning with Discern Explorer version 8.10.0, if you use a two-digit year in the character_date a space must be inserted between the character_date and the character_time. If you use a four-digit year in the character_date, an optional space can be inserted between the character_date and character_time, but the space no longer is required. The character_time can be in the following formats:

Format

Example

Description

HH

01

01:00:00

HH:MM

01:15

01:15:00

HH:MM:SS

01:15:30

01:15:30

HH:MM:SS.CC

01:15:30.45

01:15:30.45

You can use .CC, but it is ignored because the RDBMS only stores datetime values down to the second.

Before Discern Explorer version 8.10.1, a single space was required between the character_date and character_time. Discern Explorer version 8.10.1 eliminated this requirement, provided the character_date uses the YYYY format for the year.

Beginning with Discern Explorer version 8.11.0, the hour, minute, and second components are not required in the character_time, provided the colon and dot characters are used as placeholders. For example, CNVTDATETIME("01-JAN-2015 :01") returns a datetime value that is one minute after midnight on 01-JAN-2015, and CNVTDATETIME("01-JAN-2015 ::01") returns a datetime value that is one second after midnight on 01-JAN-2015.

Beginning with Discern Explorer version 8.12.2, the colon (:) separating the hour, minute, and second components are not required in the character_time, provided two digits are used for each component.  For example, CNVTDATETIME("01-JAN-2015 0115") returns a datetime value that is one hour and fifteen minutes after midnight on 01-JAN-2015. CNVTDATETIME("01-JAN-2015 011530") returns a datetime value that is one hour, fifteen minutes, and thirty seconds after midnight on 01-JAN-2015.

internal_date
A Oracle internal date that is an integer value incremented by one each day with 1 (one) representing January 1, 1800. The system variable CURDATE is equal to the Oracle internal date for today.

internal_time
An integer value representing the time in HHMM or HHMMSS format, or a floating-point (f8) number representing the time in ten-millisecond units. The system variable CURTIME3 is a floating-point number representing the time in ten-millisecond units. Use CURTIME3 in most situations that require using a system time variable. The system variables CURTIME and CURTIME2 are integer values equal to the current time in HHMM or HHMMSS format. An integer value less than or equal to 2400 is assumed to be in HHMM format. Leading zeros are ignored in integer values, so both 002359 and 2359 represent 11:59 PM.

Examples

The example below returns a datetime value that represents the very earliest possible time on 07-Oct-1992. Because no time value is given, zero is used as the time value by default:

CNVTDATETIME("07-OCT-1992")

The example below returns a datetime value that represents the very earliest possible time on 01-AUG-1996. A time value is required when using this option. The 0 (zero) is treated as 000000 in this case:

CNVTDATETIME(CNVTDATE(080196), 0)

The following example returns a datetime value that represents the current date and time:

CNVTDATETIME(CURDATE, CURTIME3)

The following example returns a datetime value that represents the very last possible time for today:

CNVTDATETIME(CURDATE, 235959)

The following example qualifies records that have been updated in the past 24 hours:

WHERE P.UPDT_DT_TM BETWEEN CNVTDATETIME(CURDATE-1, CURTIME3) AND CNVTDATETIME(CURDATE , CURTIME3)
