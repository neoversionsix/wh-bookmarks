The CNVTDATETIMEUTC2() function allows you to convert a string that represents a date and a string that represents a time to and from a UTC datetime value.

Beginning with Discern Explorer version 8.7.2, CNVTDATETIMEUTC2() ignores fractional seconds.

Syntax

CNVTDATETIMEUTC2(character_date, date_input_format, character_time, time_input_format,[option [,timezone_index]])

Metasymbols

character_date
A character description of a date

date_input_format
The format of a date that is being converted. The following are valid format characters:

Format Characters

Description

DD

Two-digit day.

MM

Two-digit month.

MMM[MMMMMMMMM]

The month can be represented using a month abbreviation or the full month name. Before version 8.10.0 of Discern Explorer a three character month abbreviation or full month name was required. Version 8.10.0 of Discern Explorer introduced the ability to use four-character month abbreviations in non-English locales. In English locales the longest month name is 9 characters. However, a full month name can be up to 12 characters to allow for longer month names in non-English locales.  Before Discern Explorer versions 8.16.0|9.04.0 the first three characters of the month abbreviation or full month name must match one of the values in the CURLOCALE("MONTHABBREV") string. CNVTDATETIMEUTC2() performs a case and accent insensitive comparison of the first three characters of the month abbreviation or full month name to the CURLOCALE("MONTHABBREV") string to determine the month. If a match is not found, the month with the closest match is returned. Beginning in Discern Explorer versions 8.16.0|9.04.0, if the first three characters of the month abbreviation or full month name does not match one of the values in the CURLOCALE("MONTHABBREV") string, the first four characters of the month abbreviation or full month name is compared to the values in the CURLOCALE("MONTHABBREV2") string. If a match is not found, the full month name is compared to the values in the CURLOCALE("MONTFULL") string. If a match is still not found, the month with the closest match is returned.

YY

Two-digit year.

YYYY

Four-digit year.

other

Special literal characters, such as slashes, dashes, or spaces.

character_time
A character string that represents a time

time_input_format
The format of the time that is being converted. The following are valid format characters:

Character

Description

HH

Hours

MM

Minutes

SS

Seconds

CC

Hundredth of seconds. Allowed as an input but ignored.

other

Special literal characters, such as colons, periods, or spaces

option

The following are available options:

Option

Description

0

Noop so return datetime with no conversion

1

Convert the datetime from local to UTC if UTC mode is on else return as is

2

Convert the datetime from UTC to local if UTC mode is on else return as is

3

Convert the datetime from local to UTC regardless of UTC mode

4

Convert the datetime from UTC to local regardless of UTC mode

timezone_index

Index number of the time zone to use for the conversion. Defaults to local time zone. This option allows emulating another target time zone for datetime conversions.

Examples

If the CurTimeZoneApp is America/Chicago, The Call Echo() command in the following example returns 11-Mar-2007 05:30:00 Central Daylight Time if UTC is on:

set my_dt_tm = cnvtdatetimeutc2("20070311","YYYYMMDD","023000","HHMMSS",1,35)
call echo(datetimezoneformat(my_dt_tm,curtimezoneapp,"DD-MMM-YYYY HH:mm:ss ZZZZZZ"))

If the CurTimeZoneApp is America/Chicago, and UTC is on, the Call Echo() command in the following example will return the following based on each iteration of the for loop changing the value of the option parameter used by CnvtDateTimeUTC2():

10-Mar-2007 20:30:00 Central Standard Time
11-Mar-2007 05:30:00 Central Daylight Time
10-Mar-2007 12:30:00 Central Standard Time
11-Mar-2007 05:30:00 Central Daylight Time
10-Mar-2007 12:30:00 Central Standard Time

for (opt = 1 to 5)
	set option = opt -1
	set my_dt_tm = cnvtdatetimeutc2("20070311","YYYYMMDD","023000","HHMMSS",option,35)
	call echo(datetimezoneformat(my_dt_tm,curtimezoneapp,"DD-MMM-YYYY HH:mm:ss ZZZZZZ"))
endfor
