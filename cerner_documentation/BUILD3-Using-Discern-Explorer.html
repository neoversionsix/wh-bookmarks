Release Considerations

Build3 is introduced in Discern Explorer version 8.12.6.

The BUILD3() function allows you to return a concatenated expression based on the mode, with integer and float (numeric) values converted to strings. Float values are rounded to two decimal places. Float values are displayed using up to 14 characters including the decimal point. Integer values are displayed using up to 11 characters.

Syntax

BUILD3(mode,expression{,expression})

Metasymbols

expression

A value of any data type

mode

OptionDescription
1Do not remove leading spaces from numeric expressions. If a converted numeric expression exceeds 14 characters, it is truncated.
2Do not remove leading spaces from numeric expressions. If a converted numeric expression exceeds 14 characters, it is replaced with asterisks.
3Remove leading spaces from numeric expressions. If a converted numeric expression exceeds 14 characters, it is truncated.
4

Remove leading spaces from numeric expressions. If a converted numeric expression exceeds 14 characters, it is filled with asterisks.

Examples

The following example echoes the options listed on the above table:
Mode:          1 ABC          12.35 DEFG        789 HIJ         789.12 KLM 12345678901234
Mode:          2 ABC          12.35 DEFG        789 HIJ         789.12 KLM **************
Mode:3 ABC 12.35 DEFG789 HIJ 789.12 KLM 12345678901234
Mode:4 ABC 12.35 DEFG789 HIJ 789.12 KLM **************
drop program ccl_test_build3 go
create program ccl_test_build3
for(mode_cnt = 1 to 4)
    call echo(BUILD3(mode_cnt, "Mode:", mode_cnt, " ", "ABC ",12.3456," ", "DEFG",789," HIJ ", 789.123, " KLM ", 123456789012345.123)) 
endfor
end go
ccl_test_build3 go
