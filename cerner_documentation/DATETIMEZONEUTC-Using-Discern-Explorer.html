The DATETIMEZONEUTC( ) function returns the adjusted datetime based on the time zone index. The FORMAT( ) function or a Display Option with a display qualifier of 4 can be used to display the adjusted datetime value in a meaningful format. If Coordinated Universal Time (UTC) mode is turned off, then DATETIMEZONEUTC( ) returns the datetime as is. No conversion is performed.

Syntax

DATETIMEZONEUTC(<datetime>,<timezone_index>[,<option>])

Metasymbols

datetime
A datetime value.

timezone_index
An integer representing the time zone. If timezone_index is zero, then the application time zone is returned.

Note

Oracle uses a Windows standard that has defined time zone names and their associated index numbers.

option

Option

Description

1

Converts a UTC datetime to the local datetime using the index, and then back to UTC (the default option if omitted).

2

Converts the local datetime to a UTC datetime using the index, and then back to local.

Note

If UTC mode is turned off, the function returns the datetime as is and performs no conversion.

Examples

When the following example is executed in an environment set to Central Daylight Time (CDT) with UTC turned on, the CALL ECHO command displays 27-JUN-2013 04:59:59.00, since 75 is the time zone index for Central Daylight Time:

SET C_DT_TM = DATETIMEZONEUTC(CNVTDATETIME("26-Jun-2013 23:59:59"),75,1) GO
CALL ECHO(FORMAT(C_DT_TM,";4;Q")) GO

When the example below is executed in an environment set to Central Daylight Time (CDT) with UTC turned on, the CALL ECHO command displays 27-JUN-2013 02:59:59.00, since 71 is the time zone index for Arizona Time. Arizona does not implement daylight saving time, resulting in a difference of two hours between Arizona time and Central Daylight time, and a one hour difference is between Arizona time and Central Standard time.

SET A_DT_TM = DATETIMEZONEutc(CNVTDATETIME("26-Jun-2013 23:59:59"),71,1) GO
CALL ECHO(FORMAT(A_DT_TM,";4;Q")) GO

When the example below is executed in an environment set to Central Daylight Time (CDT) with UTC turned on, the CALL ECHO command displays 21. The CNVTDATETIME( ) function returns the UTC datetime for 26-JUN-2013 23:59:59 in CDT. The DATETIMEZONEUTC( ) function adjusts that datetime by two hours because of the difference between CDT and Arizona Time. The HOUR( ) function converts that datetime to local time and returns 21.

CALL ECHO(HOUR(DATETIMEZONEUTC(CNVTDATETIME("26-JUN-2013 23:59:59"),71))) GO
