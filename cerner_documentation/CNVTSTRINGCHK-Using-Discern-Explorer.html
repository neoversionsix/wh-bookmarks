CNVTSTRINGCHK ( ) converts a numeric value to a string value. Checks for overflow on the character expression and fills with asterisks if a truncation occurs. The returned string is 11 characters in length by default. Extraneous leading or trailing spaces can be removed using the TRIM( ) function. If the numeric value contains more than 11 digits, a length must be specified to prevent the result from truncating. The decimal and digits to the right of the decimal are ignored by default. If the numeric value includes digits to the right of a decimal point, a length and precision must be specified to prevent those digits from being ignored.

Before Discern Explorer version 8.10.2. If the number of digitss to the left of the decimal exceed the specified length, asterisks are returned. The number of asterisks returned is 11 or the specified length. If the number of digits to the left of the decimal is less than the specified length, one decimal mark and a number of digits up to the specified precision, is returned to fill out to the total specified length for the string.

In Discern Explorer version 8.10.2 and higher. If the total number of digits, including the decimal and digits to the right of the decimal, exceed the specified length, asterisks are returned. The number of asterisks returned is 11 or the specified length.

Syntax

CNVTSTRINGCHK(numeric_value[,length[,precision,justify]])

Metasymbols

justify
Defines whether the converted string is left-justified or right-justified. When this option is not specified, Left-Justification is used. Length and precision must also be specified. The following option is valid:
R Right-justified

length
An integer that defines the number of characters in the returned string. If not specified, a default of 11 is used.

numeric_value
An expression that is a numeric value.

precision
An integer representing the number of digits to the right of the decimal. Length must also be specified.

Examples

In the below example, NEWSTRING is set to 789.

NEWSTRING = CNVTSTRINGCHK(789,3)

In the below example, NEWSTRING1 is set to 1234567890123.456.

NEWSTRING1 = CNVTSTRINGCHK(1234567890123.456 ,17,3)

Before Discern Explorer version 8.10.2, NEWSTRING1 in the below example was set to 123456789.12. The example specifies a length of 12, so the total return length is 12 characters. The number of digits to the left of the decimal is less than 12 characters so the asterisks are not returned. The total length of the input number including the decimal is 13, so the last digit to the right of the decimal is dropped and the 12 character string is returned. In Discern Explorer version 8.10.2 and higher, NEWSTRING1 is set to ************. Since the example specifies a length of 12 and the total length of the input number is 13, the asterisks are returned to indicate an overflow.

NEWSTRING1 = CNVTSTRINGCHK(123456789.123 ,12,3)

In the below example, NEWSTRING2 is set to ***********. Since the numeric expression exceeds 11 characters and length and precision are not used, the resulting character string is truncated to 11 characters and filled with asterisks to indicate the truncation occurred.

NEWSTRING2 = CNVTSTRINGCHK(1234567890123.456)

In the below example, NEWSTRING3 is set to 123. Since length and precision are not used, the resulting character string does not include the digits that were to the right of the decimal.

NEWSTRING3 = CNVTSTRINGCHK(123.456)
