    Overview

    This Reference Page is a guide for identifying which servers must be cycled after you make certain changes. This is not intended to be a complete server reference, and additional items can be omitted. See the Servers & Services Reference Pages for information on specific servers. 

    Note

    When code values are added or updated, the code cache must be updated with the new information. The appropriate method is to allow the code cache to be refreshed automatically. The code cache is refreshed every 10 minutes by default and refreshes only code values that have changed. The second method is to enter Crmtest 6024 to update the specified code set. Crmtest refreshes every value in the specified code set, not just the changes. If the code set is large, performance can be impacted. The third method is to use the Code utility to insert the code set. The Code utility is available only to the root system administrator.

    Release Considerations

    All Millennium releases before the Cerner Millennium 2012.01.24 Service Release use CPM Code Cache Manager server (SCP 52). With the Cerner Millennium 2012.01.24 Service Release, the CPM Code Cache Manager server was split into two servers to isolate the two types of actions the server performed previously. The CPM Code Cache Manager server is updated to be the CPM Code Cache Query server (SCP 52) for all queries to the code cache. The CPM Code Cache Refresh server (SCP 49) is added with this service package for all updates to the code cache.

    Note

    When cycling the CPM Script Server (SCP 51), you must cycle all servers that include CPMSCRIPT#### services and settings.
